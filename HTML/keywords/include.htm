Справочник ключевых слов

#include

Включает файл в текущий скрипт.

<strong>#include</strong> <filename>
<strong>#include</strong> "[path\]filename"

Параметры

filename Имя файла текущего скрипта для включения. Путь необязателен. Это должна быть строка – это не может быть переменной.
Если используется <...>, имя файла принимается как относительное к каталогу библиотеки включения (обычно C:\Program Files\AutoIt3\Include). Библиотека включения содержит много предварительно написанных пользовательских функций для использования!
Если используется "...", имя файла принимается как относительное к текущему скрипту.

Примечания

Другие скрипты могут быть включены в скрипт Autoit с помощью команды #include – они могут быть в формате .au3 или .a3x. Содержимое включённого файла вставляется в скрипт в точке команды #include – в большинстве случаев это начало скрипта, так что объявления переменных или констант в включённом тексте доступны для остальной части скрипта.

Если вы включите один и тот же файл, содержащий пользовательскую функцию, более одного раза, вы получите ошибку "Duplicate function". При написании файла включения, который может использоваться таким образом, убедитесь, что первая строка содержит #include-once, чтобы предотвратить включение этого файла более одного раза.

Есть специальное значение реестра, которое можно создать в "HKEY_CURRENT_USER\Software\AutoIt v3\AutoIt", называемое "Include". Это должно быть значение REG_SZ (строка). Содержимое этого значения – это список каталогов, разделённых точками с запятой, которые должны быть перепроверены для файлов при разрешении #include в дополнение к стандартным местоположениям.

Порядок поиска, используемый AutoIt, зависит от того, какую форму #include вы используете. Таблицы ниже показывают порядок поиска каталогов, используя обе формы.

Использование #include <>

Стандартная библиотека Путь текущего работающего интерпретатора с добавленным "\Include" ищется.
Пользовательские библиотеки Значение реестра, упомянутое выше, читается и каждый каталог ищется в порядке их появления.
Каталог скрипта Каталог текущего выполняемого скрипта.

Использование #include "" (Это противоположность #include <>).
Каталог скрипта Каталог текущего выполняемого скрипта.
Пользовательские библиотеки Значение реестра, упомянутое выше, читается и каждый каталог ищется в обратном порядке их появления.
Стандартная библиотека Путь текущего работающего интерпретатора с добавленным "\Include" ищется.

Примечание об использовании опции /AutoIt3ExecuteScript. Так как стандартная библиотека ищется в каталоге текущего интерпретатора, функции стандартной библиотеки не будут найдены; эта библиотека будет найдена только при запуске через AutoIt3.exe. Рекомендуется скомпилировать скрипт в формате .a3x перед попыткой запуска его с /AutoIt3ExecuteScript.

Aut2Exe использует тот же алгоритм, что и AutoIt3.exe, с единственной разницей в том, что он ищет подкаталог Include как находящийся в соседнем каталоге самого себя (..\Include).

Если Opt("TrayIconDebug",1), только 64 имена файлов включения могут отображаться в traytooltip. для остальных имя файла не будет отображаться.

Пример

#include "include-TIME.au3"

#include <MsgBoxConstants.au3>

; Запуск "include.au3" выведет два окна сообщения:
; одно со временем, затем одно с "Example"

MsgBox($MB_SYSTEMMODAL, "MsgBox #2", "Example")

Exit
