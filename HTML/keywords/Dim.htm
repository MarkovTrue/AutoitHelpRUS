Справочник ключевых слов

Dim / Global / Local / Const

Объявить переменную, константу или создать массив.

<strong>Global | Local</strong> [Const] $variable [ = initializer ]
<strong>Global | Local</strong> [Const] $aArray<strong>[</strong>subscript 1<strong>]</strong>...<strong>[</strong>subscript n<strong>]</strong> [ = initializer ]

Параметры

Const [необязательный] Если присутствует, ключевое слово Const создаёт константу вместо переменной.
$variable Имя объявляемой переменной/константы.
initializer Значение, которое будет изначально присвоено переменной. Const должен включать инициализатор. Инициализатор может быть вызовом функции.
subscript Количество элементов измерения массива (нумерация с 0 до n-1).

Примечания

Ключевые слова Dim/Local/Global выполняют схожие функции:
1. Объявить переменную перед её использованием (как в VBScript)
2. Создать массив

Примечание: В AutoIt вы можете создать переменную просто присвоив значение ($myvar = 0), но многие люди любят явно их объявлять. Если активно AutoItSetOption("MustDeclareVars", 1), то переменные должны быть объявлены до использования.

Вы также можете объявить несколько переменных на одной строке:

Local $vVar_1, $vVar_2, $vVar_3


И инициализировать переменные:

Local $vVar_1 = 10, $vVar_2 = "20", $vVar_3 = 30


Создание констант можно делать аналогичным образом:

Const $CONST_1 = 1, $CONST_2 = 2, $CONST_3 = 3
Global Const $PI = 3.14, $MEANING_OF_LIFE = 42
Local Const $iApples = 500


Однажды созданную константу нельзя изменить. Также нельзя преобразовать существующую переменную в константу.

Для инициализации массива укажите значения для каждого элемента в квадратных скобках, разделённые запятыми. Для нескольких измерений вложите инициализаторы. Вы можете указать меньше элементов в инициализаторе, чем объявлено, но не больше. Вызовы функций также можно поместить в инициализаторы массива. Если вызов функции возвращает массив, то один элемент массива будет содержать возвращённый массив.

Local $aArray_1[12] = [3, 7.5, "string"], $aArray_1[5] = [8, 4, 5, 9, 1]
Local $aGrid[2][4] = [["Paul", "Jim", "Richard", "Louis"], [485.44, 160.68, 275.16, 320.00]]
Global $aTest[5] = [3, 1, StringSplit("Abe|Jack|Bobby|Marty", "|"), Cos(0)]


Разница между Dim, Local и Global в области видимости, в которой они создаются:
Dim = Локальная область видимости, если имя переменной не существует глобально (в этом случае переиспользуется глобальная переменная!)
Global = Принудительно создаёт переменную в глобальной области видимости
Local = Принудительно создаёт переменную в локальной области видимости/области видимости функции

Вы должны использовать Local или Global вместо Dim, чтобы явно указать, какая область видимости требуется для переменной/константы/массива.

При использовании переменных сначала проверяется локальная область видимости, затем глобальная.

При создании массивов вы ограничены до 64 измерений и/или всего 16 миллионов элементов.

Особенность AutoIt – возможность копировать массивы вот так:
$mycopy = $myarray
В этом случае $mycopy будет точной копией $myarray и будет иметь те же размеры – инструкция Dim не требуется для предварительного изменения размера массива. Если активно AutoItSetOption ( "MustDeclareVars", 1), то переменная $mycopy всё равно должна быть объявлена первой, но не требуется её размер. Если переменная $mycopy уже была массивом или одним значением, она будет стёрта перед выполнением копии.

Для удаления массива (может быть, потому что это большой глобальный массив и вы хотите освободить память), просто присвойте ему одно значение:
$aArray = 0
Это освободит массив и преобразует его обратно в одиночное значение 0.

Объявление того же имени переменной снова сотрёт все значения массива и переустановит размеры на новое определение. Объявление переменной с простым значением в той же области видимости не изменит значение в переменной.

Если вы объявляете переменную с тем же именем, что и параметр, используя Local внутри пользовательской функции, произойдёт ошибка. Global можно использовать для присвоения глобальным переменным внутри функции, но если локальная переменная (или параметр) имеет то же имя, что и глобальная переменная, будет использоваться только локальная переменная. Рекомендуется давать локальным и глобальным переменным разные имена.

См. также

AutoItSetOption, UBound, ReDim, Static

Пример

Пример 1

; Объявление переменных
Local $i, $j = 23, $k
Global $g_fPI = 3.14159, $g_iRADIUS
Local $iDaysWorking = 5

; Объявление массивов
Global $g_aChessBoard[8][8]
Global $g_aEmptyArray[0]
Global $g_aAutoSize[] = [1, 2, 3, 4]
Local $aStates[2], $aWindowsStats[4]
Local $a2Dimensions[2][3] = [[1, 2, 3], [4, 5, 6]]

; Объявление переменных-констант
Const $iX1 = 11, $iY1 = 23, $iZ1 = 55
Global Const $PI = 3.14159, $E = 2.71828
Local Const $DAYS_WORKING = 5

#include <MsgBoxConstants.au3>

Dim $vVariableThatIsGlobal = "This is a variable that has ""Program Scope"" aka Global."

MsgBox($MB_SYSTEMMODAL, "", "An example of why Dim can cause more problems than solve them.")

Example()

Func Example()
    ; Это выглядит нормально, так как отображает следующий текст: This is a variable that has "Program Scope" aka Global.
    MsgBox($MB_SYSTEMMODAL, "", $vVariableThatIsGlobal)

    ; Вызвать случайную функцию.
    Local $vReturn = SomeFunc()

    ; Глобальная переменная ($vVariableThatIsGlobal) изменилась, так как я совершенно забыл, что у меня было дублирующееся имя переменной в "SomeFunc".
    MsgBox($MB_SYSTEMMODAL, $vReturn, "The variable has now changed: " & $vVariableThatIsGlobal)
EndFunc   ;==>Пример

Func SomeFunc()
    ; Это должно создать переменную в локальной области видимости, если имя переменной ещё не существует.
    ; Ради аргумента я совершенно забыл, что уже объявил переменную с тем же именем.
    ; Ну, я хочу, чтобы это менялось только в функции, а не в переменной вверху скрипта.
    ; Должно быть хорошо, правда? Подумайте ещё раз.
    Dim $vVariableThatIsGlobal = ""

    For $i = 1 To 10
        $vVariableThatIsGlobal &= $i ; Это вернёт 12345678910, полностью стирая предыдущее содержимое $vVariableThatIsGlobal.
    Next
    Return $vVariableThatIsGlobal
EndFunc   ;==>SomeFunc
