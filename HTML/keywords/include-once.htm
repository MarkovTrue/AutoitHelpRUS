Справочник ключевых слов

#include-once

Указывает, что текущий файл должен быть включен только один раз.

#include-once

Примечания

Довольно часто одна и та же строка "#include <filename.au3>" встречается в нескольких файлах, включённых в скрипт. Если бы один и тот же файл был включен несколько раз, это, вероятно, создало бы ошибку "Duplicate function" или "Cannot redeclare a Const". Поэтому при написании скрипта, предназначенного для использования в качестве файла включения, добавьте #include-once, чтобы предотвратить включение этого файла более одного раза. Обратите внимание, что строка #include-once должна быть размещена в верхней части скрипта и перед любыми другими строками #include.

Не рекомендуется добавлять строку #include-once к скриптам, которые не предназначены для использования в качестве файлов включения в других скриптах.

См. также

#include

Пример

; Использовать #include-once при создании UDF, чтобы файл не был включен несколько раз
; если вы вызываете один и тот же UDF в несколько файлов.

#include-once

#include <MsgBoxConstants.au3>

; Добавить #include "Include-once.au3" в верхнюю часть вашего скрипта.
; Тогда ваш скрипт может вызвать IncludeFunc().
; Выполнение выведет MsgBox.

Func IncludeFunc()
    MsgBox($MB_SYSTEMMODAL, "", "This is an example of including a file.")
EndFunc   ;==>IncludeFunc
