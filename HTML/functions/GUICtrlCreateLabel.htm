Справочник функций

GUICtrlCreateLabel

Создаёт статический элемент управления Label (подпись) для GUI.

GUICtrlCreateLabel ( "text", left, top [, width [, height [, style = -1 [, exStyle = -1]]]] )

Параметры

text Текст элемента управления.
left Левая граница элемента управления. Если указано -1, значение будет вычислено согласно GUICoordMode.
top Верхняя граница элемента управления. Если указано -1, значение будет вычислено согласно GUICoordMode.
width [необязательный] Ширина элемента управления (по умолчанию – автоподгонка по ширине текста).
height [необязательный] Высота элемента управления (по умолчанию – автоподгонка по высоте текста).
style [необязательный] Задаёт стиль элемента управления. См. приложение "Стили элементов управления GUI".
по умолчанию (-1) : нет.
принудительные стили : $SS_NOTIFY, $SS_LEFT
exStyle [необязательный] Задаёт расширенный стиль элемента управления. См. таблицу расширенных стилей.

Возвращаемое значение

Успех: Возвращает идентификатор нового элемента управления (controlID).
Ошибка: Возвращает 0.

Примечания

Для установки или изменения параметров элемента управления см. функции GUICtrlUpdate...().

Для объединения стилей со стилем по умолчанию используйте BitOR($GUI_SS_DEFAULT_LABEL, новый_стиль, ...).
Для использования указанных выше значений необходимо включить в скрипт #include <StaticConstants.au3>.

Изменение размера по умолчанию: $GUI_DOCKAUTO – автоматическое изменение размера и позиции.

Расширенный стиль $GUI_WS_EX_PARENTDRAG позволяет перетаскивать родительское окно для окон без заголовка (без стиля $WS_CAPTION в GUICreate()).

Чтобы сделать фон прозрачным, используйте GUICtrlSetBkColor(-1, $GUI_BKCOLOR_TRANSPARENT).

Если необходимо создать подпись, перекрывающую другие элементы управления, задайте стиль $WS_CLIPSIBLINGS.
Это незначительно снижает производительность, как описано в форуме $WS_CLIPSIBLINGS.

См. также

GUICoordMode (Option), GUICtrlUpdate..., GUIGetMsg

Пример

Пример 1

#include <GUIConstantsEx.au3>

Example()

Func Example()
    GUICreate("My GUI") ; создаст диалоговое окно, отображаемое по центру

    GUISetHelp("notepad.exe") ; при нажатии F1 запускает Notepad
    Local $iOldOpt = Opt("GUICoordMode", 2)

    Local $iWidthCell = 70
    GUICtrlCreateLabel("Line 1 Cell 1", 10, 30, $iWidthCell) ; первая ячейка, ширина 70
    GUICtrlCreateLabel("Line 2 Cell 1", -1, 0) ; следующая строка
    GUICtrlCreateLabel("Line 3 Cell 2", 0, 0) ; следующая строка и следующая ячейка
    GUICtrlCreateLabel("Line 3 Cell 3", 0, -1) ; следующая ячейка той же строки
    GUICtrlCreateLabel("Line 4 Cell 1", -3 * $iWidthCell, 0) ; следующая строка, ячейка 1

    GUISetState(@SW_SHOW) ; отобразит пустое диалоговое окно

    ; Цикл до выхода пользователя.
    While 1
        Switch GUIGetMsg()
            Case $GUI_EVENT_CLOSE
                ExitLoop

        EndSwitch
    WEnd

    $iOldOpt = Opt("GUICoordMode", $iOldOpt)
EndFunc   ;==>Example

GUICtrlCreateLabel() overlapping controls

; == GUICtrlCreateLabel() overlapping controls

#include <GUIConstantsEx.au3>
#include <WindowsStylesConstants.au3>

Local $hGUI = GUICreate("Example", 300, 200)

Local $idLabel = GUICtrlCreateLabel(" Label overlapping the whole client area", 0, 0, 300, 200, $WS_CLIPSIBLINGS)
; $WS_CLIPSIBLINGS даёт небольшую просадку производительности
; см. https://social.msdn.microsoft.com/Forums/en-US/dcd6a33c-2a6f-440f-ba0b-4a5fa26d14bb/when-to-use-wsclipchildren-and-wsclipsibilings-styles?forum=vcgeneral
; лучше не накладывать элементы управления
;~ Local $idLabel = GUICtrlCreateLabel(" Label NOT OVERLAPPING others controls", 0, 0, 290, 170)

Local $idBtn_Close = GUICtrlCreateButton("Close", 210, 170, 85, 25)
GUICtrlSetState(-1, $GUI_ONTOP)

GUISetState()

While True
    Switch GUIGetMsg()
        Case $GUI_EVENT_CLOSE, $idBtn_Close
            ExitLoop
        Case $idLabel
            ConsoleWrite("Label" & @CRLF)
    EndSwitch
WEnd
