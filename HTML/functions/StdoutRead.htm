Справочник функций

StdoutRead

Читает данные из потока STDOUT ранее запущенного дочернего процесса.

StdoutRead ( process_id [, peek = False [, binary = False]] )

Параметры

process_id Идентификатор дочернего процесса (PID), возвращённый предыдущим вызовом Run.
peek [необязательный] Если True, прочитанные символы не удаляются из потока.
binary [необязательный] Если True, данные читаются в двоичном формате вместо текстового (по умолчанию текст).

Возвращаемое значение

Успех: Возвращает прочитанные данные; @extended содержит количество прочитанных байт.
Ошибка: Устанавливает @error ≠ 0 при достижении EOF, если STDOUT не был перенаправлен или при иной ошибке.

Примечания

StdoutRead() читает данные из стандартного потока вывода консольного дочернего процесса, который обычно используется консольными приложениями для вывода на экран. При вызове Run() для дочернего процесса параметр STD I/O должен включать значение $STDOUT_CHILD (2) для корректной работы (см. Run()).
StdoutRead() не блокирует выполнение и возвращает управление немедленно. Для получения всех данных нужно вызывать функцию в цикле.
Просмотр потока (peek) не удаляет данные из буфера, но возвращает доступные данные в обычном режиме.
По умолчанию данные возвращаются в текстовом формате. При указании двоичного режима данные возвращаются в двоичном формате.

См. также

Run, RunAs, StderrRead, StdinWrite, StdioClose

Пример

#include <Array.au3> ; Нужно только для _ArrayDisplay.
#include <AutoItConstants.au3>
#include <MsgBoxConstants.au3>

; Рекурсивно показать список файлов каталога.
Example()

Func Example()
    Local $sFilePath = @ScriptDir ; Искать в каталоге текущего скрипта.
    Local $sFilter = "*.*" ; Искать все файлы текущего каталога. Список валидных wildcards – по 'Wildcards' в справке.

    ; Если путь не является каталогом, выйти из функции Example.
    If Not StringInStr(FileGetAttrib($sFilePath), "D") Then
        Return SetError(1, 0, 0)
    EndIf

    ; Убрать хвостовые обратные слэши и добавить один в конце.
    $sFilePath = StringRegExpReplace($sFilePath, "[\\/]+\z", "") & "\"

    #cs
Параметры командной строки DIR:
/B – простой вывод.
/A-D – искать все файлы, исключая папки.
/S – искать в подпапках.
#ce
    Local $iPID = Run(@ComSpec & ' /C DIR "' & $sFilePath & $sFilter & '" /B /A-D /S', $sFilePath, @SW_HIDE, $STDOUT_CHILD)
    ; Для поиска файлов с именами в кодировке Unicode используйте параметр командной строки /U.

    ; Ждать завершения процесса с PID, возвращённым функцией Run.
    ProcessWaitClose($iPID)

    ; Читать поток Stdout процесса с PID, возвращённым Run. Можно также использовать цикл While – см. пример для StderrRead.
    Local $sOutput = StdoutRead($iPID)

    ; Разбить вывод StdoutRead на массив через StringSplit. Символы возврата каретки (@CRLF) убираются, разделитель – @CRLF (перевод строки).
    Local $aArray = StringSplit(StringTrimRight(StringStripCR($sOutput), StringLen(@LF)), @LF)
    If @error Then
        MsgBox($MB_SYSTEMMODAL, "", "It appears there was an error trying to find all the files in the current script directory.")
    Else
        ; Показать результаты.
        _ArrayDisplay($aArray)
    EndIf
EndFunc   ;==>Example
