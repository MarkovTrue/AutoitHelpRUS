Справочник функций

Run

Запускает внешнюю программу.

Run ( "program" [, "workingdir" [, show_flag [, opt_flag]]] )

Параметры

program Полный путь к запускаемой программе (EXE, BAT, COM или PIF; см. раздел Примечания).
workingdir [необязательный] Рабочий каталог. Пустая строка ("") означает текущий рабочий каталог. Это не путь к программе.
show_flag [необязательный] Флаг отображения окна запускаемой программы:
@SW_HIDE – скрытое окно (или ключевое слово Default)
@SW_MINIMIZE – свёрнутое окно
@SW_MAXIMIZE – развёрнутое окно.
opt_flag [необязательный] Управляет взаимодействием родительского и дочернего процессов.
$STDIN_CHILD (0x1) = предоставить дескриптор потока STDIN дочернего процесса
$STDOUT_CHILD (0x2) = предоставить дескриптор потока STDOUT дочернего процесса
$STDERR_CHILD (0x4) = предоставить дескриптор потока STDERR дочернего процесса
$STDERR_MERGED (0x8) = один дескриптор для STDOUT и STDERR; подразумевает $STDOUT_CHILD и $STDERR_CHILD
$STDIO_INHERIT_PARENT (0x10) = передать дочернему процессу потоки STDIO родителя; несовместим с другими STDIO-флагами; полезен только если родитель скомпилирован как консольное приложение
$RUN_CREATE_NEW_CONSOLE (0x10000) = дочерний консольный процесс создаётся в собственном окне, а не в окне родителя; полезен только для консольных приложений

Константы определены в AutoItConstants.au3.

Возвращаемое значение

Успех: Возвращает PID запущенного процесса.
Ошибка: Возвращает 0 и устанавливает @error ≠ 0.

Примечания

Пути, содержащие пробелы, необходимо заключать в кавычки.

Для выполнения DOS (консольных) команд используйте: Run(@ComSpec & " /c " & 'commandName', "", @SW_HIDE)    ; не забудьте пробел перед "/c"

После запуска программы выполнение скрипта продолжается. Чтобы дождаться завершения дочернего процесса, используйте RunWait().

Передав нужные значения в параметр Standard I/O, вы можете взаимодействовать с дочерним процессом через StderrRead(), StdinWrite() и StdoutRead(). Складывайте флаги (или используйте $STDERR_CHILD, $STDIN_CHILD, $STDOUT_CHILD) для управления несколькими потоками.

Для закрытия потоков необходимо выполнение следующих условий:
1) Дочерний процесс закрыл свой конец потока (это происходит при завершении процесса).
2) AutoIt должен прочитать все захваченные потоки до конца данных.
3) Если дочернему процессу предоставлен STDIN, необходимо вызвать StdinWrite() для его закрытия. После того как все потоки признаются ненужными, внутренние ресурсы освобождаются автоматически.

StdioClose() позволяет принудительно закрыть потоки STDIO.

См. также

ConsoleRead, ProcessClose, RunAs, RunAsWait, RunWait, ShellExecute, ShellExecuteWait, StderrRead, StdinWrite, StdioClose, StdoutRead

Пример

Example()

Func Example()
    ; Запустить Notepad с развёрнутым окном.
    Local $iPID = Run("notepad.exe", "", @SW_SHOWMAXIMIZED)

    ; Ждать до 10 секунд появления окна Блокнота.
    WinWait("[CLASS:Notepad]", "", 10)

    ; Пауза 2 секунды.
    Sleep(2000)

    ; Закрыть процесс Блокнота по PID, возвращённому Run.
    ProcessClose($iPID)
EndFunc   ;==>Example
