Справочник функций

AutoItSetOption

Изменяет поведение различных функций и параметров AutoIt.

AutoItSetOption ( "option" [, param] )

Параметры

option Опция для изменения. См. раздел Примечания.
param [необязательный] Значение, присваиваемое опции. Тип и смысл зависят от самой опции. См. раздел Примечания. Если параметр не задан, функция просто возвращает текущее значение опции. Для сброса опции к значению по умолчанию можно использовать ключевое слово Default.

Возвращаемое значение

Успех: Возвращает предыдущее значение указанной опции.
Ошибка: Устанавливает @error ≠ 0. Ошибка возникает при недопустимых параметрах (например, если опции не существует).

Примечания

Opt() можно использовать как альтернативу AutoItSetOption().

Доступные опции:

Опция Параметр
CaretCoordMode Задаёт способ использования координат в функциях работы с кареткой – абсолютные или относительно текущего активного окна:
0 = относительно активного окна
1 = (по умолчанию) абсолютные экранные координаты
2 = относительно клиентской области активного окна
ExpandEnvStrings Изменяет интерпретацию литеральных строк и символов %. По умолчанию строки воспринимаются буквально. Эта опция позволяет использовать переменные %environment% внутри строк, например "Временный каталог: %temp%".
0 = (по умолчанию) не разворачивать переменные окружения
1 = разворачивать переменные окружения
Без этой опции обычный способ: "Временный каталог: " & EnvGet("temp")
ExpandVarStrings Изменяет интерпретацию литеральных строк и символов переменных/макросов ($ и @). По умолчанию строки воспринимаются буквально. Эта опция позволяет использовать переменные и макросы внутри строк, например "Значение var1: $var1$".
0 = (по умолчанию) не разворачивать переменные
1 = разворачивать переменные (в этом режиме для литерального $ или @ их нужно удваивать: "Знак одного доллара $$").
GUICloseOnESC При нажатии ESC в GUI отправляется сообщение $GUI_EVENT_CLOSE. Эта опция включает и отключает такое поведение.
0 = Не отправлять $GUI_EVENT_CLOSE при нажатии ESC.
1 = (по умолчанию) Отправлять $GUI_EVENT_CLOSE через Send() при нажатии ESC.
GUICoordMode Изменяет правила позиционирования элемента управления, задаваемого GUICtrlSetPos().
0 = относительно начала предыдущего элемента управления (верхний левый угол).
1 = (по умолчанию) абсолютные координаты, всё равно относительно диалогового окна.
2 = ячейка позиционируется относительно текущей ячейки. Значение -1 для левого или верхнего параметра означает использование координаты предыдущей ячейки.
Таким образом следующая ячейка вниз (-1, смещение); следующая ячейка вправо (смещение, -1); текущая ячейка (-1, -1).
Понятно, что "смещение" не может быть равным -1, так как обозначает отсутствие сдвига. Но если вы используете кратную ширину, то выбирайте пропуск или возврат назад.
GUIDataSeparatorChar Задаёт символ-разделитель подэлементов в GUICtrlSetData().
Символ по умолчанию – '|'.
GUIOnEventMode Включает/отключает уведомления функций OnEvent.
0 = (по умолчанию) отключено.
1 = включено.
GUIResizeMode Изменяет правило изменения размера по умолчанию для элемента управления.
0 = (по умолчанию) сохранять стандартное поведение.
<1024 = любой тип изменения размера – см. GUICtrlSetResizing().
GUIEventOptions Изменяет поведение специальных событий или возвращаемые значения GUI-функций.
0 = (по умолчанию) стандартное поведение Windows при нажатии Minimize, Restore, Maximize или изменении размера.
1 = подавить стандартное поведение Windows при нажатии Minimize, Restore, Maximize или изменении размера окна – только отправляется уведомление.
MouseClickDelay Изменяет длительность короткой паузы между кликами мыши.
Время паузы в миллисекундах (по умолчанию 10).
MouseClickDownDelay Изменяет длительность удержания клика перед отпусканием.
Время паузы в миллисекундах (по умолчанию 10).
MouseClickDragDelay Изменяет длительность короткой паузы в начале и в конце операции перетаскивания мышью.
Время паузы в миллисекундах (по умолчанию 250).
MouseCoordMode Задаёт способ использования координат в функциях мыши – абсолютные или относительно текущего активного окна:
0 = относительно активного окна
1 = (по умолчанию) абсолютные экранные координаты
2 = относительно клиентской области активного окна
MustDeclareVars Если эта опция включена, все переменные перед использованием должны быть предварительно объявлены через Local, Global или (в некоторых случаях) Dim – это защищает от ошибок из-за опечаток.
0 = (по умолчанию) Переменные можно не объявлять заранее
1 = Переменные должны быть объявлены заранее. Подробности см. в Dim / Global / Local / Const.
PixelCoordMode Задаёт способ использования координат в пиксельных функциях – абсолютные или относительно окна, заданного через hwnd (по умолчанию активное окно):
0 = относительно указанного окна
1 = (по умолчанию) абсолютные экранные координаты
2 = относительно клиентской области указанного окна
SendAttachMode Указывает, подключает ли AutoIt потоки ввода при использовании функции Send(). Без подключения (режим по умолчанию 0) определение состояния CapsLock/ScrollLock/NumLock может быть ненадёжным в NT4. При режиме 1 синтаксис Send("{... down/up}") работать не будет, и могут возникать проблемы при отправке клавиш "зависшим" окнам. ControlSend() всегда подключается и не зависит от этого режима.
0 = (по умолчанию) не подключать
1 = подключать
SendCapslockMode Указывает, должен ли AutoIt сохранять состояние CapsLock перед вызовом Send() и восстанавливать его после.
0 = не сохранять/не восстанавливать
1 = (по умолчанию) сохранять и восстанавливать
SendKeyDelay Изменяет длительность короткой паузы между отправляемыми нажатиями клавиш. Значение 0 полностью убирает задержку.
Время паузы в миллисекундах (по умолчанию 5).
SendKeyDownDelay Изменяет длительность удержания клавиши перед отпусканием при отправке нажатия. Для приложений, которым требуется время для обработки нажатий, значение может потребоваться увеличить. Значение 0 полностью убирает задержку.
Время паузы в миллисекундах (по умолчанию 5).
SetExitCode Коды выхода AutoIt при возникновении фатальной ошибки.
0 = (по умолчанию) не устанавливать
1 = Устанавливать @exitCode при фатальной ошибке – см. Коды выхода при критических ошибках AutoIt3
TCPTimeout Задаёт время, после которого TCP-функции останавливаются при отсутствии обмена.
Время в миллисекундах до таймаута (по умолчанию 100).
TrayAutoPause Скрипт приостанавливается при клике по иконке в трее.
0 = без паузы
1 = (по умолчанию) пауза. Если DefaultMenu отсутствует, паузы не будет.
TrayIconDebug Если включено, показывает текущую строку скрипта в подсказке иконки в трее – помогает при отладке.
0 = (по умолчанию) без отладочной информации
1 = показывать отладочную информацию
TrayIconHide Скрывает иконку AutoIt в трее. Примечание: иконка всё равно появится примерно на 750 мс при старте.
0 = (по умолчанию) показывать иконку
1 = скрыть иконку
TrayMenuMode Расширяет поведение иконки/меню трея скрипта. Можно комбинировать (суммировать) следующие значения.
0 = (по умолчанию) стандартные пункты меню (Script Paused/Exit) добавляются к пользовательскому меню; пункты автоматически отмечаются/снимаются при клике; двойной клик по иконке возвращает controlID пункта $TRAY_DEFAULT.
1 = без стандартного меню
2 = пункты не отмечаются/снимаются автоматически при клике
4 = не возвращать controlID пункта $TRAY_DEFAULT при двойном клике по иконке
8 = отключить автоматическую отметку групп radio-пунктов
TrayOnEventMode Включает/отключает уведомления функций OnEvent для трея.
0 = (по умолчанию) отключено
1 = включено
WinDetectHiddenText Указывает, может ли скрытый текст окна "видеться" функциями поиска окна.
0 = (по умолчанию) Не определять скрытый текст
1 = Определять скрытый текст
WinSearchChildren Разрешает процедурам поиска окон искать и дочерние окна, а не только верхнего уровня.
0 = (по умолчанию) Искать только окна верхнего уровня
1 = Искать окна верхнего уровня и дочерние
WinTextMatchMode Изменяет способ сопоставления текста окна при операциях поиска.
1 = (по умолчанию) полный / медленный режим
2 = быстрый режим
В быстром режиме AutoIt обычно "видит" только текст диалога, текст кнопок и заголовки некоторых элементов управления. В режиме по умолчанию видно гораздо больше текста (например, содержимое окна Блокнота).
Если возникают проблемы с производительностью при большом количестве поисков окон, переход в "быстрый" режим может помочь.
WinTitleMatchMode Изменяет способ сопоставления заголовков окон при операциях поиска.
1 = (по умолчанию) Сопоставлять заголовок с начала
2 = Сопоставлять любую подстроку в заголовке
3 = Точное совпадение заголовка
4 = Расширенный режим (сохранён только для обратной совместимости – см. Заголовки и текст окон (расширенно))
-1 … -4 = Сопоставление без учёта регистра по соответствующему типу сопоставления.
WinWaitDelay Изменяет длительность короткой паузы скрипта после успешной операции с окном.
Время паузы в миллисекундах (по умолчанию 250).

Пример

; скопируйте те, которые хотите изменить ;значение по умолчанию указано первым

Opt("CaretCoordMode", 1)         ;1=абсолютные, 0=относительные, 2=клиентские
Opt("ExpandEnvStrings", 0)       ;0=не разворачивать, 1=разворачивать
Opt("ExpandVarStrings", 0)       ;0=не разворачивать, 1=разворачивать
Opt("GUICloseOnESC", 1)          ;1=ESC закрывает, 0=ESC не закрывает
Opt("GUICoordMode", 1)           ;1=абсолютные, 0=относительные, 2=ячейки
Opt("GUIDataSeparatorChar", "|") ;"|" по умолчанию
Opt("GUIOnEventMode", 0)         ;0=отключено, 1=режим OnEvent включён
Opt("GUIResizeMode", 0)          ;0=без изменения размера, <1024 особые режимы
Opt("GUIEventOptions", 0)        ;0=по умолчанию, 1=только уведомление, 2=индекс вкладки в GUICtrlRead
Opt("MouseClickDelay", 10)       ;10 миллисекунд
Opt("MouseClickDownDelay", 10)   ;10 миллисекунд
Opt("MouseClickDragDelay", 250)  ;250 миллисекунд
Opt("MouseCoordMode", 1)         ;1=абсолютные, 0=относительные, 2=клиентские
Opt("MustDeclareVars", 0)        ;0=нет, 1=требуется предварительное объявление
Opt("PixelCoordMode", 1)         ;1=абсолютные, 0=относительные, 2=клиентские
Opt("SendAttachMode", 0)         ;0=не подключать, 1=подключать
Opt("SendCapslockMode", 1)       ;1=сохранять и восстанавливать, 0=нет
Opt("SendKeyDelay", 5)           ;5 миллисекунд
Opt("SendKeyDownDelay", 1)       ;1 миллисекунда
Opt("TCPTimeout", 100)           ;100 миллисекунд
Opt("TrayAutoPause", 1)          ;0=без паузы, 1=пауза
Opt("TrayIconDebug", 0)          ;0=без информации, 1=показывать номер строки для отладки
Opt("TrayIconHide", 0)           ;0=показывать, 1=скрыть иконку в трее
Opt("TrayMenuMode", 0)           ;0=добавлять, 1=без стандартного меню, 2=без авто-отметки, 4=не возвращать menuitemID
Opt("TrayOnEventMode", 0)        ;0=отключить, 1=включить
Opt("WinDetectHiddenText", 0)    ;0=не определять, 1=определять
Opt("WinSearchChildren", 1)      ;0=нет, 1=искать также дочерние окна
Opt("WinTextMatchMode", 1)       ;1=полный режим, 2=быстрый
Opt("WinTitleMatchMode", 1)      ;1=с начала, 2=подстрока, 3=точное, 4=расширенный, -1 … -4 = без учёта регистра
Opt("WinWaitDelay", 250)         ;250 миллисекунд
