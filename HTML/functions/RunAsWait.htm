Справочник функций

RunAsWait

Запускает внешнюю программу от имени другого пользователя и приостанавливает выполнение скрипта до её завершения.

RunAsWait ( "username", "domain", "password", logon_flag, "program" [, "workingdir" [, show_flag [, opt_flag]]] )

Параметры

username Имя пользователя для входа.
domain Домен для проверки подлинности.
password Пароль пользователя.
logon_flag $RUN_LOGON_NOPROFILE (0) – интерактивный вход без профиля.
$RUN_LOGON_PROFILE (1) – интерактивный вход с профилем.
$RUN_LOGON_NETWORK (2) – только сетевые учётные данные.
$RUN_LOGON_INHERIT (4) – наследовать окружение вызывающего процесса вместо окружения пользователя.

Константы определены в AutoItConstants.au3.
program Полный путь к запускаемой программе (EXE, BAT, COM или PIF; см. раздел Примечания).
workingdir [необязательный] Рабочий каталог. Если не указан, используется значение @SystemDir. Это не путь к программе.
show_flag [необязательный] Флаг отображения окна запускаемой программы:
@SW_HIDE – скрытое окно (или ключевое слово Default)
@SW_MINIMIZE – свёрнутое окно
@SW_MAXIMIZE – развёрнутое окно.
opt_flag [необязательный] Управляет различными параметрами взаимодействия родительского и дочернего процессов.
$RUN_CREATE_NEW_CONSOLE (0x10000) = дочерний консольный процесс создаётся со своим окном вместо использования окна родителя. Полезен только когда родитель скомпилирован как консольное приложение.

Константа определена в AutoItConstants.au3.

Возвращаемое значение

Успех: Возвращает код завершения запущенной программы.
Ошибка: Устанавливает @error ≠ 0.

Примечания

Пути с пробелами нужно заключать в кавычки.

Важно указать рабочий каталог, к которому есть доступ у пользователя, под которым выполняется запуск, иначе функция завершится с ошибкой.

Загружать профиль пользователя рекомендуется только если он действительно нужен. При определённых условиях есть небольшая вероятность, что профиль застрянет в памяти. Если скрипт, использующий RunAs(), выполняется под учётной записью SYSTEM (например, как служба) и профиль пользователя загружен, нужно следить, чтобы скрипт продолжал работать до закрытия дочернего процесса.

При запуске от имени администратора должна быть включена служба вторичного входа в систему (RunAs()), иначе функция завершится с ошибкой. Это не относится к запуску под учётной записью SYSTEM.

После запуска указанной программы скрипт приостанавливается до её завершения. Чтобы запустить программу и сразу продолжить выполнение скрипта, используйте функцию RunAs().

Некоторые программы как будто возвращают управление сразу, хотя ещё работают; такие программы порождают другой процесс, и для таких случаев можно использовать функцию ProcessWaitClose().

Параметры "загрузка профиля" и "только сетевые учётные данные" несовместимы. Использование обоих даёт неопределённый результат.

В поколении Windows XP есть проблема, которая мешает работе перенаправления STDIO и флага показа окна. Подробнее о затронутых версиях и об исправлении см. статью базы знаний Microsoft KB818858. Пользователей с Windows XP SP2 и новее или Windows Vista и новее это не затрагивает.

См. также

Пример

#include <AutoItConstants.au3>
#include <MsgBoxConstants.au3>
#include <WinAPIError.au3>

Example()

Func Example()
    ; Замените имя пользователя и пароль на подходящие для вашей системы.
    Local $sUserName = "Username"
    Local $sPassword = "Password"

    ; Запустить Блокнот от имени указанного пользователя и ждать его закрытия.
    Local $iReturn = RunAsWait($sUserName, @ComputerName, $sPassword, $RUN_LOGON_NOPROFILE, "notepad.exe")

    If @error Then
        Local $sLastError = _WinAPI_GetLastErrorMessage()
        MsgBox($MB_SYSTEMMODAL + $MB_ICONERROR, "Error", "Notepad has not Run :" & @CRLF & @CRLF & $sLastError)
    Else
        ; Показать код возврата процесса Блокнота.
        MsgBox($MB_SYSTEMMODAL, "", "The return code from Notepad was: " & $iReturn)
    EndIf
EndFunc   ;==>Example
