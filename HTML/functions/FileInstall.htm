Справочник функций

FileInstall

Включает файл в скомпилированный скрипт и извлекает его при выполнении.

FileInstall ( "source", "dest" [, flag = 0] )

Параметры

source Путь к исходному файлу для включения в компиляцию. Должен быть строковым литералом – переменные и результаты функций недопустимы. Допускается относительный путь (с использованием .\ или .\.\) относительно файла скрипта (.au3).
dest Путь назначения файла; если указан только каталог, добавьте завершающий обратный слеш. Может быть переменной.
flag [необязательный] Флаг, определяющий, перезаписывать ли существующие файлы:
$FC_NOOVERWRITE (0) = (по умолчанию) не перезаписывать существующие файлы
$FC_OVERWRITE (1) = перезаписывать существующие файлы

Константы определены в FileConstants.au3.

Возвращаемое значение

Успех: Возвращает 1.
Ошибка: Возвращает 0.

Примечания

Функция FileInstall() предназначена для включения файлов в скомпилированный скрипт AutoIt. При выполнении соответствующей инструкции эти файлы "извлекаются" из скомпилированного скрипта. Следует учитывать, что файлы, например изображения, могут значительно увеличить размер скомпилированного скрипта.
Исходный файл необходимо указывать строковым литералом – использовать переменные, макросы, вычисления и вызовы функций нельзя. Файл должен быть доступен во время компиляции, тогда как переменные, вычисления и вызовы функций разрешаются только при выполнении скрипта – значительно позже компиляции – и потому не подходят для определения исходного файла.
В исходном пути символы подстановки не допускаются.

Единственное исключение из сказанного: в качестве источника можно использовать @ScriptFullPath – это позволяет включить в компиляцию сам исходный скрипт.

При вызове из некомпилированного скрипта функция выполняет обычное копирование файла (для удобного тестирования до компиляции).
Установленные файлы сохраняют исходные временны́е метки создания и изменения.

Каталог назначения должен существовать до вызова функции, иначе FileInstall() вернёт 0 и не создаст ни файл, ни путь. Для создания каталога см. DirCreate().

Атрибуты существующего файла могут препятствовать его перезаписи. Воспользуйтесь FileDelete() или FileSetAttrib(), чтобы обеспечить беспрепятственную установку файла.

Два вызова FileInstall() нельзя объединять в одной инструкции. Следующая строка не будет работать в скомпилированном скрипте:
    If FileInstall("fileA1", "fileA2") And FileInstall("fileB1", "fileB2") Then

См. также

DirCreate, FileDelete, FileSetAttrib

Пример

Local $bFileInstall = False ; Измените на True и скорректируйте пути к файлам.

; Устанавливает файл C:\Test.bmp в каталог скрипта.
If $bFileInstall Then FileInstall("C:\Test.bmp", @ScriptDir & "\Test.bmp")
