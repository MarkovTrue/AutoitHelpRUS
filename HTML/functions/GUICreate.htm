Справочник функций

GUICreate

Создаёт GUI-окно.

GUICreate ( "title" [, width [, height [, left = -1 [, top = -1 [, style = -1 [, exStyle = -1 [, parent = 0]]]]]]] )

Параметры

title Заголовок диалогового окна.
width [необязательный] Ширина клиентской области окна.
height [необязательный] Высота клиентской области окна.
left [необязательный] Левая позиция диалогового окна. По умолчанию (-1) окно центрируется. Если задан, параметр top тоже должен быть задан.
top [необязательный] Верхняя позиция диалогового окна. По умолчанию (-1) окно центрируется.
style [необязательный] Задаёт стиль окна. См. приложение "Стили элементов управления GUI".
Используйте -1 для стиля по умолчанию, включающего комбинацию $WS_MINIMIZEBOX, $WS_CAPTION, $WS_POPUP, $WS_SYSMENU.
Некоторые стили включаются всегда: $WS_CLIPSIBLINGS, а также $WS_SYSMENU при указании $WS_MAXIMIZEBOX или $WS_SIZEBOX.
exStyle [необязательный] Задаёт расширенный стиль окна. См. таблицу расширенных стилей ниже.
Используйте -1 для значения по умолчанию – без расширенных стилей.
Принудительные стили: $WS_EX_WINDOWEDGE.
parent [необязательный] Дескриптор ранее созданного окна – новое окно становится его дочерним.

Возвращаемое значение

Успех: Возвращает дескриптор окна Windows.
Ошибка: 0 если окно не удаётся создать; устанавливает значение @error = 1.

Примечания

По умолчанию диалоговое окно не допускает изменение размера и не разворачивается. Для этого в параметр style можно передать $WS_SIZEBOX или $WS_MAXIMIZEBOX.
Обратите внимание: если задан только один стиль, он заменяет все стили по умолчанию. Например, одно только $WS_SIZEBOX не установит $WS_MINIMIZEBOX, $WS_CAPTION, $WS_POPUP, $WS_SYSMENU. Оптимальный способ создать окно с изменяемым размером – использовать $WS_OVERLAPPEDWINDOW.
При использовании $WS_EX_MDICHILD позиция задаётся относительно клиентской области родительского окна. С $WS_EX_LAYERED можно получить прозрачное изображение поверх фонового рисунка, определённого в родительском окне.
Добавление стиля $WS_CLIPCHILDREN позволяет избежать мерцания при изменении размера GUI, содержащего, например, элемент управления Edit.
Чтобы разрешить перетаскивание GUI без $WS_CAPTION, используйте $WS_EX_CONTROLPARENT в параметре exStyle.

Для объединения стилей со стилем по умолчанию используйте BitOR($GUI_SS_DEFAULT_GUI, новый_стиль, ...).
Указанный размер – это размер клиентской области окна. Рамка и заголовок сделают окно несколько больше. Использование элементов управления меню также изменит высоту окна.

Таблица расширенных стилей

Расширенный стиль Описание
$WS_EX_ACCEPTFILES Позволяет элементу редактирования или ввода внутри созданного GUI-окна принимать имена файлов через перетаскивание. Элемент управления должен также иметь состояние $GUI_DROPACCEPTED, установленное через GUICtrlSetState(). Для других элементов информацию о перетаскивании можно получить через @GUI_DragId, @GUI_DragFile, @GUI_DropId.
$WS_EX_APPWINDOW Принудительно отображает окно верхнего уровня на панели задач, когда оно видимо.
$WS_EX_CLIENTEDGE Задаёт окно с рамкой утопленного края.
$WS_EX_CONTEXTHELP Добавляет значок вопроса в заголовок окна. Нельзя использовать совместно с $WS_MAXIMIZEBOX или $WS_MINIMIZEBOX.
$WS_EX_DLGMODALFRAME Создаёт окно с двойной рамкой; при желании можно добавить заголовок, указав стиль WS_CAPTION в параметре style.
$WS_EX_MDICHILD Создаёт дочернее окно, которое перемещается вместе с родительским (имитация MDI-окна; максимизация/минимизация не имитируются).
$WS_EX_OVERLAPPEDWINDOW Объединяет стили $WS_EX_CLIENTEDGE и $WS_EX_WINDOWEDGE.
$WS_EX_STATICEDGE Создаёт окно с трёхмерной рамкой, предназначенное для элементов, не принимающих ввод пользователя.
$WS_EX_TOPMOST Задаёт, что окно должно располагаться поверх всех не-topmost-окон и оставаться на переднем плане даже при деактивации.
$WS_EX_TRANSPARENT Окно выглядит прозрачным, так как пиксели соседних дочерних окон уже отрисованы.
$WS_EX_TOOLWINDOW Создаёт инструментальное окно, предназначенное для использования в виде плавающей панели инструментов. Инструментальное окно имеет заголовок меньшей высоты, а его текст отображается уменьшенным шрифтом. Такое окно не отображается на панели задач и в диалоге переключения задач (Alt+Tab). Если у инструментального окна есть системное меню, его значок не отображается в заголовке, однако меню можно вызвать сочетанием Alt+Пробел.
$WS_EX_WINDOWEDGE Задаёт окно с рамкой выпуклого края.
$WS_EX_LAYERED Создаёт послойное (layered) окно. Нельзя использовать для дочерних окон.

Для использования указанных выше значений необходимо включить в скрипт #include <WindowsStylesConstants.au3>.

Примечание: дескриптор, возвращаемый этой функцией, является настоящим дескриптором Windows и может использоваться так же, как результат WinGetHandle().
WinMove(), вызванный до GuiSetState(), не изменит позицию/размер уже определённых элементов управления.

Пример

Пример 1

#include <GUIConstantsEx.au3>

Example()

Func Example()
    ; Создать GUI с различными элементами управления.
    Local $hGUI = GUICreate("Example", 400, 400)
    Local $idBtn_OK = GUICtrlCreateButton("OK", 310, 370, 85, 25)

    Local $aWindow_Size = WinGetPos($hGUI)
    ConsoleWrite('Window Width = ' & $aWindow_Size[2] & @CRLF)
    ConsoleWrite('Window Height = ' & $aWindow_Size[3] & @CRLF)
    Local $aWindowClientArea_Size = WinGetClientSize($hGUI)
    ConsoleWrite('Window Client Area Width = ' & $aWindowClientArea_Size[0] & @CRLF)
    ConsoleWrite('Window Client Area Height = ' & $aWindowClientArea_Size[1] & @CRLF)

    ; Отобразить GUI.
    GUISetState(@SW_SHOW, $hGUI)

    ; Цикл до выхода пользователя.
    While 1
        Switch GUIGetMsg()
            Case $GUI_EVENT_CLOSE, $idBtn_OK
                ExitLoop

        EndSwitch
    WEnd

    ; Удалить предыдущий GUI и все элементы управления.
    GUIDelete($hGUI)
EndFunc   ;==>Example

Пример 2

#include <GUIConstantsEx.au3>
#include <WindowsStylesConstants.au3>

Example()

Func Example()
    Local $sFilePath = "..\GUI\logo_autoit_210x72.gif"

    ; Создать GUI с различными элементами управления.
    Local $hGUI = GUICreate("Example", 400, 100)
    GUICtrlCreatePic("..\GUI\msoobe.jpg", 0, 0, 400, 100)

    ; Отобразить GUI.
    GUISetState(@SW_SHOW, $hGUI)

    Local $hChild = GUICreate("", 210, 72, 20, 15, $WS_POPUP, BitOR($WS_EX_LAYERED, $WS_EX_MDICHILD), $hGUI)

    ; Создаём элемент Picture с прозрачным изображением.
    GUICtrlCreatePic($sFilePath, 0, 0, 210, 72)

    ; Отображаем дочерний GUI.
    GUISetState(@SW_SHOW)

    ; Цикл до выхода пользователя.
    While 1
        Switch GUIGetMsg()
            Case $GUI_EVENT_CLOSE
                ExitLoop

        EndSwitch
    WEnd

    ; Удалить предыдущие GUI и все элементы управления.
    GUIDelete($hGUI)
    GUIDelete($hChild)
EndFunc   ;==>Example
