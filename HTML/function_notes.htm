Примечания к функциям

Функции в AutoIt являются объектами первого класса. Помимо прочего это значит, что функцию можно присвоить переменной, передать как аргумент или вернуть из другой функции.
За исключением отдельных правил объявления, связанных с областью видимости (имена встроенных функций зарезервированы, а имена UDF можно переопределить только локально), имена функций не имеют особого статуса в языке.

#include <MsgBoxConstants.au3>

Local $vFunc = MsgBox

$vFunc($MB_OK, "Hi there!", "Lovely day today.")

 

Call(_Test)

Func _Test()
    ConsoleWrite("! Testing" & @CRLF)
EndFunc

 

Необязательные параметры

Многие функции содержат необязательные параметры, которые можно опустить. Но если вы хотите указать необязательный параметр, нужно указать и все предшествующие ему параметры!
Например, рассмотрим Run ( "filename", ["workingdir" [, flag]] ). Если вы хотите задать flag, вы обязаны указать workingdir.

Если необязательный параметр нужно задать, а перед ним стоят один или несколько других необязательных, для них нужно указать значения по умолчанию. Как правило, функции принимают ключевое слово Default, когда вы хотите использовать значение по умолчанию. Подробнее смотрите в описании соответствующего необязательного параметра.


Многие функции Win___ содержат необязательный параметр text. Этот параметр помогает различать окна с одинаковыми заголовками.

Индикация успеха/ошибки во встроенных функциях AutoIt

Одни функции используют возвращаемое значение для индикации успеха/ошибки, другие устанавливают флаг @error. Некоторые делают и то, и другое....

Если используется метод возвращаемого значения, в справке указано ожидаемое значение для успеха/ошибки, но значение успеха обычно ненулевое, чтобы код легко читался...

If SomeUserFunc() Then ;...функция сработала.
If Not SomeUserFunc() Then ;...функция не сработала.



Когда в документации указано, что возвращаемое значение = none, AutoIt всё равно возвращает значение, чтобы избежать ошибок. Обычно возвращается 1, но полагаться на это значение не стоит.

Если используется метод флага @error, то @error = 0 всегда означает успех. Другие значения @error указывают на проблему и описаны в справке по функции.

Local $sFileRead = FileReadLine("C:\someFile.txt")
If @error = -1 Then ; Достигнут конец файла.



Если функция использует метод флага @error, флаг нужно проверять сразу после возврата из функции, так как @error сбрасывается в 0 при входе в следующую функцию. Не используйте и не читайте возвращаемое значение функции, если установлен флаг @error: в этом случае возвращаемое значение обычно не определено...

Индикация успеха/ошибки в пользовательских функциях

Возвращаемое значение, макросы @error и @extended обычно задаёт автор с помощью функций Return/SetError/SetExtended в коде. Конкретные используемые значения смотрите в документации соответствующей пользовательской библиотеки.
