Калькулятор

В этом уроке мы попробуем создать простой калькулятор, как тот, что встроен в MS-Windows. Используя возможности GUI AutoIt.
Предполагается, что вы уже знакомы с созданием и запуском скриптов AutoIt, как показано в    примере.


Создание диалога GUI – (Дизайн)

В этой части мы начнём с создания диалога GUI. Хорошо иметь представление о дизайне на основе функциональности нашей программы.
То есть, сколько и какие типы элементов управления нам потребуются. В этом случае мы используем тот же дизайн, что Windows Calc.
На рис. 1 видно, как будет выглядеть наш калькулятор в конце этой части.

Рис. 1

Чтобы начать создание диалога, сначала подключим несколько библиотек, которые объявляют константы, используемые в нашем диалоге. Эти константы содержат числовые значения стилей и параметров, определяющих внешний вид каждого элемента управления, а также значения для событий и состояний. (Обычно GUIConstantsEx.au3 уже встроена и находится в папке установки AutoIt). Далее мы увидим, как их применять, а пока подключим эту библиотеку, написав в скрипт следующие строки:

#include <EditConstants.au3>
#include <GUIConstantsEx.au3>
#include <StaticConstants.au3>
#include <WindowsStylesConstants.au3>


Как показано в основном руководстве, для начала создания диалога нужно определить окно – его название, размер и позицию с помощью GUICreate(). Значения окна будут следующими:
Название = "Calculator"
Ширина = 260
Высота = 230
Поэтому следующие строки кода будут такими:

GUICreate("Calculator", 260, 230)


Сохранение идентификатора (ID) каждого элемента
При определении элементов мы должны создать переменные для каждой кнопки. Когда мы это делаем, переменная сохраняет ID элемента управления, который позволяет нам манипулировать его действиями во время выполнения. Посмотрите    пример.

Наименование переменных
Имена переменных – дело личное, но важно быть понятным при выборе названия для лучшего понимания того, что представляет элемент управления в коде. Как видно в следующем примере, используется синтаксис: $idBtn1, что означает "ID GUI элемента управления – это кнопка, цифра №1", параметры этой кнопки таковы:
Текст ="1", Слева = 54, Сверху = 138, Ширина = 36, Высота = 29

$idBtn1 = GUICtrlCreateButton("1", 54, 138, 36, 29)

Как я уже говорил, это делает код более понятным.

Создание каждого элемента

Цикл сообщений
Как показано в одном    уроке функция GUIGetMsg() возвращает значение события в диалоге, которое получает переменная в данном случае $msg и проверяется каждый раз, когда цикл Do.. Until повторяется. Для этой части мы добавим только одно простое событие, которое повторяется в бесконечном цикле до тех пор, пока не произойдёт событие, равное константе $GUI_EVENT_CLOSE (равно -3), что приведёт к закрытию окна и завершению скрипта. Таким образом, следующие строки для нашего скрипта будут:

Local $msg
Do
    $msg = GUIGetMsg()

Until $msg = $GUI_EVENT_CLOSE


Применение стилей
До этого момента мы уже могли завершить дизайн диалога. Вы можете уже запустить скрипт, и он покажет почти готовый калькулятор, но обратите внимание, что калькулятор выглядит не совсем как ожидалось. Элемент управления EDIT, который представляет экран цифр, отображается неправильно, и нужно применить определённые стили оформления (см. рис. 2). Чтобы исправить внешний вид, мы будем использовать стили EDIT:

Рис. 2

Для этой части нам нужно изменить только две строки кода.

Первое изменение: в первом дополнении мы добавим в строку переменной $idEdtScreen определение для элемента управления (Edit) с режимом только для чтения ($ES_READONLY) и выравниванием вправо ($ES_RIGHT), чтобы объединить эти две переменные с помощью функции BitOR() как показано в строке кода ниже. Мы также добавим $WS_EX_STATICEDGE к параметру расширенного стиля (exStyle), чтобы придать границе углублённый вид. Для справки по таблице стилей элемента управления EDIT нажмите здесь. (для подробного объяснения каждого стиля).

Local $idEdtScreen = GUICtrlCreateEdit("0.", 8, 2, 239, 23, BitOR($ES_READONLY, $ES_RIGHT), $WS_EX_STATICEDGE)


Второе изменение: во втором дополнении мы добавим в строку переменной $idLblMemory определение элемента управления LABEL, который будет показывать состояние памяти, которое мы увидим во второй части. Мы определим этот элемент управления (Label) со стилем углублённости ($SS_SUNKEN) как показано в строке кода. Для справки по таблице стилей элемента управления LABEL нажмите здесь.

Local $idLblMemory = GUICtrlCreateLabel("", 12, 39, 27, 26, $SS_SUNKEN)


После внесения этих изменений попробуйте запустить скрипт. Изменения повлияют на внешний вид, показанный на рис. 3, теперь наш калькулятор выглядит как на рис. 1 в начале. Вы также можете посмотреть    как справку.
