Режим OnEvent

В режиме OnEvent вместо постоянного опроса GUI на предмет произошедших событий GUI временно приостанавливает скрипт и вызывает заранее определённую функцию для обработки события. Например, если пользователь нажимает Button1, GUI приостанавливает основной скрипт и вызывает заранее определённую пользовательскую функцию, обрабатывающую Button1. По завершении вызова функции основной скрипт возобновляется. Этот режим похож на работу форм в Visual Basic.

Пока GUI работает, основной скрипт может выполнять любые обычные действия, но для простоты примеров мы заставим основной скрипт "простаивать" в бесконечном цикле While.
Цикл While должен ограничиваться вызовом Sleep(), чтобы снизить использование CPU. Никаких действий с GUI внутри цикла быть не должно.

Режим по умолчанию – MessageLoop, поэтому перед использованием режима OnEvent необходимо вызвать Opt("GUIOnEventMode", 1).

Базовый формат OnEvent

Общая структура OnEvent-кода такова:

While 1
    Sleep(100) ; Sleep для снижения использования CPU
WEnd

Func Event1()
    ; Код для обработки события идёт здесь
EndFunc   ;==>Event1

Func Event2()
    ; Код для обработки события идёт здесь
EndFunc   ;==>Event2

События GUI

В режиме OnEvent ваш GUI генерирует следующие "события":

  • Событие контрола
  • Системное событие

Оба типа событий вызовут пользовательскую функцию, если она задана для GUI (GUISetOnEvent) или для контрола (GUICtrlSetOnEvent). Если функция для события не задана, оно просто игнорируется. Внутри вызванной функции для удобства обработки события устанавливаются различные макросы.

Макрос Описание
@GUI_CtrlId Идентификатор контрола, отправившего сообщение, ИЛИ идентификатор системного события.
@GUI_WinHandle Дескриптор GUI, отправившего сообщение.
@GUI_CtrlHandle Дескриптор контрола, отправившего сообщение (если применимо).

Примечание: вполне допустимо использовать одну и ту же функцию для нескольких событий. В этом случае нужно лишь выполнять действия в зависимости от макроса @GUI_CtrlId. Например, можно зарегистрировать все системные события на одну функцию.

 

Событие контрола

При щелчке по контролу или его изменении отправляется событие контрола. Событие отправляется функции, заданной через GUICtrlSetOnEvent. Внутри пользовательской функции @GUI_CtrlId равен controlID, который был возвращён при создании контрола функцией GUICtrlCreate....

 

Системное событие

Системные события – такие как закрытие GUI – отправляются аналогично событиям контролов, но тип события определяется через @GUI_CtrlId. Событие отправляется функции, заданной через GUISetOnEvent. Возможные значения системных событий приведены ниже:

$GUI_EVENT_CLOSE
$GUI_EVENT_MINIMIZE
$GUI_EVENT_RESTORE
$GUI_EVENT_MAXIMIZE
$GUI_EVENT_PRIMARYDOWN
$GUI_EVENT_PRIMARYUP
$GUI_EVENT_SECONDARYDOWN
$GUI_EVENT_SECONDARYUP
$GUI_EVENT_MOUSEMOVE
$GUI_EVENT_RESIZED
$GUI_EVENT_DROPPED

Пример GUI

На основной странице справочника по GUI мы начали простой пример Hello World, который выглядел так:

#include <GUIConstantsEx.au3>

GUICreate("Hello World", 200, 100)
GUICtrlCreateLabel("Hello world! How are you?", 30, 10)
GUICtrlCreateButton("OK", 70, 50, 60)
GUISetState(@SW_SHOW)
Sleep(2000)

 

Теперь дополним код с использованием OnEvents и некоторых описанных выше сообщений событий.

#include <GUIConstantsEx.au3>
#include <MsgBoxConstants.au3>

Opt("GUIOnEventMode", 1) ; Переключиться на режим OnEvent

Local $hMainGUI = GUICreate("Hello World", 200, 100)
GUISetOnEvent($GUI_EVENT_CLOSE, "CLOSEButton")
GUICtrlCreateLabel("Hello world! How are you?", 30, 10)
Local $iOKButton = GUICtrlCreateButton("OK", 70, 50, 60)
GUICtrlSetOnEvent($iOKButton, "OKButton")
GUISetState(@SW_SHOW, $hMainGUI)

While 1
    Sleep(100) ; Sleep для снижения использования CPU
WEnd

Func OKButton()
    ; Примечание: в этот момент @GUI_CtrlId будет равна $iOKButton,
    ; и @GUI_WinHandle будет равна $hMainGUI
    MsgBox($MB_OK, "GUI Event", "You selected OK!")
EndFunc   ;==>OKButton

Func CLOSEButton()
    ; Примечание: в этот момент @GUI_CtrlId будет равна $GUI_EVENT_CLOSE,
    ; и @GUI_WinHandle будет равна $hMainGUI
    MsgBox($MB_OK, "GUI Event", "You selected CLOSE! Exiting...")
    Exit
EndFunc   ;==>CLOSEButton

Вот и всё. Очевидно, чем больше окон и контролов вы создаёте, тем сложнее становится код, но приведённый выше пример показывает основы.

Расширенные операции и несколько окон

Идентификаторы контролов уникальны даже при наличии нескольких окон – но как обрабатывать несколько окон?

Вот пример, аналогичный приведённому выше, но с дополнительным "фиктивным" окном.

#include <GUIConstantsEx.au3>
#include <MsgBoxConstants.au3>

Opt("GUIOnEventMode", 1) ; Переключиться на режим OnEvent

Global $hMainGUI = GUICreate("Hello World", 200, 100)
GUISetOnEvent($GUI_EVENT_CLOSE, "CLOSEButton")
GUICtrlCreateLabel("Hello world! How are you?", 30, 10)
Local $iOKButton = GUICtrlCreateButton("OK", 70, 50, 60)
GUICtrlSetOnEvent($iOKButton, "OKButton")

Global $hDummyGUI = GUICreate("Dummy GUI for testing", 200, 100)
GUISetOnEvent($GUI_EVENT_CLOSE, "CLOSEButton")

GUISwitch($hMainGUI)
GUISetState(@SW_SHOW)

While 1
    Sleep(100) ; Sleep для снижения использования CPU
WEnd

Func OKButton()
    ; Примечание: в этот момент @GUI_CtrlId будет равна $iOKButton
    MsgBox($MB_OK, "GUI Event", "You selected OK!")
EndFunc   ;==>OKButton

Func CLOSEButton()
    ; Примечание: в этот момент @GUI_CtrlId будет равна $GUI_EVENT_CLOSE,
    ; @GUI_WinHandle будет либо $hMainGUI либо $hDummyGUI
    If @GUI_WinHandle = $hMainGUI Then
        MsgBox($MB_OK, "GUI Event", "You selected CLOSE in the main window! Exiting...")
        Exit
    EndIf
EndFunc   ;==>CLOSEButton

 

Первое существенное изменение – вызов функции GUISwitch: при создании нового окна оно становится "текущим" окном для последующих операций GUI (включая создание контролов). В нашем случае мы хотим работать с основным окном "Hello World", а не с тестовым, поэтому "переключаемся". Некоторые функции GUI позволяют указать дескриптор окна в самом вызове функции – такие функции выполняют переключение автоматически. В нашем примере это можно было бы сделать так:

GUISetState(@SW_SHOW, $hMainGUI)

 

Также обратите внимание, что мы использовали одну и ту же OnEvent-функцию для обработки кнопки "закрыть" в обоих окнах, а затем по @GUI_WinHandle определили, какое окно отправило сообщение – и закрывали GUI только тогда, когда нажата кнопка закрытия и сообщение пришло из основного окна. При желании можно так же легко использовать отдельные функции для каждого окна.
