Режим MessageLoop

В режиме MessageLoop большую часть времени скрипт проводит в плотном цикле. Этот цикл просто опрашивает GUI с помощью функции GUIGetMsg. При возникновении события возвращаемое значение GUIGetMsg покажет подробности (нажата кнопка, GUI закрыт и т. п.).

Режим MessageLoop – режим обработки сообщений по умолчанию для GUI AutoIt; альтернативный режим – OnEvent.

В режиме MessageLoop события приходят, только пока вы активно вызываете GUIGetMsg, поэтому вызывать её нужно много раз в секунду – иначе GUI станет неотзывчивым.

Базовый формат MessageLoop

Общая структура кода MessageLoop:

Local $iMsg = 0

While 1
    $iMsg = GUIGetMsg()
    ...
    ...
WEnd

 

Обычно такой плотный цикл загрузил бы CPU на 100% – но к счастью функция GUIGetMsg автоматически переводит CPU в простой, когда нет ожидающих событий. Не ставьте ручной Sleep в цикл из опасения нагрузить CPU – это только сделает GUI неотзывчивым.

События GUI

GUIGetMsg возвращает три типа событий:

  • Нет события
  • Событие контрола
  • Системное событие

 

Нет события

Когда событий, ожидающих обработки, нет, GUIGetMsg возвращает 0. В типичном GUI это самое частое событие.

 

Событие контрола

Когда контрол щёлкнут или его состояние изменилось, отправляется событие контрола – это положительное число, соответствующее controlID, который был возвращён при создании контрола через GUICtrlCreate....

 

Системное событие

Системные события (например, закрытие GUI) – отрицательные числа. Возможные события приведены ниже и определены в GUIConstantsEx.au3:

$GUI_EVENT_CLOSE
$GUI_EVENT_MINIMIZE
$GUI_EVENT_RESTORE
$GUI_EVENT_MAXIMIZE
$GUI_EVENT_PRIMARYDOWN
$GUI_EVENT_PRIMARYUP
$GUI_EVENT_SECONDARYDOWN
$GUI_EVENT_SECONDARYUP
$GUI_EVENT_MOUSEMOVE
$GUI_EVENT_RESIZED
$GUI_EVENT_DROPPED

Пример GUI

На главной странице справочника GUI мы начали простой пример Hello World:

#include <GUIConstantsEx.au3>

GUICreate("Hello World", 200, 100)
GUICtrlCreateLabel("Hello world! How are you?", 30, 10)
GUICtrlCreateButton("OK", 70, 50, 60)
GUISetState(@SW_SHOW)
Sleep(2000)

 

Теперь завершим код с использованием MessageLoop и нескольких сообщений-событий, описанных выше. Из-за большого числа возможных сообщений принято использовать оператор Switch для читаемости.

#include <GUIConstantsEx.au3>
#include <MsgBoxConstants.au3>

Local $hGUI = GUICreate("Hello World", 200, 100)
GUICtrlCreateLabel("Hello world! How are you?", 30, 10)
Local $iOKButton = GUICtrlCreateButton("OK", 70, 50, 60)
GUISetState(@SW_SHOW, $hGUI)

Local $iMsg = 0
While 1
    $iMsg = GUIGetMsg()
    Switch $iMsg
        Case $iOKButton
            MsgBox($MB_SYSTEMMODAL, "GUI Event", "You selected the OK button.")

        Case $GUI_EVENT_CLOSE
            MsgBox($MB_SYSTEMMODAL, "GUI Event", "You selected the Close button. Exiting...")
            ExitLoop
    EndSwitch
WEnd

GUIDelete($hGUI)

Вот и всё. Очевидно, чем больше окон и контролов вы создаёте, тем сложнее становится код, но в примере выше показаны основы.

Расширенный GUIGetMsg и несколько окон

Идентификаторы контролов уникальны даже при наличии нескольких окон, поэтому код выше будет работать с контролами и в нескольких окнах. Однако при обработке событий типа $GUI_EVENT_CLOSE или $GUI_MOUSEMOVE нужно знать, какое окно сгенерировало событие. Для этого вызывайте GUIGetMsg так:

$iMsg = GUIGetMsg(1)

 

При вызове с параметром 1 вместо значения события возвращается массив, содержащий событие (в $aArray[0]) и дополнительную информацию, например, дескриптор окна (в $aArray[1]). Если бы в предыдущем примере было создано два окна, корректный код выглядел бы так:

#include <GUIConstantsEx.au3>
#include <MsgBoxConstants.au3>

Local $hMainGUI = GUICreate("Hello World", 200, 100)
GUICtrlCreateLabel("Hello world! How are you?", 30, 10)
Local $iOKButton = GUICtrlCreateButton("OK", 70, 50, 60)

Local $hDummyGUI = GUICreate("Dummy window for testing", 200, 100)

GUISwitch($hMainGUI)
GUISetState(@SW_SHOW)

Local $aMsg = 0
While 1
    $aMsg = GUIGetMsg(1)

    Select
        Case $aMsg[0] = $iOKButton
            MsgBox($MB_SYSTEMMODAL, "GUI Event", "You selected OK!")

        Case $aMsg[0] = $GUI_EVENT_CLOSE And $aMsg[1] = $hMainGUI
            MsgBox($MB_SYSTEMMODAL, "GUI Event", "You selected CLOSE on the main GUI! Exiting...")
            ExitLoop
    EndSelect
WEnd

 

Первое существенное изменение – вызов GUISwitch: при создании нового окна оно становится "текущим" для последующих GUI-операций (включая создание контролов). В нашем случае мы хотим работать с главным окном "Hello World", а не с тестовым, поэтому переключаемся. Некоторые GUI-функции позволяют указать дескриптор окна прямо в вызове – такие функции переключаются автоматически. В нашем примере это можно было сделать так:

GUISetState(@SW_SHOW, $hMainGUI)

 

Следующее изменение – способ вызова GUIGetMsg и проверка событий. Обратите внимание на использование $msg[0] и $msg[1] – теперь скрипт выходит, только если событие закрытия пришло и от нашего главного окна.
