Критические изменения в последних версиях

На этой странице список всех изменений в последних версиях, которые почти наверняка сломают существующие скрипты. Внимательно прочитайте этот список при обновлении, если вы не следили за изменениями в бета-версиях. Если вас затрагивает хотя бы один из этих пунктов, вам придётся изменить скрипты.

v3.3.16.0

UDF:

  • ColorConstants.au3 приведён в соответствие со стандартом W3C Colors. MAROON и SKYBLUE получили новые значения; CREAM, MEDBLUE, MDGRAY, MONEYGREEN удалены.
  • SecurityConstants.au3 изменён во избежание конфликтов имён. $RIGHTS_DELETE и $SYNCHRONIZE переименованы в $STANDARD_*.

v3.3.14.3

UDF:

  • Документация _MathCheckDiv() удалена. Функция по-прежнему есть в Math.au3, но может быть удалена позднее. Обновите скрипты: используйте Mod() вместо неё.
  • Значение при ошибке для _Degree() и _Radian() изменено с пустой строки на 0.
  • _WinAPI_DefSubclassProc(), _WinAPI_GetWindowSubclass(), _WinAPI_RemoveWindowSubclass() и _WinAPI_SetWindowSubclass() перенесены из WinAPIShellEx.au3 в WinAPISys.au3.
  • Исправлена опечатка в AutoItConstants.au3: $EXITCLOSE_BYSUTDOWN → $EXITCLOSE_BYSHUTDOWN (использовалась с OnAutoItExitRegister()).
  • WinAPI_GetOPenFileName() и _WinAPI_GetSaveFileName() теперь возвращают расширение файла только если оно было введено вручную.

v3.3.14.2

UDF:

  • _SQLite_Startup() больше не скачивает файлы SQLite DLL с autoitscript.com автоматически. Большинство пользователей не подозревали, что это происходило при каждом запуске скрипта и создавало значительную нагрузку на сервер. Теперь DLL SQLite нужно скачивать вручную. Подробнее см. документацию _SQLite_Startup().

v3.3.14.0

AutoIt:

  • Если Min и Max равны, Random() теперь возвращает это значение, и флаг @error не устанавливается.
  • Под Vista или Server 2008 для поддержки GDI+ v1.1 функция _GDIPlus_Startup() требует явно указать DLL.
  • Файловые операции с текстовыми файлами, открытыми без FileOpen() и явных флагов Unicode, теперь автоопределяют кодировку немного иначе, приближая поведение к современным редакторам. Это касается всех файловых функций, принимающих имя файла, например FileRead("filename.txt"). В частности:
    • Файлы с BOM открываются в режиме, соответствующем BOM. Проверяются BOM UTF-8 и UTF-16.
    • Файлы UTF-8 и UTF-16 без BOM определяются автоматически и открываются в нужном режиме.
    • Файлы, содержащие нулевые байты, по умолчанию открываются в двоичном режиме ($FO_BINARY), если не определены как корректный UTF-16. Ранее они открывались в режиме ANSI. Для отмены используйте флаг $FO_ANSI.
    • Файлы, содержащие только символы 1–127, по умолчанию открываются в режиме UTF-8 без BOM ($FO_UTF8_NOBOM). Ранее они открывались в режиме ANSI. Для отмены используйте флаг $FO_ANSI.
    • Файлы, содержащие только символы 1–255, по умолчанию открываются в режиме ANSI ($FO_ANSI).
    • В результате FileGetEncoding() теперь возвращает 512 ($FO_ANSI) или 256 ($FO_UTF8_NOBOM) вместо 0 (0 было недокументированным значением для ANSI).
  • Новые файлы по умолчанию создаются в режиме UTF-8 без BOM ($FO_UTF8_NOBOM). Для изменения используйте FileOpen().
  • Файлы UTF-16 без BOM теперь определяются автоматически в большинстве случаев.

UDF:

  • Excel UDF, функция _Excel_BookOpen(): если параметр $bReadOnly = False и доступ на запись не удалось получить, раньше устанавливался @error = 4 и книга открывалась только для чтения. Теперь @error = 0, @extended = 1, книга открывается только для чтения.
  • Word UDF, функция _Word_DocOpen(): если параметр $bReadOnly = False и доступ на запись не удалось получить, раньше устанавливался @error = 4 и документ открывался только для чтения. Теперь @error = 0, @extended = 1, документ открывается только для чтения.

v3.3.12.0

UDF:

  • Параметр $hPrintCallback (ранее $sPrintCallback) в _SQLite_Startup() теперь принимает объект функции, а не строку с именем функции. Причина: из UDF удалён вызов Call().
  • Параметр $hFunctionCall (ранее $sFunctionCall) в _FTP_ProgressDownload() и _FTP_ProgressUpload() теперь принимает объект функции, а не строку с именем функции. Причина: из UDF удалён вызов Call().
  • _ArrayDelete() устанавливает @error ≠ 0, если $iElement выходит за пределы диапазона. Подробнее см. документацию.
  • _StringBetween() изменила поведение и порядок параметров. Подробнее см. документацию.
  • _GUICtrlListView_GetView() и _GUICtrlListView_SetView() ошибочно использовали 0 для подробного режима и 1 для крупных значков; правильно: 0 для крупных значков, 1 для подробного режима.
  • Array UDF переписан. Функции и/или параметры изменены, удалены или добавлены. Подробный список см. здесь.
  • Excel UDF переписан. Функции и/или параметры переименованы, удалены или добавлены. Подробный список см. здесь.

v3.3.10.0

AutoIt:

  • Возвращаемое значение ShellExecute() изменено: вместо 1 теперь возвращается PID запущенного процесса или -1, если PID недоступен.

UDF:

  • Документация _Iif() удалена. Функция по-прежнему есть в Misc.au3, но может быть удалена позднее. Обновите скрипты: используйте тернарные операторы вместо неё.
  • _IEErrorHandlerRegister() удалена. Она больше не нужна, так как ошибки COM перестали быть фатальными.
  • Документация _StringEncrypt() удалена. Функция по-прежнему есть в String.au3, но может быть удалена позднее. Обновите скрипты: используйте функции Crypt вместо неё.
    Пример альтернативы см. в _Crypt_EncryptData().
  • Документация _StringReverse() удалена. Функция по-прежнему есть в String.au3, но может быть удалена позднее. Обновите скрипты: используйте встроенную StringReverse() вместо неё.
  • С добавлением функций WinAPIEx часть констант из Constants.au3 перенесена в WinAPIConstants.au3.
  • Word UDF переписан. Функции и/или параметры переименованы, удалены или добавлены. Подробный список см. здесь.
  • $tagGDIPPENCODERPARAMS удалён, используйте $tagGDIPENCODERPARAMS.
  • $tagGDIPENCODERPARAMS: используйте $tagGDIPENCODERPARAM в качестве первой структуры массива.
  • В $tagGDIPENCODERPARAM поле "count" переименовано в "nbOfValues".
  • Поля $tagBITMAPINFO получили префикс "bi".
  • _ArrayUnique() переписана с улучшенной производительностью. Последний параметр определяет, возвращать ли количество строк в $aArray[0].
  • ToolTip UDF переписан. Функции и/или параметры переименованы, удалены или добавлены. Подробный список см. здесь.
  • _ArrayDisplay() переписана с новыми параметрами. Подробнее см. документацию.

v3.3.4.0

Часть следующих функций устарела. Устаревшие функции больше не документируются, но продолжают работать. Они будут удалены после версии 3.3.4.0. Настоятельно рекомендуется обновить скрипты, использующие устаревшие функции, для работы с новым поведением. Некоторые функции уже удалены и отмечены соответственно.

AutoIt:

  • Режим raw-чтения для FileOpen() удалён.
  • InetGet("abort"), @InetGetActive и @InetGetBytesRead удалены. Список новых функций для замены:
    • InetGet("abort") – вызовите новую InetClose() с дескриптором, возвращённым InetGet(), чтобы прервать загрузку.
    • @InetGetActive – вызовите новую InetGetInfo() без параметров, чтобы узнать число активных загрузок.
    • @InetGetBytesRead – вызовите новую InetGetInfo() с дескриптором, возвращённым InetGet(), чтобы узнать количество прочитанных байт.
  • AdlibEnable() и AdlibDisable() удалены. Используйте новые функции AdlibRegister() и AdlibUnRegister().
  • OnAutoItStart() удалена. Используйте новую директиву препроцессора #OnAutoItStartRegister.
  • OnAutoItExit() удалена. Используйте новые функции OnAutoItExitRegister() и OnAutoItExitUnRegister().

v3.3.2.0

Часть следующих функций устарела. Устаревшие функции больше не документируются, но продолжают работать. Они будут удалены после версии 3.3.2.0. Настоятельно рекомендуется обновить скрипты, использующие устаревшие функции, для работы с новым поведением. Некоторые функции уже удалены и отмечены соответственно.

AutoIt:

  • ShellExecute() и ShellExecuteWait() больше не используют глагол "open" по умолчанию. См. раздел Примечания у этих функций.
  • Возвращаемое значение InetGet() изменено. Важно прочитать и понять эти изменения: некорректное использование InetGet() может привести к утечке ресурсов.
  • InetGet("abort"), @InetGetActive и @InetGetBytesRead устарели. Список новых функций для замены:
    • InetGet("abort") – вызовите новую InetClose() с дескриптором, возвращённым InetGet(), чтобы прервать загрузку.
    • @InetGetActive – вызовите новую InetGetInfo() без параметров, чтобы узнать число активных загрузок.
    • @InetGetBytesRead – вызовите новую InetGetInfo() с дескриптором, возвращённым InetGet(), чтобы узнать количество прочитанных байт.
  • Опция FtpBinaryMode в AutoItSetOption() удалена. Теперь InetGet() принимает флаг для указания режима передачи.
  • Псевдоним URLDownloadToFile() для InetGet() удалён.
  • AdlibEnable() и AdlibDisable() устарели. Используйте новые функции AdlibRegister() и AdlibUnRegister().
  • OnAutoItStart() устарела. Используйте новую директиву препроцессора #OnAutoItStartRegister.
  • OnAutoItExit() устарела. Используйте новые функции OnAutoItExitRegister() и OnAutoItExitUnRegister().
  • Опция OnExitFunc в AutoItSetOption() удалена. Используйте новые функции OnAutoItExitRegister() и OnAutoItExitUnRegister().
  • GUICreate() с $WS_EX_MDICHILD исправлена: позиционирование теперь относительно клиентской области, как задокументировано.
  • ProcessWait() теперь возвращает PID, а не 1 при успешном завершении.
  • WinWait(), WinWaitActive(), WinActivate(), WinActive() и WinMove() теперь возвращают HWND вместо 1 при успехе.
  • Макрос @YDAY теперь возвращает значения в диапазоне 001–366 вместо 1–366, приводя его в соответствие с другими языками (C/C++) и остальными макросами даты, которые возвращают строки с ведущими нулями.
  • RegEnumKey() и RegEnumVal() теперь возвращают пустую строку вместо сообщения об ошибке.

UDF:

  • Последний необязательный параметр _StringBetween() удалён.
  • _StringAddThousandsSep() удалена: функция слишком многократно пересматривалась из-за разногласий о её поведении.
  • _SQLite_SaveMode() переименована в _SQLite_SafeMode().

v3.3.0.0

AutoIt:

  • Удалена поддержка Windows 9x и Windows NT 4.0. Используйте AutoIt v3.2.12.1, если она по-прежнему нужна.
  • @ProcessorArch переименован в @OSArch, так как прежнее название вводило в заблуждение.
  • RegRead() и RegWrite() больше не используют hex-строки для типа REG_BINARY – теперь применяются встроенные двоичные типы данных.

AutoItX:

  • Родная версия DLL теперь использует исключительно строки Unicode (LPWSTR и LPCWTSR). Если по-прежнему нужны строки ANSI, используйте v3.2.12.1. Самый простой обходной путь – функция-обёртка для конвертации Unicode ↔ ANSI. Ограниченный набор функций (WinWait... и Send) имеет ANSI-версии, но расширение этого набора не планируется без значительного спроса.

v3.2.12.0

AutoIt:

  • RunAsSet() удалена. Используйте новые функции RunAs() и RunAsWait(): они обеспечивают несколько лучшую безопасность (хранить важные пароли в скриптах по-прежнему не рекомендуется) и расширенную функциональность.
  • Опция RunErrorsFatal удалена. Run(), RunWait(), ShellExecute() и ShellExecuteWait() теперь устанавливают @error при неудаче запуска. AutoIt больше не генерирует фатальную ошибку в этом случае.
  • StdoutRead(), StderrRead() и ConsoleRead() существенно изменены. Они больше не блокируют выполнение и возвращают управление немедленно. Параметры изменились: чтение по количеству символов больше недоступно; параметр peek перемещён на вторую позицию; третий параметр указывает возврат данных в двоичном формате (по умолчанию – текст). Макрос @extended теперь содержит количество прочитанных байт/символов.
  • StdinWrite(), ConsoleWrite() и ConsoleWriteError() теперь по-другому работают с двоичными данными: данные записываются как есть, без конвертации в строку. Возвращаемое значение – количество записанных байт вместо 1.
  • Удалена опция ColorMode ( Opt() ).
  • Исправлено: Random(Min,Max,1). Ранее Max никогда не возвращалось, поэтому скрипты, искусственно увеличивавшие Max на 1, теперь нужно изменить.

UDF:

  • (Подробности и помощь с этими изменениями – в этой теме форума.)
  • GUIConstants.au3 теперь идентична GUIConstantsEx.au3. Это означает, что многие скрипты с расширенными функциями GUI потребуют дополнительных директив #include для подключения файлов с константами, которые ранее включались ошибочно.
  • GUIDefaultConstants.au3 упразднена. Теперь нужно подключать отдельный файл констант для каждого типа элемента управления, например: ListBoxConstants.au3, ComboConstants.au3, EditConstants.au3 и т.д.
  • Документация _ArrayCreate() удалена. Функция по-прежнему присутствует, но может быть удалена позднее. Обновите скрипты: используйте синтаксис инициализации массивов, встроенный в AutoIt.

v3.2.10.0

AutoIt

  • DllCall(): типы short_ptr, long_ptr, int_ptr устарели. Используйте short*, long* и int* соответственно.

UDF

  • _GUICtrlXXXYYY() переименованы в _GUICtrlXXX_YYY() без функциональных изменений.
