Заголовки и текст окон (расширенно)

AutoIt работает в одном из трёх режимов "сопоставления окон". Режимы задаются через AutoItSetOption с опцией WinTitleMatchMode.

 

Режим 1 (по умолчанию)

Совпадение по началу заголовка.

В этом режиме окно с заголовком Untitled - Notepad будет соответствовать "Untitled - Notepad", "Untitled", "Un" и т. д.

Пример:

WinWait("Untitled")

 

Режим 2

Совпадение по любой подстроке заголовка.

В этом режиме окно с заголовком Untitled - Notepad будет соответствовать "Untitled - Notepad", "Untitled", "Notepad", "pad" и т. д.

Пример:

WinWait("Notepad")

 

Режим 3

Точное совпадение заголовка.

В этом режиме окно с заголовком Untitled - Notepad будет соответствовать только "Untitled - Notepad".

 

Режим 4 (оставлен для обратной совместимости)

Расширенный режим.

Заменён расширенными описаниями окон, работающими в любом режиме.

 

Режимы от -1 до -4

То же сопоставление, но без учёта регистра.

Расширенные описания окон

В качестве параметра title можно использовать специальное описание. Оно идентифицирует окно по следующим свойствам:

  • TITLE - заголовок окна.
  • CLASS - внутреннее имя класса окна.
  • REGEXPTITLE - заголовок окна, заданный регулярным выражением (если регулярное выражение неверное, @error устанавливается в 2).
  • REGEXPCLASS - имя класса окна, заданное регулярным выражением (если регулярное выражение неверное, @error устанавливается в 2).
  • LAST - последнее окно, использованное в предыдущей оконной функции AutoIt (см. функции управления окнами).
  • ACTIVE - текущее активное окно.
  • X \ Y \ W \ H – позиция и размер окна.
  • INSTANCE - номер экземпляра (с 1) при совпадении всех заданных свойств.

Одно или несколько свойств указываются в параметре title в формате:

[PROPERTY1 : Value1; PROPERTY2:Value2]

 

Примечание: если значение должно содержать ";", его нужно удвоить.

Например, дождаться окна с именем класса "Notepad":

WinWaitActive("[CLASS:Notepad]", "")

 

Например, закрыть текущее активное окно:

WinClose("[ACTIVE]", "")

 

Например, дождаться второго экземпляра окна с заголовком "My Window" и именем класса "My Class":

WinWaitActive("[TITLE:My Window; CLASS:My Class; INSTANCE:2]", "")

 

Например, перечислить окна, у которых заголовок соответствует регулярному выражению:

WinList("[REGEXPTITLE:(?i)(.*SciTE.*|.*Internet Explorer.*)]")

Дескрипторы окон / HWND

Тип variant в AutoIt нативно поддерживает дескрипторы окон (HWND). Дескриптор окна – это специальное значение, которое Windows назначает каждому окну при его создании. При наличии дескриптора его можно использовать вместо параметра title в любых функциях, использующих соглашение title/text. Преимущество в том, что если открыто несколько копий приложения с одинаковым title/text, их можно однозначно различать через дескрипторы. Когда вы передаёте дескриптор окна в качестве title, параметр text полностью игнорируется.

Различные функции, например WinGetHandle, WinList и GUICreate, возвращают такие дескрипторы. Важно: дескриптор окна не классифицируется ни как число, ни как строка – у него собственный специальный тип.

Примечание: дескрипторы окон работают независимо от текущего значения WinTitleMatchMode.

Пример

Local $hWnd = WinGetHandle("Untitled - Notepad", "")
WinClose($hWnd)
