Запуск скриптов

При запуске AutoIt вам будет предложено открыть файл скрипта. Файл скрипта – это обычный текстовый файл, содержащий ключевые слова и функции AutoIt, которые сообщают AutoIt, что нужно сделать. Файлы скриптов создаются в простом текстовом редакторе вроде notepad.exe, но есть и гораздо лучшая альтернатива – специально модифицированная версия редактора SciTE.

Хотя скрипты AutoIt v3 – обычные текстовые файлы, обычно им присваивают расширение .au3, чтобы отличать скрипт от обычного текстового файла. Если вы установили AutoIt полным установщиком, скрипт можно запустить простым двойным щелчком. Также по правому щелчку на файле .au3 доступны команды открытия, редактирования и компиляции скрипта.

 

Пример скрипта. Обратите внимание: символ ; используется для комментирования участков кода:

#include <MsgBoxConstants.au3>

; Это мой первый скрипт
MsgBox($MB_SYSTEMMODAL, "My First Script!", "Hello World!")

 

Более сложные скрипты могут включать функции. Их обычно размещают ближе к концу скрипта или ниже секции объявления глобальных переменных.

#include <MsgBoxConstants.au3>

MsgBox($MB_SYSTEMMODAL, "My second script!", "Hello from the main script!")

Example_Func()

Func Example_Func()
    Return MsgBox($MB_SYSTEMMODAL, "My Second Script!", "Hello from the functions!")
EndFunc   ;==>Example_Func

Параметры командной строки

В AutoIt можно передавать параметры командной строки в собственный исполняемый файл. Переданные параметры доступны через константные переменные $CmdLine и $CmdLineRaw. Попытка присвоить им новые данные приведёт к ошибке AutoIt – их нельзя менять во время выполнения скрипта. Обратите внимание: эти переменные существуют независимо от того, переданы параметры командной строки или нет.

Специальный массив $CmdLine инициализируется в начале работы скрипта параметрами командной строки. Если запускать скрипт, а не исполняемый файл, то имя ScriptName.au3 будет проигнорировано как параметр.

Если передаются строки с пробелами, их нужно экранировать "двойными кавычками" в строке команды.

$CmdLine[0] ; Содержит общее количество элементов в массиве.
$CmdLine[1] ; Первый параметр.
$CmdLine[2] ; Второй параметр.
...
$CmdLine[nth] ; n-ный параметр, например 10, если массив содержит 10 элементов.


Если запускать скрипт напрямую через AutoIt3.exe:

AutoIt3.exe myScript.au3 param1 "This is a string parameter" 99

$CmdLine[0] ; Здесь 3 параметра.
$CmdLine[1] ; Содержит param1, а не myScript.au3 – он игнорируется при запуске нескомпилированного скрипта.
$CmdLine[2] ; Содержит "This is a string parameter".
$CmdLine[3] ; Содержит 99.
$CmdLineRaw ; Содержит myScript.au3 param1 "This is a string parameter" 99.


Если запускать скомпилированный исполняемый файл с параметрами командной строки:

myProg.exe param1 "This is a string parameter" 99

$CmdLine[0] ; Здесь 3 параметра.
$CmdLine[1] ; Содержит param1.
$CmdLine[2] ; Содержит "This is a string parameter".
$CmdLine[3] ; Содержит 99.


Специальные ключи командной строки AutoIt

Интерпретатор AutoIt3.exe (или интерпретаторная заглушка любого скомпилированного скрипта AutoIt) обычно может запускать скрипты AutoIt прямо из командной строки. Во всех случаях ключ /ErrorStdOut позволяет перенаправить фатальную ошибку в StdOut, чтобы её мог перехватить, например, редактор SciTE. Этот ключ работает как с интерпретатором, так и со скомпилированным скриптом.

Запуск скрипта через интерпретатор:

AutoIt3.exe [/ErrorStdOut] [/AutoIt3ExecuteScript] filename [params ...]                 Выполнить AutoIt3-скрипт "filename" с необязательными параметрами.

В простейшем виде: AutoIt3.exe myScript.au3 запустит обычный AutoIt-скрипт myScript.au3 без параметров.

Запуск скомпилированного скрипта:

Compiled.exe [/ErrorStdOut] [params ...]
                Выполнить скомпилированный AutoIt3-скрипт, созданный с помощью Aut2Exe.

Запуск одной строки кода:

AutoIt3.exe [/ErrorStdOut] /AutoIt3ExecuteLine "command line"
                Выполнить одну строку кода.

Команда ниже выполнит одну строку кода и покажет MsgBox с "Hello World!". Значок в трее не будет отображаться.

Run(@AutoItExe & ' /AutoIt3ExecuteLine "MsgBox(4096, ''Hello World!'', ''Hi!'')"')

Запуск скрипта другим скомпилированным скриптом:

Compiled.exe [/ErrorStdOut] /AutoIt3ExecuteScript file [params ...]
                Выполнить другой AutoIt-скрипт из скомпилированного AutoIt3-исполняемого файла.

Compiled.exe [/ErrorStdOut] /AutoIt3ExecuteLine "command line"
                 Выполнить одну строку кода (как у AutoIt3.exe выше).

Это значит, что нет необходимости держать копию AutoIt3.exe рядом со скомпилированным файлом – интерпретаторная заглушка скомпилированного файла его заменит. Поэтому, если есть хотя бы один скомпилированный скрипт, другие AutoIt-скрипты можно запускать без AutoIt3.exe на машине – без предустановки и без добавления через FileInstall.

Важные примечания

Однако исполняемый файл должен быть скомпилирован с директивой #pragma compile(AutoItExecuteAllowed, True), так как настройка по умолчанию не позволяет использовать параметры /AutoItExecuteScript или /AutoItExecuteLine. Кроме того, поскольку интерпретаторная заглушка ищет нужные стандартные файлы #include в текущем каталоге, UDF-функции и различные константы могут быть недоступны – эти файлы обычно лежат только в полной установке AutoIt. Поэтому скрипты, которые планируется запускать таким образом, рекомендуется компилировать в формат .a3x, а не оставлять обычными .au3-файлами.

При работе с командной строкой важно правильно использовать одинарные и двойные кавычки – даже для удвоенных одинарных кавычек.
