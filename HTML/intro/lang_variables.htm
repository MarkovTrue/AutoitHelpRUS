Переменные

Переменная – это место в памяти для хранения данных, к которым можно быстро обратиться. Представьте её как почтовый ящик в памяти, куда можно класть и из которого можно забирать информацию. Например, можно создать переменную для хранения ответа пользователя на вопрос или результата математического выражения.

У каждой переменной есть имя (опять же, как у почтового ящика); оно должно начинаться с символа $ и может содержать только буквы, цифры и подчёркивание _. Примеры имён:

    $Var1

    $vVariable

    $my_Variable


Имена переменных нечувствительны к регистру: MyVariable() то же, что MyvARiAblE().

Каждая переменная хранится как variant.

Для облегчения сопровождения скрипта рекомендуется использовать стандартное соглашение об именовании переменных.

Объявление переменных

Переменные объявляются и создаются ключевыми словами Local и Global; можно использовать и Dim, но это не рекомендуется.

Local $vVariable

Можно объявить несколько переменных за раз:

Global $vVariable1, $vVariable2

Можно также присвоить переменной значение без предварительного объявления, но многие предпочитают явные объявления.

$vVariable = "Create and Assign"

Объявление констант

Константы объявляются и создаются ключевым словом Const и должны быть инициализированы значением:

Const $iConst1 = 1, $iConst2 = 12

Константы можно объявлять и инициализировать также ключевым словом Enum:

Enum $eConst1 = 1, $eConst2, $eConst3 ; 1, 2, 3
Enum Step 2 $eIncr0, $eIncr2, $eIncr4 ; 0, 2, 4
Enum Step *2 $eMult1, $eMult2, $eMult4 ; 1, 2, 4

После присвоения значения константе её нельзя изменить.

Область видимости

Область видимости переменной определяется тем, когда и как она объявлена. Если переменная объявлена в начале скрипта вне функций, она находится в глобальной области и доступна для чтения/записи из любого места скрипта.

Если переменная объявлена внутри функции, она находится в локальной области и может использоваться только внутри этой функции. Переменные, созданные внутри функций, автоматически уничтожаются при выходе из функции.

По умолчанию переменные, объявленные через Dim или присвоенные внутри функции, имеют локальную область видимости, если нет глобальной переменной с тем же именем (тогда переиспользуется глобальная). Это можно изменить, используя ключевые слова Local и Global для принудительного задания нужной области видимости.

Подробности использования ключевого слова Static в объявлении области видимости – по ссылке.

Массивы и Maps

В AutoIt есть 2 типа коллекций: массивы и Maps.

Массивы намного быстрее для произвольного доступа и могут иметь несколько измерений – размер измерения фиксируется при объявлении (хотя его можно изменить через ReDim). Массивы индексируются целыми числами, отражающими порядок элементов, начиная с element[0]; элементы извлекаются по целочисленному индексу или перебираются простым циклом For...Next.

Maps лучше подходят для записей/словарей и имеют единственное измерение. Они индексируются целыми или строковыми ключами (целые не отражают порядок элементов) и динамически меняют размер при добавлении/удалении значений. К значению можно обращаться только по исходному ключу – ключи можно перебирать функцией MapKeys.

Массивы

Массив – это переменная, содержащая ряд элементов данных. Каждый элемент доступен по индексу, который соответствует положению элемента в массиве; в AutoIt первый элемент массива – всегда element[0]. Элементы массива хранятся в определённом порядке и могут быть отсортированы.

Если в [ ] указать размер измерения – это объявит массив:

Local $aVar[3] ; Массив


Присвоение значений элементам при объявлении делает переменную массивом – эти три строки функционально эквивалентны:

Local $vVar[3] = [1, 2, 3] ; Массив
Local $vVar[] = [1, 2, 3] ; Массив
Local $vVar = [1, 2, 3] ; Массив


Пример:

Нужно сохранить список имён: "Jasper", "Beethoven", "Pinky" и "Fidget". Можно использовать четыре отдельные переменные, но массив – эффективнее:

Local $aArray[4]
$aArray[0] = "Jasper"
$aArray[1] = "Beethoven"
$aArray[2] = "Pinky"
$aArray[3] = "Fidget"

Для доступа к конкретному значению массива используется индекс:

$sString = $aArray[2] ; $sString содержит "Pinky"

Вместо номера индекса можно подставить другую переменную или выражение – так можно строить сложные способы присваивания и доступа к элементам массива.

Все элементы массива можно перебрать циклом For...Next:

; UBound возвращает общее количество элементов – так как первый [0] наивысший индекс на один меньше
For $i = 0 To UBound($aArray) - 1
    ConsoleWrite(aArray[$i] & @CRLF)
Next


Массивы могут быть многомерными – когда используется несколько серий индексов. Представьте строки и столбцы таблицы:

$aArray[0][0] = "Upper-Left"
$aArray[1][0] = "Lower-Left"
$aArray[0][1] = "Upper-Right"
$aArray[1][1] = "Lower-Right"

(Это значения только для примера.)

В массиве можно использовать до 64 измерений. Общее количество элементов не может превышать 2^24 (16 777 216).

Массивы должны быть объявлены до использования – с указанием области видимости через "Global/Local/Static" и либо размера, либо присваивания некоторых элементов.

Local $aArray1[2] ; Массив с 2 элементами, ни один не присвоен
Local $aArray[] = [8, 4, 5, 9, 1] ; Массив с 5 элементами, все присвоены
Local $aArray[7] = [3, 7.5, "string"] ; Массив с 7 элементами, присвоены только первые 3

Типы данных в массивах

Элемент массива может содержать любой тип данных AutoIt:

$aArray[0] = 1
$aArray[1] = True
$aArray[2] = "Text"
$aArray[3] = $aAnotherArray

Массив, хранящийся внутри другого массива, доступен напрямую, но этот способ немного медленнее, чем доступ к другим типам данных:

Local $aInternal[3] = ["A", "B", "C"] ; Объявить массив
Local $aContainer[1] = [$aInternal] ; Объявить контейнерный массив, содержащий первый
$sString = ($aContainer[0])[1] ; $sString содержит "B"
; Обратите внимание на требование заключить определение первого элемента массива в ( )

Maps

Внимание: эта возможность экспериментальная. Она может не работать, содержать ошибки или быть изменена/удалена без предупреждения.

НЕ СООБЩАЙТЕ ОБ ОШИБКАХ И НЕ ЗАПРАШИВАЙТЕ НОВЫЕ ВОЗМОЖНОСТИ ДЛЯ ЭТОЙ ВОЗМОЖНОСТИ.

Map – переменная, содержащая ряд элементов данных, каждый из которых – пара "ключ/значение". К элементу можно обращаться только по ключу – строковому или целочисленному. Однако целочисленный ключ не связан с положением элемента в Map – они хранятся без определённого порядка и не могут быть отсортированы. Строковые ключи в AutoIt, что почти уникально, чувствительны к регистру: "MyKey" и "mykey" – не одно и то же.

Использование пустых [ ] объявляет Map:

Local $mVar[] ; Карта


Пример:

Нужно хранить несколько переменных в коллекции, чтобы передавать их в функцию одним параметром. Map позволяет использовать строковые ключи – так получается удобное и понятное обращение к значению, например ControlID-серии контролов индексируются по их именам. Обратите внимание: к элементам можно обращаться через [] или точечную нотацию:

Local $mControls[]
$mControls["Input"] = GUICtrlCreateInput(....)
$mControls.Combo = GUICtrlCreateCombo(...)
$mControls["Button"] = GUICtrlCreateButton(...)

Для доступа к конкретному значению Map используется ключ – снова допустима любая из двух нотаций, и она может отличаться от использованной при присвоении:

$idControlID = $mMap.Input ; Переменная содержит ControlID для ввода
$idControlID = $mMap["Combo"] ; Переменная содержит ControlID для Combo

Целочисленные ключи должны использовать нотацию [] и не равны своим строковым эквивалентам:

$mMap[3] = "Integer 3" ; Это отдельные ключи
$mMap["3"] = "String 3"

Maps должны быть объявлены до использования – с указанием области видимости через "Global/Local/Static".

Local $mControls[]

Типы данных в Maps

Элемент Map может содержать любой тип данных AutoIt:

$mMap["Integer"] = 1
$mMap.Boolean = True
$mMap["String"] = "Text"
$mMap.Array = $aArray
$mMap["Map"] = $mAnotherMap

К внутренней Map можно обращаться напрямую:

Local $mInternal[] ; Объявить карту
$mInternal["Internal"] = "AutoIt3" ; Назначить элемент
Local $mContainer[] ; Объявить контейнерную карту
$mContainer.Bin = $mInternal ; Назначить первую карту как элемент
; Все они возвращают "AutoIt3"
$sString = $mContainer["Bin"]["Internal"]
$sString = $mContainer.Bin.Internal
$sString = $mContainer["Bin"].Internal
$sString = $mContainer.Bin["Internal"]

Имена переменных

Соглашение об именовании переменных в AutoIt основано на венгерской нотации. Префикс определяет логический, а не физический тип данных: так он подсказывает, для чего предназначена переменная или что она представляет. Префикс не кодирует фактический тип данных – он определяется при присваивании. Принятые стандарты см. в таблице ниже.

префикс покрываемый тип пример
a массивы $aArray[0]
b логические значения $bBool = True
d двоичные данные $dBinary = Binary("0x80000000")
e константная переменная Local Const $eEulersConstant = 2.7182818284590452
f с плавающей точкой $fFloat = 0.123
h дескрипторы (включая дескрипторы GUI) $hGUI = GUICreate("My GUI")
i целое число $iInteger = 10
id идентификатор контрола AutoIt $idButton_Ok = GUICtrlCreateButton("OK", 5, 5)
m Maps $mMap[]
n произвольное число (без предпочтений) $nNumber = 0
o объекты $oShell = ObjCreate("shell.application")
p указатели $pRect = DllStructGetPtr($tRECT)
s строки (включая символы) $sString = "Hello world"
t структуры $tSTRUCT = DllStructCreate($tagSTRUCT)
tag определение структуры $tagDATE = "struct; word Year;word Month;word Day; endstruct"
v Variant $vData = ClipGet()

Имена переменных строятся по схеме:

префикс "$" тип (нижний регистр) [необязательно] подтип (нижний регистр) имя переменной (первая буква – заглавная)
$ a h Handles

Примеры:

; Присвоить локальной переменной целое число 7
$iWeekDays = 7

; Присвоить локальной переменной значение Pi
$fPi = 3.14159265358979

; Присвоить локальной переменной массив строк
Local $asArray[7] = ["mon", "tue", "wed", "thu", "fri", "sat", "sun"]

; Присвоить локальной переменной массив чисел
Local $anArray[4] = [0, 0.25, 3 / 4, 12]

Инициализация переменных

При инициализации переменных есть несколько важных моментов. Плохая практика – занимать память данными, которые сразу не нужны. Поэтому рекомендуется объявлять и инициализировать переменные непосредственно перед использованием. Если переменной нужно присвоить значение по умолчанию, которое позже будет перезаписано, оно должно быть того же типа (или наиболее логичного представления данного типа) и занимать как можно меньше памяти.

Примеры:

; Несогласованные типы данных считаются плохими
Local $iInteger = "0"
Local $sString = False

; Правильная инициализация
Local $iInteger = 0
Local $sString = ""


В таблице ниже приведены рекомендуемые значения по умолчанию для каждого типа данных. У некоторых типов есть несколько возможных значений по умолчанию для инициализации.

значение по умолчанию покрываемый тип
Binary("") $d
"" $s, $v
0 $a, $h, $i, $id, $m, $n, $o, $p, $t, $tag, $v
0.0 $f
Null $o, $s, $v
False (or True) $b


Пример:

; Объявить и инициализировать переменную с рекомендуемым значением по умолчанию
Local $vUndefined = Null

; Некоторое время спустя
$vUndefined = 0xB0AD1CEA ; Назначить подходящее значение по мере необходимости

Чтобы уменьшить "раздутие" кода, несколько переменных можно объявлять в одной строке. При этом обычно рекомендуется объявлять в одной строке переменные одного типа – так код легче читать и сопровождать в будущем; впрочем, оптимальный вид зависит от ситуации.

Пример:

; Не рекомендуется
Local $sString = "", $iInteger = 0, $asArray = ["a","b","c"] ; Смешанные типы данных

; Рекомендуется
Local $iLeft = 10, $iTop = 10 ; Целые числа
Local $idButton_Go = GUICtrlCreateButton("Go", $iLeft, $iTop) ; ControlIds
Local $idButton_Quit = GUICtrlCreateButton("Quit", 50, 10) ; ControlIds

В некоторых языках инициализация переменных при объявлении обязательна – но в AutoIt это не так. С точки зрения типа данных, переменные, объявленные без инициализации, следует считать неопределёнными.

Области видимости переменных

Переменные также следует именовать в соответствии с их областью видимости.

глобальная UDF-переменная глобальная переменная локальная переменная
$__g_iSomeVar $g_iSomeVar $iSomeVar

Так удастся избежать нежелательных переприсваиваний.
