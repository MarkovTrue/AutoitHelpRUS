Запуск в 64-битных версиях Windows

Исторически AutoIt был 32-разрядным приложением. Начиная с релиза v3.2.10.0 добавлены нативные x64-версии некоторых компонентов:

  • AutoIt (AutoIt3_x64.exe)
  • Aut2Exe (Aut2Exe_x64.exe)
  • Au3Info (Au3Info_x64.exe)
  • AutoItX (AutoItX3_x64.dll)

Если при установке обнаруживается, что AutoIt запускается на x64-системе, вам предложат установить и сконфигурировать x64-версии. Эти версии полностью совместимы с x64, однако скрипты, использующие DllCall/DllStruct с пользовательскими структурами, могут содержать значения, нарушающие 64-разрядную совместимость (например, 32-битные целые для указателей).

Запустить x86-версию AutoIt можно щелчком правой кнопки по скрипту и выбором "Run Script (x86)".

Чтобы узнать, работаете ли вы в 64-разрядной редакции Windows, используйте макрос @OSArch; чтобы узнать, какая используется версия AutoIt – 32- или 64-разрядная – используйте @AutoItX64.

Запуск 32-разрядной версии AutoIt на x64-системе

Для файлов в Windows предусмотрен специальный механизм перенаправления для некоторых системных каталогов:

Каталоги 32-разрядное значение 64-разрядное значение
@SystemDir @WindowsDir & "\System32" @WindowsDir & "\SYSWOW64"
@ProgramFilesDir {SystemDrive} & "\Program Files (x86)" {SystemDrive} & "\Program Files"

Получить доступ к 64-разрядной версии этих каталогов можно, отключив механизм перенаправления:

#include <WinAPIFiles.au3>

_WinAPI_Wow64EnableWow64FsRedirection(True)

Дополнительные сведения см. на MSDN.

Для работы с реестром используйте HKCR64 или HKLM64, чтобы обойти перенаправление – см. документацию по функциям реестра.
