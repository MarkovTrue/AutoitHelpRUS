1. Как запустить DOS-программу из AutoIt?

Чтобы запустить что-то вроде команды DOS "Dir", нужно сделать это через командный интерпретатор (command.com или cmd.exe в зависимости от ОС). Макрос @ComSpec содержит правильное расположение этого файла. Используйте функцию RunWait() – она ждёт завершения DOS-программы перед переходом к следующей строке скрипта. Пример запуска DOS-команды Dir для диска C: (то есть выполнение команды command.com /c Dir C:\):

RunWait(@ComSpec & " /c Dir C:\")

2. Почему Run() работает только с .exe? А .msi / .txt и другие?

"Запускаемых" расширений всего несколько – .exe, .bat, .com, .pif. Другие типы файлов вроде .txt и .msi на самом деле выполняются другой программой. Когда вы дважды щёлкаете по "my_installer.msi", на самом деле в фоне выполняется "msiexec.exe my_installer.msi". Чтобы запустить .msi из AutoIt, нужно сделать так:

RunWait("msiexec my_installer.msi")

 

Или используйте команду "start", которая сама определит, чем запускать файл:

RunWait(@ComSpec & " /c Start my_installer.msi")

Или функцию ShellExecuteWait, которая тоже сама определит, чем запускать файл:

ShellExecuteWait("my_installer.msi")

3. Почему я получаю ошибки при использовании двойных кавычек (")?

Чтобы использовать двойные кавычки внутри строки, их нужно "удвоить". То есть на каждую кавычку, которую хотите получить, ставьте две. Например, чтобы присвоить переменной строку: A word in "this" sentence has quotes around it! – нужно сделать так:

$vVar = "A word in ""this"" sentence has quotes around it!"

Либо используйте одинарные кавычки:

$vVar = 'A word in "this" sentence has quotes around it!'

5. Почему я не могу вывести переменную через "My var is $vVariable"?

Если у вас есть переменная $msg и вы хотите показать её в MsgBox, такой код НЕ сработает:

#include <MsgBoxConstants.au3>

MsgBox($MB_OK, "Example", "My variable is $msg")

На самом деле выведется буквально My variable is $msg. Нужно объединить строку и переменную с помощью оператора &:

#include <MsgBoxConstants.au3>

MsgBox($MB_OK, "Example", "My variable is " & $msg)

 

Дополнительно: если в строку нужно вставить несколько переменных, удобнее использовать StringFormat(). Например, чтобы вставить $vVar1 – $vVar5 в строку, проще написать:

#include <MsgBoxConstants.au3>

Local $sMessage = StringFormat("Var1 is %s, Var2 is %s, Var3 is %s, Var4 is %s", $vVar1, $vVar2, $vVar3, $vVar4)
MsgBox($MB_SYSTEMMODAL, "", $sMessage)

6. При использовании Send() для отправки переменной происходят странные вещи?

Если вы отправляете содержимое переменной, помните: специальные символы Send (! ^ + {SPACE}) будут интерпретированы как особые нажатия – обычно это нежелательно. Чтобы это обойти, используйте RAW-режим Send() – он не преобразует специальные клавиши:

Send($vMyVar, 1)

7. В чём разница между возвращаемым значением и @error?

Обычно возвращаемое значение указывает на успех функции. Но если функция уже что-то возвращает (например, WinGetText()), нужен другой способ узнать, успешно ли она выполнилась – тогда вместо этого устанавливается @error.

8. Как завершить скрипт по горячей клавише?

Лёгкий вопрос. Если хотите, чтобы скрипт завершался при нажатии определённой комбинации клавиш, используйте функцию HotKeySet() – она запускает пользовательскую функцию при нажатии заданной клавиши. Эта функция должна содержать только ключевое слово Exit.

Код, который завершает скрипт по нажатию CTRL+ALT+x:

HotKeySet("^!x", "MyExit")
...
...
; Остальная часть скрипта
...
...
Func MyExit()
    Exit
EndFunc   ;==>MyExit

Технические ограничения AutoIt3 см. в разделе Ограничения и значения по умолчанию.

Дополнительная информация о теоретических ограничениях AutoIt3.

Значение Описание

Попробуйте зарегистрировать его командой "regsvr32 hhctrl.ocx" из командной строки или проверьте, не повреждён ли файл.
