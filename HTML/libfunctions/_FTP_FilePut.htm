Справочник функций

_FTP_FilePut

Загружает файл на FTP-сервер.

#include <FTPEx.au3>
_FTP_FilePut ( $hFTPSession, $sLocalFile, $sRemoteFile [, $iFlags = 0 [, $iContext = 0]] )

Параметры

$hFTPSession возвращается функцией _FTP_Connect()
$sLocalFile Локальный файл.
$sRemoteFile Удалённое расположение файла.
$iFlags [необязательный] Условия передачи файла. Выберите один тип передачи и при необходимости флаги управления кешированием.

Тип передачи – одно из значений:
$FTP_TRANSFER_TYPE_ASCII (1) – передача методом ASCII FTP (Type A); управляющая и форматирующая информация преобразуется в локальные эквиваленты.
$FTP_TRANSFER_TYPE_BINARY (2) – передача методом Image FTP (Type I); файл передаётся без изменений. Метод по умолчанию.
$FTP_TRANSFER_TYPE_UNKNOWN (0) – соответствует $FTP_TRANSFER_TYPE_BINARY.
$INTERNET_FLAG_TRANSFER_ASCII (1) – передача в режиме ASCII.
$INTERNET_FLAG_TRANSFER_BINARY (2) – передача в двоичном режиме.

Для управления кешированием можно указать одно или несколько из следующих значений:
$INTERNET_FLAG_HYPERLINK (0x00000400) – принудительная перезагрузка, если сервер не вернул время истечения срока действия и дату последнего изменения.
$INTERNET_FLAG_NEED_FILE (0x00000010) – создаёт временный файл, если файл не может быть кеширован.
$INTERNET_FLAG_RELOAD (0x80000000) – принудительная загрузка с исходного сервера, минуя кеш.
$INTERNET_FLAG_RESYNCHRONIZE (0x00000800) – перезагружает HTTP-ресурс, если он изменился с момента последней загрузки; все FTP- и Gopher-ресурсы перезагружаются всегда.
$iContext [необязательный] Переменная, содержащая определённое приложением значение, которое связывает этот поиск с данными приложения.
Используется только если приложение уже вызывало _FTP_SetStatusCallback() для установки функции обратного вызова статуса.

Возвращаемое значение

Успех: Возвращает 1.
Ошибка: Возвращает 0 и устанавливает @error ≠ 0.

Примечания

Параметры $sLocalFile и $sRemoteFile могут быть как частичными, так и полными именами относительно текущей папки.
