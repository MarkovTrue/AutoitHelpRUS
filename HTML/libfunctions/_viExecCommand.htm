Справочник функций

_viExecCommand

Отправляет команду или запрос на прибор или устройство через интерфейс VISA (GPIB / TCP).

#include <Visa.au3>
_viExecCommand ( $hSession, $sCommand [, $iTimeoutMS = -1 [, $sMode = @LF]] )

Параметры

$hSession Дескриптор VISA (СТРОКА) ИЛИ дескриптор сеанса VISA (ЦЕЛОЕ ЧИСЛО)
Это может быть строка или целое число (дескриптор):
* СТРОКА – ДЕСКРИПТОР VISA – это строка, которая задаёт ресурс для установления сеанса связи.
Пример дескриптора: "GPIB::20::0".
Эта функция поддерживает все допустимые дескрипторы VISA, включая приборы GPIB, TCP, VXI и серийного интерфейса (ASRL).
Подробное объяснение дескрипторов VISA показано в разделе примечаний этой функции.
В качестве СОКРАЩЕНИЯ вы также можете использовать СТРОКУ, содержащую номер адреса (например, "20") прибора GPIB вместо ввода полного дескриптора (в этом случае, "GPIB::20::0")
* ЦЕЛОЕ ЧИСЛО – дескриптор сеанса VISA – это целое число, возвращённое функцией _viOpen().
Рекомендуется использовать _viOpen() и дескрипторы сеансов VISA вместо дескрипторов, если вы планируете неоднократно взаимодействовать с прибором или устройством, так как в противном случае каждый раз при контакте с прибором вы будете нести затраты на открытие и закрытие канала связи.
После завершения работы с прибором не забудьте закрыть канал функцией _viClose().
$sCommand Команда или запрос к выполнению (например, "*IDN?" или "SOURCE:POWER -20 dBM")
Запрос ДОЛЖЕН содержать ВОПРОСИТЕЛЬНЫЙ ЗНАК (?)
Когда команда является ЗАПРОСОМ, функция будет автоматически ждать ответа от прибора (или до истечения тайм-аута операции).
$iTimeoutMS [необязательный] Тайм-аут операции в МИЛЛИСЕКУНДАХ.
Это важно главным образом только для ЗАПРОСОВ.
Если не указано, будет использован последний установленный тайм-аут. Если он никогда не был установлен, будет использован тайм-аут по умолчанию (который зависит от реализации VISA).
Тайм-ауты также можно устанавливать отдельно функцией _viSetTimeout().
В зависимости от типа шины (GPIB, TCP и т. д.) тайм-аут может быть установлен не на точное значение, которое вы запросили. Вместо этого будет использован ближайший действительный тайм-аут больше, чем запрошенный.
$sMode [необязательный] Управляет режимом, в котором вызывается VISA viPrintf, когда $sCommand не является запросом.
По умолчанию @LF, что означает "режим присоединения @LF".
Некоторые приборы, и в частности многие карты GPIB, не учитывают атрибут символа завершения; в этих случаях нужно добавить завершитель @LF.
Поскольку это наиболее распространённый случай, по умолчанию режим установлен на @LF, который добавляет @LF к команде SCPI; вы также можете установить этот режим на @CR и @CRLF, если ваша карта использует эти завершители.
Если вы не хотите использовать завершитель, установите этот параметр на пустую строку ("")
Кроме того, некоторые карты поддерживают выполнение "sprintf" на строке SCPI перед отправкой её через интерфейс VISA.
Для тех, кто это делает, можно, установив этот параметр на "str", "защитить" интерфейс VISA от случайного применения последовательности выхода при обнаружении "/" в строке команды VISA.
Обычно это НЕ требуется и должно быть установлено только если ваша карта GPIB или прибор это требуют.

Возвращаемое значение

Возвращает значение, зависящее от того, является ли команда запросом (QUERY) и была ли операция успешной.

* Command, NON QUERY:
Успех: Возвращает 0.
Ошибка: -1 if the VISA DLL could not be open
or a NON ZERO value representing the VISA error code (see the VISA programmer's guide)

Успех: the answer of the instrument to the QUERY
Ошибка: -1 if the VISA DLL could not be open
-3 if the VISA DLL returned an unexpected number of results
or a NON ZERO value representing the VISA error code (see the VISA programmer's guide)

Примечания

* VISA-запросы возвращают только первую строку ответа устройства.
В большинстве случаев это не проблема, так как большинство устройств отвечают одной строкой.

* Ниже описаны НАИБОЛЕЕ РАСПРОСТРАНЁННЫЕ ДЕСКРИПТОРЫ VISA
Есть и другие типы. Подробнее см. руководство программиста VISA (доступно на www.ni.com).
Необязательные сегменты показаны в квадратных скобках ([]).
Обязательные заполняемые сегменты обозначены угловыми скобками (<&gr;).

Интерфейс    Синтаксис
------------------------------------------------------------
GPIB INSTR    GPIB[board]::primary address
    [::secondary address] [::INSTR]
GPIB INTFC    GPIB[board]::INTFC
TCPIP SOCKET    TCPIP[board]::host address::port::SOCKET
Serial INSTR    ASRL[board][::INSTR]
PXI INSTR    PXI[board]::device[::function][::INSTR]
VXI INSTR    VXI[board]::VXI logical address[::INSTR]
GPIB-VXI INSTR    GPIB-VXI[board]::VXI logical address[::INSTR]
TCPIP INSTR    TCPIP[board]::host address[::LAN device name]
    [::INSTR]

Ключевое слово GPIB используется для GPIB-приборов.
Ключевое слово TCPIP используется для TCP/IP-связи.
Ключевое слово ASRL используется для последовательных приборов.
Ключевое слово PXI используется для PXI-приборов.
Ключевое слово VXI используется для VXI-приборов через встроенные или MXIbus-контроллеры.
Ключевое слово GPIB-VXI используется для VXI-приборов через GPIB-VXI-контроллер.

Значения по умолчанию для необязательных параметров приведены ниже.

Необязательный сегмент    Значение по умолчанию
---------------------------------------
board    0
secondary address    нет
LAN device name    inst0

Пример

; - Предполагает, что прибор установлен на адрес GPIB 3
; Если прибор находится по другому адресу, измените "GPIB::3::0" на
; соответствующий дескриптор. То же самое для вызова _viOpen
; Демонстрирует использование _viExecCommand в автономном режиме и в сочетании
; с _viOpen и _viClose.
; Также демонстрирует функцию _viGTL

#include <MsgBoxConstants.au3>
#include <Visa.au3>

Local $h_Session = 0

; Запросить ID прибора по адресу GPIB 3
MsgBox($MB_SYSTEMMODAL, "Step 1", "Simple GPIB query using a VISA Descriptor")
Local $s_Answer = _viExecCommand("GPIB::3::0", "*IDN?", 10000) ; таймаут 10 секунд
MsgBox($MB_SYSTEMMODAL, "GPIB QUERY result", $s_Answer) ; Показать ответ

MsgBox($MB_SYSTEMMODAL, "Step 2", "Go to LOCAL using VISA Descriptor")
_viGTL("GPIB::1::0") ; Перейти в локальный режим (выйти из удалённого управления)

MsgBox($MB_SYSTEMMODAL, "Step 3", "Simple GPIB query using a VISA address shortcut")
$s_Answer = _viExecCommand("1", "*IDN?") ; Адрес ДОЛЖЕН БЫТЬ СТРОКОЙ
MsgBox($MB_SYSTEMMODAL, "GPIB QUERY result", $s_Answer) ; Показать ответ
MsgBox($MB_SYSTEMMODAL, "Info", "Now let's use _viOpen and _viClose")

MsgBox($MB_SYSTEMMODAL, "Step 4", "Open the instrument connection with _viOpen")
Local $h_Instr = _viOpen(3)
MsgBox($MB_SYSTEMMODAL, "Instrument Handle obtained", "$h_Instr = " & $h_Instr) ; Показать дескриптор сессии
; Запросить прибор

MsgBox($MB_SYSTEMMODAL, "Step 5", "Query the instrument using the VISA instrument handle")
$s_Answer = _viExecCommand($h_Instr, "*IDN?") ; $h_Instr теперь НЕ СТРОКА!
MsgBox($MB_SYSTEMMODAL, "GPIB QUERY result", $s_Answer) ; Показать ответ
; Запросить снова. Нет необходимости открывать соединение заново

MsgBox($MB_SYSTEMMODAL, "Step 6", "Query again. There is no need to OPEN the link again")
$s_Answer = _viExecCommand($h_Instr, "*IDN?")
MsgBox($MB_SYSTEMMODAL, "GPIB QUERY result", $s_Answer) ; Показать ответ

MsgBox($MB_SYSTEMMODAL, "Step 7", "Go to LOCAL using VISA instrument handle")
_viGTL($h_Instr) ; Перейти в локальный режим (опционально)

MsgBox($MB_SYSTEMMODAL, "Step 8", "Close the Instrument connection using _viClose")
_viClose($h_Instr) ; Закрыть соединение с прибором
