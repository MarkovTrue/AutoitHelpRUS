Справочник функций

_SQLite_Open

Открывает или создаёт базу данных SQLite.

#include <SQLite.au3>
_SQLite_Open ( [$sDatabase_Filename = ":memory:" [, $iAccessMode = Default [, $iEncoding = $SQLITE_ENCODING_UTF8 [, $bFKConstraints = False]]]] )

Параметры

$sDatabase_Filename [необязательный] Имя файла базы данных; по умолчанию откроет базу данных в памяти.
$iAccessMode [необязательный] флаги режима доступа. По умолчанию $SQLITE_OPEN_READWRITE + $SQLITE_OPEN_CREATE
$iEncoding [необязательный] флаг режима кодирования, используется только при создании. По умолчанию $SQLITE_ENCODING_UTF8
$bFKConstraints [необязательный] Ограничения внешних ключей, True = ON
False = OFF (по умолчанию).

Возвращаемое значение

Успех: Возвращает дескриптор базы данных.
Ошибка: Возвращает 0.
@error: -1 – SQLite сообщил об ошибке (проверьте значение @extended)
1 – ошибка при вызове SQLite API "sqlite3_open_v2"
2 – ошибка при преобразовании имени файла в UTF-8
3 – _SQLite_Startup() ещё не был вызван
@extended: значение, которое можно сравнить с константами $SQLITE_*

Примечания

Нет необходимости сохранять дескриптор базы данных, кроме случаев использования нескольких баз данных в одной сессии.
Функции, использующие дескриптор, по умолчанию используют последнюю открытую.

Для создания базы данных, закодированной в UFT16, создайте её с $iEncoding = $SQLITE_ENCODING_UTF16.

Можно открыть базы данных в памяти, временные и постоянные:
    $hDb = _SQLite_Open() ; открывает временную приватную БД в памяти
    $hDb = _SQLite_Open(Default, ...)    ; то же
    $hDb = _SQLite_Open(':memory:', ...)    ; то же

    $hDb = _SQLite_Open('', ...) ; открывает временную приватную БД на диске

    $hDb = _SQLite_Open('abc.db', ...) ; открывает или создаёт постоянную общедоступную БД на диске с названием "abc.db"

    На случай если вы себя запутаете:
    $hDb = _SQLite_Open('./:memory:', ...) ; открывает или создаёт постоянную общедоступную БД на диске с названием ":memory:"

В отличие от постоянных БД на диске, БД в памяти и временные БД не могут быть общедоступны и использоваться для IPC (межпроцессного взаимодействия) и уничтожаются в конце соединения.

Ограничения внешних ключей можно изменять, используя _SQLite_ForeignKeys().

См. также

_SQLite_Close, _SQLite_ForeignKeys

Пример

#include <File.au3>
#include <MsgBoxConstants.au3>
#include <SQLite.au3>
#include <SQLite.dll.au3>

_SQLite_Startup()
If @error Then
    MsgBox($MB_SYSTEMMODAL, "SQLite Error", "SQLite3.dll Can't be Loaded!")
    Exit -1
EndIf
ConsoleWrite("_SQLite_LibVersion=" & _SQLite_LibVersion() & @CRLF)

_SQLite_Open() ; Создание БД в памяти (:memory:) и неиспользование её дескриптора
If @error Then
    MsgBox($MB_SYSTEMMODAL, "SQLite Error", "Can't create a memory Database!")
    Exit -1
EndIf
_SQLite_Close()

Local $hMemDb = _SQLite_Open() ; Создать :memory: базу данных
If @error Then
    MsgBox($MB_SYSTEMMODAL, "SQLite Error", "Can't create a memory Database!")
    Exit -1
EndIf

Local $hTmpDb = _SQLite_Open('') ; Создание временной БД на диске
If @error Then
    MsgBox($MB_SYSTEMMODAL, "SQLite Error", "Can't create a temporary Database!")
    Exit -1
EndIf

Local $sDbName = _TempFile()
Local $hDskDb = _SQLite_Open($sDbName) ; Открытие постоянной БД на диске
If @error Then
    MsgBox($MB_SYSTEMMODAL, "SQLite Error", "Can't open or create a permanent Database!")
    Exit -1
EndIf

; используем 3 БД по мере необходимости, ссылаясь на их дескрипторы

; закрытие созданных БД в любом порядке
_SQLite_Close($hTmpDb) ; временные БД автоматически удаляются при закрытии
_SQLite_Close($hDskDb) ; БД – обычный файл, который можно открыть снова позже
_SQLite_Close($hMemDb)

; эта БД нам не нужна
FileDelete($sDbName)

_SQLite_Shutdown()
