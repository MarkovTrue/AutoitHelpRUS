Справочник функций

_WinAPI_ReadDirectoryChanges

Получает информацию, описывающую изменения в указанном каталоге.

#include <WinAPIFiles.au3>
_WinAPI_ReadDirectoryChanges ( $hDirectory, $iFilter, $pBuffer, $iLength [, $bSubtree = 0] )

Параметры

$hDirectory Дескриптор каталога для мониторинга. Этот каталог должен быть открыт с правом доступа $FILE_LIST_DIRECTORY.
$iFilter Критерии фильтра, которые функция проверяет для определения завершения операции ожидания.
Этот параметр может быть одним или несколькими из следующих значений.
$FILE_NOTIFY_CHANGE_FILE_NAME
$FILE_NOTIFY_CHANGE_DIR_NAME
$FILE_NOTIFY_CHANGE_ATTRIBUTES
$FILE_NOTIFY_CHANGE_SIZE
$FILE_NOTIFY_CHANGE_LAST_WRITE
$FILE_NOTIFY_CHANGE_LAST_ACCESS
$FILE_NOTIFY_CHANGE_CREATION
$FILE_NOTIFY_CHANGE_SECURITY
$pBuffer Указатель на буфер, выровненный по DWORD, который внутренне используется этой функцией для получения данных.
Для создания буфера вы можете использовать функцию _WinAPI_CreateBuffer().
Чтобы избежать сбоя скрипта, используйте буфер размером не менее 64 КБ.
Если буфер больше 64 КБ и приложение отслеживает каталог по сети, функция завершится с ошибкой.
Это связано с ограничением размера пакета протоколов совместного доступа к файлам.
$iLength Размер буфера в байтах.
$bSubtree [необязательный] Указывает, отслеживать ли подкаталоги указанного каталога, допустимые значения:
True – отслеживает дерево каталогов с корнем в указанном каталоге.
False – отслеживает только указанный каталог (по умолчанию).

Возвращаемое значение

Успех: Возвращает двумерный массив со следующей информацией:
[0][0] – количество строк в массиве (n)
[0][1] – не используется
[n][0] – имя файла относительно дескриптора каталога.
[n][1] – тип произошедшего изменения (одна из констант $FILE_ACTION_*).
Ошибка: Устанавливает @error ≠ 0; вызовите _WinAPI_GetLastError() для получения подробных сведений об ошибке.

Примечания

При первом вызове функции _WinAPI_ReadDirectoryChanges() система выделяет буфер для хранения информации об изменениях.
Этот буфер связан с дескриптором каталога до его закрытия и не меняет размер на протяжении своего времени жизни.
Изменения каталога, произошедшие между вызовами этой функции, добавляются в буфер и затем возвращаются при следующем вызове.
Если буфер переполняется, все его содержимое отбрасывается.

Для получения дескриптора каталога используйте функцию _WinAPI_CreateFileEx() с флагом $FILE_FLAG_BACKUP_SEMANTICS.

Функция _WinAPI_ReadDirectoryChanges() работает только в синхронном режиме.

Пример

#include <Array.au3>
#include <AutoItConstants.au3>
#include <FileConstants.au3>
#include <MsgBoxConstants.au3>
#include <WinAPIError.au3>
#include <WinAPIFiles.au3>
#include <WinAPIMem.au3>

Global $g_sPath = @TempDir & '\~TEST~'

DirCreate($g_sPath)
If Not FileExists($g_sPath) Then
    MsgBox(($MB_ICONERROR + $MB_SYSTEMMODAL), 'Error', 'Unable to create folder.')
    Exit
EndIf

OnAutoItExitRegister('OnAutoItExit')

Local $hDirectory = _WinAPI_CreateFileEx($g_sPath, $OPEN_EXISTING, $FILE_LIST_DIRECTORY, BitOR($FILE_SHARE_READ, $FILE_SHARE_WRITE), $FILE_FLAG_BACKUP_SEMANTICS)
If @error Then
    _WinAPI_ShowLastError('', 1)
EndIf

Local $pBuffer = _WinAPI_CreateBuffer(8388608)

Local $aData
While 1
    $aData = _WinAPI_ReadDirectoryChanges($hDirectory, BitOR($FILE_NOTIFY_CHANGE_FILE_NAME, $FILE_NOTIFY_CHANGE_DIR_NAME), $pBuffer, 8388608, 1)
    If Not @error Then
        _ArrayDisplay($aData, '_WinAPI_ReadDirectoryChanges')
    Else
        _WinAPI_ShowLastError('', 1)
    EndIf
WEnd

Func OnAutoItExit()
    DirRemove($g_sPath, $DIR_REMOVE)
EndFunc   ;==>OnAutoItExit
