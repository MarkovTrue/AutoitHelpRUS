Справочник функций

_SQLite_Startup

Загружает SQLite3.dll.

#include <SQLite.au3>
_SQLite_Startup ( [$sDll_Filename = "" [, $bUTF8ErrorMsg = False [, $iForceLocal = 0 [, $hPrintCallback = $__g_hPrintCallback_SQLite [, $bAutoItTypeConversion = False]]]]] )

Параметры

$sDll_Filename [необязательный] Имя файла DLL. По умолчанию "sqlite3.dll" или "sqlite3_x64.dll" в режиме X64.
$bUTF8ErrorMsg [необязательный] принуждение ConsoleWrite() отображать символы UTF8
$iForceLocal [необязательный] 1 = используйте определённый файл DLL. Без проверки версии. Автоматически "_x64.dll" в режиме X64.
0 = (по умолчанию) DLL будет искаться в @ScriptDir, @WorkingDir, @LocalAppDataDir & "\AutoIt v3\SQLite\", @SystemDir и @WindowsDir.
$hPrintCallback [необязательный] Переменная, присвоенная определённой пользователем функции для отображения диагностического сообщения SQLite. По умолчанию = __SQLite_ConsoleWrite(). См. раздел Примечания.
$bAutoItTypeConversion [необязательный] Принудительное преобразование из строки в переменную типа AutoIt (по умолчанию = False). См. раздел Примечания.

Возвращаемое значение

Успех: Возвращает путь к открытому файлу DLL.
Ошибка: Устанавливает @error ≠ 0.

Примечания

Если включён #include <SQLite.dll.au3>, версия SQLite проверяется.
Файлы SQLite можно загрузить с https://www.autoitscript.com/autoit3/pkgmgr/sqlite/sqlite3_setup_version.exe, которые можно хранить в @ScriptDir, @WorkingDir, @LocalAppDataDir & "\AutoIt v3\SQLite\" (по умолчанию), @SystemDir или @WindowsDir.
Для использования любым пользователем храните файлы SQLite в @SystemDir или @WindowsDir.
Для помощи при загрузке правильной версии можно использовать скрипт "Extras\AutoUpdateIt\AutoIt3SQLiteUpdate.au3" (похож на AutoUpdateIt.au3, используемый для проверки версии AutoIt или бета).

Если $bUTF8ErrorMsg не равно нулю, выводит строку в формате UTF8 в консоль.
Это позволяет приложениям, таким как SciTE, отображать эти символы при настройке на output.code.page=65001.
Это происходит при запуске SciTE под неанглийской версией.

Параметр $hPrintCallback указывает имя функции обратного вызова, определённой пользователем. Этой функции будет передан единственный параметр – сообщение для отображения. Обратный вызов по умолчанию – $__g_hPrintCallback_SQLite. Для примера реализации собственной функции обратного вызова см. эту функцию.

$bAutoItTypeConversion полезна, если весь скрипт хочет возвращать переменные типа AutoIt через _SQLite_FetchData().
По умолчанию возвращается строка или бинарная переменная AutoIt.

См. также

_SQLite_Shutdown, _SQlite_FetchData

Пример

Пример 1

; Версия SQLite.dll должна совпадать

#include <MsgBoxConstants.au3>
#include <SQLite.au3>
#include <SQLite.dll.au3>

Local $sSQliteDll = _SQLite_Startup()

If @error Then
    MsgBox($MB_SYSTEMMODAL, "SQLite Error", "SQLite3.dll Can't be Loaded!" & @CRLF & @CRLF & _
            "Not FOUND in @ScriptDir, @WorkingDir," & @CRLF & @TAB & "@LocalAppDataDir\AutoIt v3\SQLite," & @CRLF & @TAB & "@SystemDir or @WindowsDir")
    Exit -1
EndIf

MsgBox($MB_SYSTEMMODAL, "SQLite3.dll Loaded", $sSQliteDll & " (" & _SQLite_LibVersion() & ")")
ConsoleWrite("_SQLite_LibVersion=" & _SQLite_LibVersion() & @CRLF)
_SQLite_Shutdown()

Пример 2

; нет ссылки на версию, поэтому файл должен существовать в @ScriptDir, @WorkingDir, @SystemDir или @WindowsDir

#include <MsgBoxConstants.au3>
#include <SQLite.au3>
#include <SQLite.dll.au3>

Local $sSQliteDll
$sSQliteDll = _SQLite_Startup()
If @error Then
    MsgBox($MB_SYSTEMMODAL, "SQLite Error", "SQLite3.dll Can't be Loaded!" & @CRLF & @CRLF & _
            "Not FOUND in @ScriptDir, @WorkingDir, @SystemDir or @WindowsDir")
    Exit -1
EndIf

MsgBox($MB_SYSTEMMODAL, "SQLite3.dll Loaded", $sSQliteDll & " (" & _SQLite_LibVersion() & ")")
ConsoleWrite("_SQLite_LibVersion=" & _SQLite_LibVersion() & @CRLF)
_SQLite_Shutdown()

Пример 3

; открытие SQLite с определённым локальным файлом

#include <MsgBoxConstants.au3>
#include <SQLite.au3>
#include <SQLite.dll.au3>

Local $sLocalSQLiteDll = "local SQLite.dll" ; следует указать существующий .dll, чтобы избежать ошибки

Local $sSQliteDll = _SQLite_Startup($sLocalSQLiteDll, False, 1)
If @error Then
    MsgBox($MB_SYSTEMMODAL, "SQLite Error", "'SQLite3.dll' Can't be Loaded!")
    Exit -1
EndIf

MsgBox($MB_SYSTEMMODAL, "SQLite3.dll Loaded", $sSQliteDll & " (" & _SQLite_LibVersion() & ")")
ConsoleWrite("_SQLite_LibVersion=" & _SQLite_LibVersion() & @CRLF)
_SQLite_Shutdown()
