_WinAPI_SendInput

Синтезирует нажатия клавиш, движения мыши и щелчки кнопок.

#include <WinAPISys.au3>
_WinAPI_SendInput ( $iType, $nInputs, $tInputArray )

Параметры

$iType Тип записей массива $InputArray ($INPUT_MOUSE, $INPUT_KEYBOAARD или $INPUT_HARDWARE).
$nInputs Количество структур в массиве $tInputArray.
$tInputArray Массив структур $tag*INPUT. Каждая структура представляет событие для вставки в поток ввода клавиатуры или мыши.

Возвращаемое значение

Успех: Возвращает количество событий, успешно вставленных в поток ввода клавиатуры или мыши.
Ошибка: Установить @error и @extended.
@error: 10 – неправильный тип $iType
11 – неправильный размер $tInputArray.
12 – событие не отправлено; ввод уже заблокирован другим потоком.
13 – не все события установлены. возвращаемое значение равно количеству использованных событий.
@extended: _WinAPI_GetLastError().

Примечания

Каждая запись массива является одной из структур $tagMOUSEINPUT, $tagKEYBDINPUT или $tagHARDWAREINPUT. Тип каждой записи будет принудительно установлен в $iType.

Функция _WinAPI_SendInput() вставляет события в структурах INPUT последовательно в поток ввода клавиатуры или мыши.
Эти события не перемешиваются с другими событиями ввода клавиатуры или мыши, вставленными пользователем (с помощью клавиатуры или мыши) или с помощью вызовов keybd_event, mouse_event или других вызовов SendInput.

Полное объяснение см. на MSDN.

См. также

$tagKEYBDINPUT, $tagMOUSEINPUT, $tagHARDWAREINPUT

    #cs ==========================================================
https://www.autoitscript.com/forum/topic/210029-keypress-simulation-with-winapi-functions-directly/#comment-1518451

Используя SendInput win32 API, этот пример отправляет строку
из "a" в последний активный экземпляр блокнота (если он существует).

Примечание:
Если пользователь нажимает другие клавиши во время отправки строки "a",
они буферизуются и отображаются ПОСЛЕ строки "a"

Это огромное различие с функцией Send(), где
отображение будет перемешиваться, когда вы отправляете, например.
строку из 5000 "a" с одной инструкцией Send().

Функция SendInput работает путём внедрения серии смоделированных событий ввода в поток ввода устройства.
Эффект похож на повторный вызов функции keybd_event или mouse_event, за исключением того, что система гарантирует, что никакие другие события ввода не будут перемешиваться с смоделированными событиями.
Когда вызов завершён, возвращаемое значение указывает количество успешно воспроизведённых событий ввода. Если это значение равно нулю, ввод был заблокирован.
#ce ==========================================================

    ; Строка перемещена внутри функции, иначе скрипт не будет работать с AutoIt 3.3.14.5 ("includes chain" изменилась с 3.3.14.5 на 3.3.16.1)

    Local $iNumberOfInputs = 500 * 2 ; => 1000 вводов (нажатие/отпускание) => 500 символов отображено
    If $iNumberOfInputs < 2 Or (($iNumberOfInputs / 2) <> Int($iNumberOfInputs / 2)) Then _
            Exit MsgBox($MB_ICONERROR, "$iNumberOfInputs", "It has to be an even number, greater or equal to 2")

    ;Создание структуры для получения размера
    Local $tKeybdInput = DllStructCreate($tagKEYBDINPUT)

    ;Создание буфера массива вводов
    Local $tInputArray = DllStructCreate(StringFormat("byte bytes[%i];", DllStructGetSize($tKeybdInput) * $iNumberOfInputs))

    For $i = 0 To $iNumberOfInputs - 3 Step 2

        ;Загрузить input array[$i]
        $tKeybdInput = DllStructCreate($tagKEYBDINPUT, DllStructGetPtr($tInputArray) + ($i * DllStructGetSize($tKeybdInput)))
        $tKeybdInput.wVk = $VK_A

        ;Загрузить input array[$i + 1]
        $tKeybdInput = DllStructCreate($tagKEYBDINPUT, DllStructGetPtr($tInputArray) + (($i + 1) * DllStructGetSize($tKeybdInput)))
        $tKeybdInput.wvk = $VK_A
        $tKeybdInput.dwFlags = $KEYEVENTF_KEYUP
    Next

    ;Загрузить input array[$i]
    $tKeybdInput = DllStructCreate($tagKEYBDINPUT, DllStructGetPtr($tInputArray) + ($i * DllStructGetSize($tKeybdInput)))
    $tKeybdInput.wVk = $VK_RETURN

    ;Загрузить input array[$i + 1]
    $tKeybdInput = DllStructCreate($tagKEYBDINPUT, DllStructGetPtr($tInputArray) + (($i + 1) * DllStructGetSize($tKeybdInput)))
    $tKeybdInput.wvk = $VK_RETURN
    $tKeybdInput.dwFlags = $KEYEVENTF_KEYUP

    ;Активирование окна блокнота
    If Not WinActivate("[CLASS:Notepad]") Then
        Run("notepad.exe")
        WinWaitActive("[CLASS:Notepad]")
    EndIf

    ;Отправка ввода
    Local $nResult = _WinAPI_SendInput($INPUT_KEYBOARD, $iNumberOfInputs, $tInputArray)

    If $nResult = 0 Then
        MsgBox(0, "GetLastError", _WinAPI_GetLastErrorMessage())
    ElseIf $nResult <> $iNumberOfInputs Then
        MsgBox(0, "Warning", $nResult & " keyboard events returned (it should have been " & $iNumberOfInputs & ")")
    EndIf

EndFunc   ;==>Example

Func Example()
    #cs
INPUT input;
input.type = INPUT_MOUSE;
input.mi.mouseData = 0;
input.mi.dx = x * (65536 / GetSystemMetrics(SM_CXSCREEN)); //x being coord in pixels
input.mi.dy = y * (65536 / GetSystemMetrics(SM_CYSCREEN)); //y being coord in pixels
input.mi.dwFlags = MOUSEEVENTF_ABSOLUTE | MOUSEEVENTF_MOVE;
SendInput(1, &input, sizeof(input));
#ce

    Local $iNumberOfInputs = 1
    Local $x = 100 ; координата в пикселях
    Local $Y = 100 ; координата в пикселях

    Local $tInputArray = DllStructCreate($tagMOUSEINPUT)
    $tInputArray.mouseData = 0
    $tInputArray.dx = $x * (65536 / _WinAPI_GetSystemMetrics($SM_CXSCREEN)) ; //x – координата в пикселях
    $tInputArray.dy = $y * (65536 / _WinAPI_GetSystemMetrics($SM_CYSCREEN)) ; //y – координата в пикселях
    $tInputArray.dwFlags = $MOUSEEVENTF_ABSOLUTE + $MOUSEEVENTF_MOVE
    Local $nResult = _WinAPI_SendInput($INPUT_MOUSE, $iNumberOfInputs, $tInputArray)

    If $nResult = 0 Then
        MsgBox(0, "GetLastError", _WinAPI_GetLastErrorMessage())
    ElseIf $nResult <> $iNumberOfInputs Then
        MsgBox(0, "Warning", $nResult & " mouse events returned (it should have been " & $iNumberOfInputs & ")")
    EndIf

EndFunc   ;==>Example
