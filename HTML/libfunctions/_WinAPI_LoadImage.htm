Справочник функций

_WinAPI_LoadImage

Загружает значок, курсор или растровое изображение.

#include <WinAPIRes.au3>
_WinAPI_LoadImage ( $hInstance, $sImage, $iType, $iXDesired, $iYDesired, $iLoad )

Параметры

$hInstance Определяет экземпляр модуля, содержащего загружаемое изображение. Для загрузки образа OEM
установите этот параметр в ноль.
$sImage Определяет загружаемое изображение. Если параметр $hInstance не равен 0 и параметр $iLoad
не включает $LR_LOADFROMFILE, то $sImage – это строка, содержащая имя ресурса изображения в
модуле $hInstance. Если $hInstance равен 0 и $LR_LOADFROMFILE не указан, то младшее слово этого
параметра должно быть идентификатором образа OEM для загрузки.
$iType Задаёт тип загружаемого изображения. Этот параметр может быть одним из следующих значений:
$IMAGE_BITMAP – загружает растровое изображение
$IMAGE_CURSOR – загружает курсор
$IMAGE_ICON – загружает значок.
$iXDesired Задаёт ширину значка или курсора в пиксельях. Если это 0
и $iLoad равен $LR_DEFAULTSIZE, функция использует значение системной метрики SM_CXICON или SM_CXCURSOR
для установки ширины. Если этот параметр равен 0 и $LR_DEFAULTSIZE не
используется, функция использует фактическую ширину ресурса.
$iYDesired Задаёт высоту значка или курсора в пиксельях. Если это 0
и $iLoad равен $LR_DEFAULTSIZE, функция использует значение системной метрики SM_CYICON или SM_CYCURSOR
для установки высоты. Если этот параметр равен 0 и $LR_DEFAULTSIZE не
используется, функция использует фактическую высоту ресурса.
$iLoad Задаёт комбинацию следующих значений:
$LR_DEFAULTCOLOR – флаг по умолчанию
$LR_CREATEDIBSECTION – если параметр $iType задаёт $IMAGE_BITMAP, функция возвращает растровое изображение DIB
вместо совместимого растрового изображения. Этот флаг полезен для загрузки растрового изображения без его преобразования
в цвета устройства отображения.
$LR_DEFAULTSIZE – использует ширину или высоту, задаваемые системными метриками для курсоров или значков, если
значения $iXDesired или $iYDesired установлены в 0. Если этот флаг не указан и $iXDesired и $iYDesired
установлены в ноль, функция использует фактический размер ресурса. Если ресурс содержит несколько изображений,
функция использует размер первого изображения.
$LR_LOADFROMFILE – загружает изображение из файла, заданного параметром $sImage. Если этот флаг не
указан, $sImage – это имя ресурса.
$LR_LOADMAP3DCOLORS – поиск в таблице цветов изображения и замена следующих оттенков серого на
соответствующие 3D-цвета:
Тёмно-серый: RGB(128,128,128) COLOR_3DSHADOW
Серый: RGB(192,192,192) COLOR_3DFACE
Светло-серый: RGB(223,223,223) COLOR_3DLIGHT
$LR_LOADTRANSPARENT – получает значение цвета первого пикселя изображения и заменяет соответствующую
запись в таблице цветов цветом окна по умолчанию. Все пиксели в изображении, использующие эту запись, становятся
цветом окна по умолчанию. Это значение применяется только к изображениям, имеющим соответствующие таблицы цветов. Если $iLoad
включает как $LR_LOADTRANSPARENT, так и $LR_LOADMAP3DCOLORS, $LR_LOADTRANSPARENT имеет приоритет.
Однако запись в таблице цветов заменяется на COLOR_3DFACE вместо COLOR_WINDOW.
$LR_MONOCHROME – загружает изображение в чёрно-белом режиме
$LR_SHARED – использует общий дескриптор изображения при загрузке изображения несколько раз. Если LR_SHARED не установлен,
второй вызов LoadImage для того же ресурса загрузит изображение снова и вернёт другой дескриптор. Не
используйте $LR_SHARED для изображений нестандартного размера, которые могут измениться после загрузки или которые загружаются
из файла.

Возвращаемое значение

Успех: Возвращает дескриптор загруженного изображения.
Ошибка: Возвращает 0; вызовите _WinAPI_GetLastError() для получения подробных сведений об ошибке.

Примечания

Для приведённых констант требуется #include <WinAPIConstants.au3>
