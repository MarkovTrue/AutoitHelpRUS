_WinAPI_CreateMutex

Создаёт или открывает именованный или безымянный объект мьютекса.

#include <WinAPIProc.au3>
_WinAPI_CreateMutex ( $sMutex [, $bInitial = True [, $tSecurity = 0]] )

Параметры

$sMutex Имя объекта мьютекса. Сравнение имён чувствительно к регистру.
$bInitial [необязательный] Получает ли вызывающий процесс начальное владение объектом мьютекса, допустимые значения:
True – вызывающий поток получает начальное владение объектом мьютекса (по умолчанию).
False – вызывающий поток не получает владение объектом мьютекса.
$tSecurity [необязательный] Структура $tagSECURITY_ATTRIBUTES, задающая дескриптор безопасности для нового мьютекса.
Если параметр равен 0 (по умолчанию), мьютекс получает дескриптор безопасности по умолчанию.

Возвращаемое значение

Успех: Возвращает дескриптор созданного объекта мьютекса.
Ошибка: Возвращает 0; вызовите _WinAPI_GetLastError() для получения подробных сведений об ошибке.

Примечания

Если мьютекс именованный и объект уже существовал до вызова функции, возвращается дескриптор существующего объекта, _WinAPI_GetLastError() возвращает ERROR_ALREADY_EXISTS (183), параметр $bInitial игнорируется, а вызывающий поток не получает владение.
Однако если у вызывающей стороны ограниченные права доступа, функция завершится с ошибкой ERROR_ACCESS_DENIED (5), и тогда следует использовать _WinAPI_OpenMutex().

Любой процесс может передать дескриптор объекта мьютекса в вызов одной из функций ожидания.
Функции ожидания одного объекта возвращают управление, когда объект переходит в сигнальное состояние.
Функции ожидания нескольких объектов можно настроить на возврат либо при сигнале любого из объектов, либо при сигнале всех сразу.
Когда функция ожидания возвращает управление, ожидающий поток продолжает выполнение.

Два или более процессов могут вызвать _WinAPI_CreateMutex() для создания одного и того же именованного мьютекса.
Первый процесс действительно создаёт мьютекс, а следующие процессы с достаточными правами доступа просто открывают дескриптор существующего мьютекса.
Это позволяет нескольким процессам получить дескрипторы одного мьютекса и избавляет от необходимости следить, чтобы процесс-создатель запускался первым.
При таком подходе задавайте параметр $bInitial равным False, иначе трудно определить, какой процесс получил начальное владение.

Чтобы закрыть дескриптор, используйте _WinAPI_CloseHandle().
Система закрывает дескриптор автоматически при завершении процесса.
Объект мьютекса уничтожается после закрытия его последнего дескриптора.
