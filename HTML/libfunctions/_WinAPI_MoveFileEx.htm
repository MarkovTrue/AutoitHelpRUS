Справочник функций

_WinAPI_MoveFileEx

Перемещает файл или каталог, уведомляя приложение о его прогрессе через функцию обратного вызова.

#include <WinAPIFiles.au3>
_WinAPI_MoveFileEx ( $sExistingFile, $sNewFile [, $iFlags = 0 [, $pProgressProc = 0 [, $pData = 0]]] )

Параметры

$sExistingFile Имя существующего файла или каталога на локальном компьютере.

Если $iFlags указывает $MOVE_FILE_DELAY_UNTIL_REBOOT, файл не может существовать на удалённой общей папке, так как отложенные операции выполняются до того, как сеть станет доступной.
$sNewFile Новое имя файла или каталога на локальном компьютере.

При перемещении файла $sNewFile может находиться на другой файловой системе или томе.
Если $sNewFile находится на другом диске, вы должны установить флаг $MOVE_FILE_COPY_ALLOWED в параметре $iFlags.

При перемещении каталога $sExistingFile и $sNewFile должны находиться на одном диске.

Если $iFlags указывает $MOVE_FILE_DELAY_UNTIL_REBOOT, то $sNewFile должен быть 0, _WinAPI_MoveFileEx() регистрирует $sExistingFile для удаления при перезагрузке системы.
Если $sExistingFile относится к каталогу, система удаляет каталог при перезагрузке только если каталог пуст.
$iFlags [необязательный] Параметры перемещения. Этот параметр может быть одним или несколькими из следующих значений:
$MOVE_FILE_COPY_ALLOWED (0x0002) не может использоваться с $MOVE_FILE_DELAY_UNTIL_REBOOT
$MOVE_FILE_CREATE_HARDLINK (0x0010)
$MOVE_FILE_DELAY_UNTIL_REBOOT (0x0004) не может использоваться с $MOVE_FILE_COPY_ALLOWED
$MOVE_FILE_FAIL_IF_NOT_TRACKABLE (0x0020)
$MOVE_FILE_REPLACE_EXISTING (0x0001)
$MOVE_FILE_WRITE_THROUGH (0x0008).
$pProgressProc [необязательный] Адрес функции обратного вызова, которая вызывается каждый раз когда другая часть файла была перемещена.
(см. MSDN для дополнительной информации)
$pData [необязательный] Указатель на аргумент, передаваемый функции обратного вызова.

Возвращаемое значение

Успех: Возвращает True.
Ошибка: Возвращает False; вызовите _WinAPI_GetLastError() для получения подробных сведений об ошибке.

Примечания

Константы $MOVE_* , $PROGRESS_* требуют #include <APIFilesConstants.au3>

Если параметр $iFlags указывает $MOVE_FILE_DELAY_UNTIL_REBOOT, _WinAPI_MoveFileEx() завершится с ошибкой, если не может получить доступ к реестру (вы можете использовать #RequireAdmin).
Функция сохраняет расположение файлов для переименования при перезагрузке в следующем значении реестра:
    "HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Session Manager", "PendingFileRenameOperations"

Функция обратного вызова, на которую ссылается $pProgressProc, должна определить функцию как:

    Func _ProgressProc($iTotalFileSize, $iTotalBytesTransferred, $iStreamSize, $iStreamBytesTransferred, $iStreamNumber, $iCallbackReason, $hSourceFile, $hDestinationFile, $pData)
        ; ...
    EndFunc

Пример

Пример 1

#include <Misc.au3>
#include <WinAPIError.au3>
#include <WinAPIFiles.au3>

Opt('TrayAutoPause', 0)

Local $hProgressProc = DllCallbackRegister('_ProgressProc', 'bool', 'uint64;uint64;uint64;uint64;dword;dword;handle;handle;ptr')

FileDelete(@TempDir & '\Test*.tmp')

ProgressOn('_WinAPI_MoveFileEx()', 'Bigfile Creation...', '')
Local $sFile = @TempDir & '\Test.tmp'
Local $hFile = FileOpen($sFile, 2)
For $i = 1 To 1000000
    FileWriteLine($hFile, " ")
Next
FileClose($hFile)

ProgressOn('_WinAPI_MoveFileEx()', 'Moving...', '0%')
Sleep(500) ; чтобы показать изменение как перемещение в пределах одного устройства – просто переименовать

If Not _WinAPI_MoveFileEx($sFile, @TempDir & '\Test1.tmp', $MOVE_FILE_COPY_ALLOWED, DllCallbackGetPtr($hProgressProc)) Then
    _WinAPI_ShowLastError('Error moving ' & $sFile)
EndIf

DllCallbackFree($hProgressProc)

ProgressOff()

FileDelete(@TempDir & '\Test*.tmp')

Func _ProgressProc($iTotalFileSize, $iTotalBytesTransferred, $iStreamSize, $iStreamBytesTransferred, $iStreamNumber, $iCallbackReason, $hSourceFile, $hDestinationFile, $pData)
    #forceref $iStreamSize, $iStreamBytesTransferred, $iStreamNumber, $iCallbackReason, $hSourceFile, $hDestinationFile, $pData

    Local $iPercent = Round($iTotalBytesTransferred / $iTotalFileSize * 100)
    If $iPercent = 100 Then
        ProgressSet($iPercent, $iPercent & "%", 'Complete')
        Sleep(2000)
    Else
        ProgressSet($iPercent, $iPercent & "%")
    EndIf
    Sleep(100)

    If _IsPressed('1B') Then
        Return $PROGRESS_CANCEL
    Else
        Return $PROGRESS_CONTINUE
    EndIf
EndFunc   ;==>_ProgressProc

    FileDelete(@TempDir & '\Test*.tmp')

    ProgressOn('_WinAPI_MoveFileEx()', 'Bigfile Creation...', '')

    Local $hFile = FileOpen($sFile, 2)
    For $i = 1 To 1000000
        FileWriteLine($hFile, " ")
    Next
    FileClose($hFile)

    ProgressOn('_WinAPI_MoveFileEx()', 'Moving...', '0%')
    Sleep(500) ; чтобы показать изменение как перемещение в пределах одного устройства – просто переименовать

;~ If Not _WinAPI_MoveFileEx($sFile, @TempDir & '\Test1.tmp', $MOVE_FILE_DELAY_UNTIL_REBOOT, DllCallbackGetPtr($hProgressProc)) Then ; does not rename !!!

    ; удаление файла при следующей перезагрузке
