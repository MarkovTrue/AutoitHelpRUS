_WinAPI_ReOpenFile

Открывает заново указанный объект файловой системы с другими правами доступа, режимом совместного доступа и флагами.

#include <WinAPIFiles.au3>
_WinAPI_ReOpenFile ( $hFile, $iAccess, $iShare [, $iFlags = 0] )

Параметры

$hFile Дескриптор объекта для открытия заново. Объект должен быть создан функцией _WinAPI_CreateFileEx().
$iAccess Требуемый доступ к объекту. Если этот параметр равен 0, приложение может запрашивать атрибуты устройства
без доступа к устройству.
$GENERIC_READ
$GENERIC_WRITE
(подробнее см. в MSDN)
$iShare Режим совместного доступа к объекту. Если этот параметр равен 0, объект не может быть общим и не может быть
открыт заново, пока дескриптор не будет закрыт.
$FILE_SHARE_READ
$FILE_SHARE_WRITE
$FILE_SHARE_DELETE
$iFlags [необязательный] Атрибуты файла или устройства и флаги. Этот параметр может быть одним или несколькими из следующих значений.
$FILE_FLAG_BACKUP_SEMANTICS
$FILE_FLAG_DELETE_ON_CLOSE
$FILE_FLAG_NO_BUFFERING
$FILE_FLAG_OPEN_NO_RECALL
$FILE_FLAG_OPEN_REPARSE_POINT
$FILE_FLAG_OVERLAPPED
$FILE_FLAG_POSIX_SEMANTICS
$FILE_FLAG_RANDOM_ACCESS
$FILE_FLAG_SEQUENTIAL_SCAN
$FILE_FLAG_WRITE_THROUGH

$SECURITY_ANONYMOUS
$SECURITY_CONTEXT_TRACKING
$SECURITY_DELEGATION
$SECURITY_EFFECTIVE_ONLY
$SECURITY_IDENTIFICATION
$SECURITY_IMPERSONATION

Возвращаемое значение

Успех: Возвращает дескриптор указанного файла.
Ошибка: Возвращает 0; вызовите _WinAPI_GetLastError() для получения подробных сведений об ошибке.

Примечания

Параметр $iFlags не может содержать флаги атрибутов файла ($FILE_ATTRIBUTE_*). Они могут быть
указаны только при создании файла.

Эта функция требует Windows Vista или позднее.
