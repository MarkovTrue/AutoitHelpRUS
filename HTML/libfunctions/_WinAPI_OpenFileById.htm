Справочник функций

_WinAPI_OpenFileById

Открывает файл, соответствующий указанному идентификатору объекта.

#include <WinAPIFiles.au3>
_WinAPI_OpenFileById ( $hFile, $vID [, $iAccess = 0 [, $iShare = 0 [, $iFlags = 0]]] )

Параметры

$hFile Путь или дескриптор любого файла на томе или общей папке, где хранится файл для открытия.
$vID Идентификатор файла (FileID), структура $tagGUID (ObjectID) или строковое представление GUID в форме
"{xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx}", который идентифицирует файл для открытия.
$iAccess [необязательный] Доступ к объекту. Доступ может быть чтение, запись или оба одновременно.
Если этот параметр равен 0 (по умолчанию), приложение может запрашивать атрибуты файла и устройства без доступа к устройству.
$GENERIC_READ
$GENERIC_WRITE
(см. MSDN для дополнительной информации)
$iShare [необязательный] Режим совместного доступа к объекту, который может быть чтение, запись, оба одновременно или ничего. Если этот параметр равен 0 (по умолчанию) и функция успешна,
объект не может быть общим и не может быть открыт снова, пока дескриптор не будет закрыт.
$FILE_SHARE_DELETE
$FILE_SHARE_READ
$FILE_SHARE_WRITE
$iFlags [необязательный] Флаги файла. Когда _WinAPI_OpenFileById() открывает файл, она объединяет флаги файла с существующими
атрибутами файла и игнорирует любые предоставленные атрибуты файла.
Этот параметр может включать любую комбинацию следующих значений:
$FILE_FLAG_BACKUP_SEMANTICS
$FILE_FLAG_NO_BUFFERING
$FILE_FLAG_OPEN_NO_RECALL
$FILE_FLAG_OPEN_REPARSE_POINT
$FILE_FLAG_OVERLAPPED
$FILE_FLAG_RANDOM_ACCESS
$FILE_FLAG_SEQUENTIAL_SCAN
$FILE_FLAG_WRITE_THROUGH

Возвращаемое значение

Успех: Возвращает дескриптор указанного файла.
Ошибка: Возвращает 0 и устанавливает @error ≠ 0; вызовите _WinAPI_GetLastError() для получения подробных сведений об ошибке.

Примечания

Когда приложение закончит использовать дескриптор объекта, возвращённый этой функцией, используйте функцию _WinAPI_CloseHandle()
для закрытия дескриптора. Это не только освобождает системные ресурсы, но может иметь более широкое влияние на такие вещи
как совместное использование файла или устройства и фиксирование данных на диск.

Эта функция требует Windows Vista или более позднюю версию.

См. также

_WinAPI_CloseHandle

Пример

#include <FileConstants.au3>
#include <MsgBoxConstants.au3>
#include <WinAPIConv.au3>
#include <WinAPIFiles.au3>
#include <WinAPIHObj.au3>
#include <WinAPIShPath.au3>
#include <WinAPISys.au3>

If Number(_WinAPI_GetVersion()) < 6.0 Then
    MsgBox(($MB_ICONERROR + $MB_SYSTEMMODAL), 'Error', 'Require Windows Vista or later.')
    Exit
EndIf

; Создать временный файл
Local $sFile = _WinAPI_GetTempFileName(@TempDir)

; Создать уникальный object ID для файла
Local $tGUID = _WinAPI_CreateObjectID($sFile)

ConsoleWrite('GUID: ' & _WinAPI_StringFromGUID($tGUID) & @CRLF)

; Открыть файл по object ID и получить полный путь
Local $hFile = _WinAPI_OpenFileById(_WinAPI_PathStripToRoot(@TempDir), $tGUID, 0, BitOR($FILE_SHARE_DELETE, $FILE_SHARE_READ, $FILE_SHARE_WRITE))
$sFile = _WinAPI_GetFinalPathNameByHandleEx($hFile)
_WinAPI_CloseHandle($hFile)

ConsoleWrite('Path: ' & StringRegExpReplace($sFile, '\\+.\\', '') & @CRLF)

; Удалить файл
FileDelete($sFile)
