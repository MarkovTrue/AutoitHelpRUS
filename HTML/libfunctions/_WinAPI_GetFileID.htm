Справочник функций

_WinAPI_GetFileID

Получает 8-байтовый номер ссылки на файл файловой системы.

#include <WinAPIFiles.au3>
_WinAPI_GetFileID ( $hFile )

Параметры

$hFile Дескриптор файла или каталога, номер ссылки которого требуется получить.

Возвращаемое значение

Успех: 8-байтовый номер ссылки на файл.
Ошибка: Возвращает 0 и устанавливает @error ≠ 0.
@extended: Может содержать код ошибки NTSTATUS.

Примечания

Номер ссылки на файл назначается файловой системой и зависит от конкретной файловой системы.
(Обратите внимание, что это не то же самое, что 16-байтовый "идентификатор объекта файла", добавленный в NTFS.)

Номера ссылок на файлы, также называемые идентификаторами файлов, гарантированно уникальны только в пределах статической файловой системы.
Они не гарантированно уникальны во времени, так как файловые системы могут их переиспользовать.
Также они не гарантированно остаются постоянными.
Например, файловая система FAT генерирует номер ссылки на файл из смещения байта записи каталога файла (DIRENT) на диске.
Дефрагментация может изменить это смещение байта.
Таким образом, номер ссылки на файл FAT может изменяться со временем.

Пример

#include <WinAPIFiles.au3>
#include <WinAPIHObj.au3>

Local $hFile = _WinAPI_CreateFile(@ScriptFullPath, 2, 0, 6)

ConsoleWrite('Handle: ' & $hFile & @CRLF)
ConsoleWrite('ID: ' & _WinAPI_GetFileID($hFile) & @CRLF)

_WinAPI_CloseHandle($hFile)
