Справочник функций

_WinAPI_ChangeWindowMessageFilterEx

Изменяет фильтр сообщений изоляции привилегий пользовательского интерфейса (UIPI) для указанного окна.

#include <WinAPISysWin.au3>
_WinAPI_ChangeWindowMessageFilterEx ( $hWnd, $iMsg, $iAction )

Параметры

$hWnd Дескриптор окна.
$iMsg Сообщение, которое фильтр сообщений пропускает или блокирует.
$iAction Действие, которое должно быть выполнено. Этот параметр может быть одним из следующих значений:
$MSGFLT_ALLOW
$MSGFLT_DISALLOW

Windows 7 или выше
$MSGFLT_RESET

Возвращаемое значение

Успех: Возвращает 1 и устанавливает флаг @extended на 0 или одно из следующих значений (только в Windows 7 или выше).
$MSGFLTINFO_ALLOWED_HIGHER
$MSGFLTINFO_ALREADYALLOWED_FORWND
$MSGFLTINFO_ALREADYDISALLOWED_FORWND
$MSGFLTINFO_NONE
(подробнее см. в MSDN)
Ошибка: Возвращает 0 и устанавливает @error ≠ 0, вызовите _WinAPI_GetLastError() для получения расширенной информации об ошибке.

Примечания

UIPI – это функция безопасности, которая предотвращает получение сообщений от отправителя с более низким уровнем целостности.
Вы можете использовать эту функцию для разрешения доставки определённых сообщений в окно, даже если сообщение происходит из процесса с более низким уровнем целостности.

Определённые сообщения, значение которых меньше WM_USER, должны проходить через фильтр, независимо от параметров фильтра. Попытка использования этой функции для разрешения или блокирования таких сообщений не будет иметь эффекта.

Эта функция требует Windows Vista или выше.
