_WinAPI_SetDIBits

Устанавливает пиксели в совместимом растре, используя данные о цвете, найденные в DIB.

#include <WinAPIGdi.au3>
_WinAPI_SetDIBits ( $hDC, $hBitmap, $iStartScan, $iScanLines, $pBits, $tBMI [, $iColorUse = 0] )

Параметры

$hDC Дескриптор контекста устройства.
$hBitmap Дескриптор совместимого растра (DDB) для изменения с использованием данных о цвете из DIB.
$iStartScan Начальную строку сканирования для независимого от устройства цвета в массиве, на который указывает параметр $pBits.
$iScanLines Определяет количество строк сканирования в массиве, содержащем независимые от устройства данные о цвете.
$pBits Указатель на данные о цвете DIB, хранящиеся как массив байтов.
Формат значений растра зависит от члена biBitCount структуры $tagBITMAPINFO, на которую указывает параметр $pBMI.
$tBMI Структура $tagBITMAPINFO или указатель на неё, содержащая информацию о DIB.
$iColorUse [необязательный] Определяет, был ли предоставлен член iColors структуры $tagBITMAPINFO и, если да,
содержит ли iColors явные значения красного, зелёного, синего (RGB) или индексы палитры.
Параметр $iColorUse должен быть одним из следующих значений:
0 – таблица цветов предоставлена и содержит литеральные значения RGB
1 – таблица цветов состоит из массива 16-битных индексов в логической палитре $hDC

Возвращаемое значение

Успех: Возвращает True.
Ошибка: Возвращает False.

Примечания

Контекст устройства, определённый параметром $hDC, используется только если $iColorUse установлен в 1, иначе он игнорируется.
Растр, определённый параметром hBmp, не должен быть выбран в контекст устройства при вызове этой функции.
Строки сканирования должны быть выровнены на DWORD, кроме RLE-сжатых растров.
Начало отсчёта для DIB снизу вверх – левый нижний угол растра; начало отсчёта для DIB сверху вниз – левый верхний угол растра.
