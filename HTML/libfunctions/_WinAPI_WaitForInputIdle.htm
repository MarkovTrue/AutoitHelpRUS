_WinAPI_WaitForInputIdle

Ожидает, пока процесс ждёт пользовательского ввода без ожидающего ввода или времени ожидания.

#include <WinAPIProc.au3>
_WinAPI_WaitForInputIdle ( $hProcess [, $iTimeout = -1] )

Параметры

$hProcess Дескриптор процесса. Если этот процесс является консольным приложением или не имеет очереди сообщений,
эта функция немедленно возвращается.
$iTimeOut [необязательный] Интервал времени ожидания в миллисекундах. Если установлено значение -1, функция не возвращается, пока
процесс не будет неактивным.

Возвращаемое значение

Успех: Возвращает True.
Ошибка: Возвращает False.

Примечания

Эта функция заставляет поток приостановить выполнение, пока указанный процесс завершит инициализацию
и ждёт пользовательского ввода без ожидающего ввода. Это может быть полезно для синхронизации родительского процесса и
новосозданного дочернего процесса. Когда родительский процесс создаёт дочерний процесс, функция CreateProcess
возвращается без ожидания завершения инициализации дочернего процесса. Перед попыткой связи с
дочерним процессом родительский процесс может использовать эту функцию для определения того, когда завершится инициализация дочернего процесса. Эта функция может быть использована в любое время, не только во время запуска приложения.
