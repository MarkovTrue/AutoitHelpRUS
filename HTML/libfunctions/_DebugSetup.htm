Справочник функций

_DebugSetup

Настраивает сеанс отладки с заданным типом вывода.

#include <Debug.au3>
_DebugSetup ( [$sTitle = Default [, $bBugReportInfos = False [, $vReportType = 1 [, $vLogFile = "" [, $bTimeStamp = False]]]]] )

Параметры

$sTitle [необязательный] Заголовок окна отчёта. По умолчанию "AutoIt Debug Report".
$bBugReportInfos [необязательный] Отображать сведения BugReport. По умолчанию False.
$vReportType [необязательный] 1 – окно журнала отчётов (по умолчанию)
2 – ConsoleWrite
3 – MsgBox
4 – FileWrite, имя файла задаётся параметром $vLogFile
5 – окно Блокнота (см. раздел Примечания для Windows 11)
6 – окно журнала с таймаутом (см. раздел Примечания для Windows 11)
строка – имя пользовательской функции вывода.
$vLogFile [необязательный] Имя файла при $vReportType = 4
Таймаут в секундах при $vReportType = 6 (по умолчанию 10).
$bTimeStamp [необязательный] True, если перед каждым сообщением нужно добавлять метку времени "YYYY/MM/DD HH:MM:SS".

Возвращаемое значение

Успех: Возвращает тип вывода отчёта.
Ошибка: Устанавливает @error ≠ 0.
@error: 1 – функция уже была вызвана
2 – недопустимый тип вывода
3 – недопустимая функция обратного вызова

Примечания

Если зарегистрирована пользовательская функция вывода, при завершении AutoIt она вызывается без параметров.

Одновременно в запущенном скрипте может быть только один сеанс отладки.
Несколько скриптов можно отлаживать в одном сеансе: для этого при вызове _DebugSetup() укажите одинаковый $sTitle. Весь вывод _DebugOut() будет поступать в один сеанс.

Если закомментировать вызов _DebugSetup(), все вызовы _Debug...() перестанут работать. Но перед выпуском финальной версии скрипта, возможно, захотите убрать все вызовы _Debug...-функций.

При использовании типа "Report Log Window" скрипт завершится только при закрытии окна отчёта.

При использовании типа "Report Notepad Window" в Windows 11 окно Блокнота нельзя закрыть без запроса на его регистрацию.
Поэтому содержимое окна копируется в буфер обмена, и перед завершением скрипта оно показывается в MsgBox.
Если скрипт запущен от имени администратора, используется старый Блокнот, как в Windows 10 (см. пример 2).

Пример

Пример 1

#cs ----------------------------------------------------------------------------

AutoIt Version: 3.2.8.1
Author: David Nuttall

Функция скрипта:
Базовый скрипт для демонстрации функций отладки.

#ce ----------------------------------------------------------------------------

#include <Debug.au3>

_DebugSetup("Check Excel", True) ; Начинаем отображение среды отладки.
For $i = 1 To 4
    WinActivate("Microsoft Excel")
    ; взаимодействие с Excel
    Send("{Down}")
    _DebugOut("Moved Mouse Down") ; заставляет debug-окно Notepad взять управление
Next

Sample 2 script using Report Notepad Window (5)

; == Пример 2: скрипт с использованием Report Notepad Window (5)

; позволит запускать со старым поведением Notepad в Windows 11
#RequireAdmin

#include <Debug.au3>

_DebugSetup("Check Excel", True, 5) ; Окно Report Notepad
For $i = 1 To 4
    WinActivate("Microsoft Excel")
    ; взаимодействие с Excel
    Send("{Down}")
    _DebugOut("Moved Mouse Down") ; заставляет debug-окно Notepad взять управление
Next
