Справочник функций

_WinAPI_EnumProcessModules

Возвращает дескриптор и имя каждого модуля указанного процесса.

#include <WinAPIProc.au3>
_WinAPI_EnumProcessModules ( [$iPID = 0 [, $iFlag = $LIST_MODULES_DEFAULT]] )

Параметры

$iPID [необязательный] PID процесса. По умолчанию (0) – текущий процесс.
$iFlag [необязательный] Критерий фильтрации. Параметр действует только в Windows Vista и новее и может принимать одно из значений:
$LIST_MODULES_32BIT
$LIST_MODULES_64BIT
$LIST_MODULES_ALL
$LIST_MODULES_DEFAULT (по умолчанию)
Константы определены в APIProcConstants.au3.

Возвращаемое значение

Успех: Возвращает двумерный массив со следующей информацией:
[0][0] – количество строк массива (n)
[0][1] – не используется
[n][0] – дескриптор модуля.
[n][1] – путь к файлу с модулем.
Ошибка: Устанавливает @error ≠ 0.

Примечания

Функция не возвращает дескрипторы модулей, загруженных с флагом $LOAD_LIBRARY_AS_DATAFILE.
Подробности – см. _WinAPI_LoadLibraryEx().

Не вызывайте _WinAPI_CloseHandle() для дескрипторов, возвращённых этой функцией.
Информация берётся из снимка, поэтому ресурсы освобождать не нужно.

Если _WinAPI_EnumProcessModules() вызывается из 32-битного приложения под WOW64, она может перечислять только модули 32-битных процессов.
Для 64-битного процесса функция завершается ошибкой ERROR_PARTIAL_COPY (299).

Дополнительно

Искать EnumProcessModulesEx в библиотеке MSDN

Пример

#include <Array.au3>
#include <MsgBoxConstants.au3>
#include <WinAPIProc.au3>
#include <WinAPISys.au3>

If (Number(_WinAPI_GetVersion()) < 6.0) And (@AutoItX64) Then
    MsgBox(($MB_ICONERROR + $MB_SYSTEMMODAL), 'Error', 'This example works from a 64-bit system only in Windows Vista or later.')
    Exit
EndIf

Local $aData = _WinAPI_EnumProcessModules()

_ArrayDisplay($aData, '_WinAPI_EnumProcessModules')
