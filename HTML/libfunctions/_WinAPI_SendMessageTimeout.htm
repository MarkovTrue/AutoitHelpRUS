Справочник функций

_WinAPI_SendMessageTimeout

Отправляет указанное сообщение одному или нескольким окнам.

#include <WinAPISysWin.au3>
_WinAPI_SendMessageTimeout ( $hWnd, $iMsg [, $wParam = 0 [, $lParam = 0 [, $iTimeout = 1000 [, $iFlags = 0]]]] )

Параметры

$hWnd Дескриптор окна.
$iMsg Отправляемое сообщение.
$wParam [необязательный] Дополнительные данные, специфичные для сообщения.
$lParam [необязательный] Дополнительные данные, специфичные для сообщения.
$iTimeout [необязательный] Длительность в миллисекундах периода тайм-аута. Если сообщение является широковещательным сообщением, каждое
окно может использовать полный период тайм-аута. По умолчанию 1000.
$iFlags [необязательный] Флаги, определяющие способ отправки сообщения.
Этот параметр может быть одним или несколькими из следующих значений.
$SMTO_BLOCK
$SMTO_NORMAL
$SMTO_ABORTIFHUNG
$SMTO_NOTIMEOUTIFNOTHUNG
$SMTO_ERRORONEXIT

Возвращаемое значение

Успех: Возвращает результат обработки сообщения зависит от отправленного сообщения.
Ошибка: Устанавливает @error ≠ 0.
@extended: Содержит _WinAPI_GetLastError().

Примечания

Эта функция не предоставляет информацию об истечении времени ожидания отдельных окон при использовании $HWND_BROADCAST.

Если время истечёт, функция завершится неудачей. Если @extended равен $ERROR_TIMEOUT (1460), функция истекла по времени.
Эта функция считает поток не отвечающим, если он не отвечает в течение пяти секунд.

Система выполняет маршаллинг только для системных сообщений (находящихся в диапазоне 0–(WM_USER-1)). Для отправки других сообщений
(тех, что >= WM_USER) другому процессу необходимо выполнить пользовательский маршаллинг.
