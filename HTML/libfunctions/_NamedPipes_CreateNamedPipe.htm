_NamedPipes_CreateNamedPipe

Создаёт экземпляр именованного канала.

#include <NamedPipes.au3>
_NamedPipes_CreateNamedPipe ( $sName [, $iAccess = 2 [, $iFlags = 2 [, $iACL = 0 [, $iType = 1 [, $iRead = 1 [, $iWait = 0 [, $iMaxInst = 25 [, $iOutBufSize = 4096 [, $iInpBufSize = 4096 [, $iDefaultTimeout = 5000 [, $tSecurity = 0]]]]]]]]]]] )

Параметры

$sName Имя канала со следующим форматом: \\.\pipe\pipename.
Часть имени канала может включать любой символ, кроме обратной косой черты, включая цифры и специальные символы.
Строка имени канала может быть длиной до 256 символов.
Имена каналов не чувствительны к регистру.
$iAccess [необязательный] Режим доступа канала. Должен быть одним из следующих:
0 – поток данных в канале идёт только от клиента к серверу (входящий)
1 – поток данных в канале идёт только от сервера к клиенту (исходящий)
2 – канал является двусторонним (дуплексным)
$iFlags [необязательный] Флаги канала. Может быть любой комбинацией следующих:
1 – если вы попытаетесь создать несколько экземпляров канала с этим флагом, создание первого экземпляра будет успешным, но создание следующего экземпляра не удастся.
2 – режим перекрытия включен. Если этот режим включён, функции, выполняющие операции чтения, записи и подключения, которые могут занимать значительное время для завершения, могут вернуться немедленно.
4 – режим сквозной записи включен. Этот режим влияет только на операции записи на каналах байтового типа и только когда клиент и сервер находятся на разных компьютерах.
$iACL [необязательный] Флаги безопасности ACL. Может быть любой комбинацией следующих:
1 – вызывающему будет предоставлен доступ на запись к дискреционному ACL именованного канала
2 – вызывающему будет предоставлен доступ на запись к владельцу именованного канала
4 – вызывающему будет предоставлен доступ на запись к ACL безопасности именованного канала
$iType [необязательный] Режим типа канала. Должен быть одним из следующих:
0 – данные записываются в канал как поток байтов
1 – данные записываются в канал как поток сообщений
$iRead [необязательный] Режим чтения канала. Должен быть одним из следующих:
0 – данные читаются из канала как поток байтов
1 – данные читаются из канала как поток сообщений
$iWait [необязательный] Режим ожидания канала. Должен быть одним из следующих:
0 – режим блокировки включен. Когда дескриптор канала указан в _WinAPI_ReadFile(), _WinAPI_WriteFile() или _NamedPipes_ConnectNamedPipe(), операция не завершается до тех пор, пока нет данных для чтения, все данные записаны или подключен клиент.
1 – режим без блокировки включен. _WinAPI_ReadFile(), _WinAPI_WriteFile() и _NamedPipes_ConnectNamedPipe() всегда возвращают значение немедленно.
$iMaxInst [необязательный] Максимальное количество экземпляров, которые можно создать для этого канала
$iOutBufSize [необязательный] Количество байтов для резервирования для буфера вывода
$iInpBufSize [необязательный] Количество байтов для резервирования для буфера ввода
$iDefaultTimeOut [необязательный] Значение тайм-аута по умолчанию в миллисекундах
$tSecurity [необязательный] структура $tagSECURITY_ATTRIBUTES, которая задаёт дескриптор безопасности для нового именованного канала и определяет, могут ли дочерние процессы наследовать возвращённый дескриптор.
Если $tSecurity равен 0, именованный канал получает дескриптор безопасности по умолчанию, и дескриптор не может быть унаследован.
ACL в дескрипторе безопасности по умолчанию для именованного канала предоставляют полный контроль учётной записи LocalSystem, администраторам и создателю-владельцу.
Они также предоставляют доступ на чтение членам группы Everyone и анонимной учётной записи.

Возвращаемое значение

Успех: Возвращает дескриптор конца сервера экземпляра именованного канала.
Ошибка: Возвращает -1.
