Справочник функций

_Singleton

Обеспечивает парадигму проектирования, при которой может запускаться только один экземпляр скрипта.

#include <Misc.au3>
_Singleton ( $sOccurrenceName [, $iFlag = 0] )

Параметры

$sOccurrenceName Строка для идентификации экземпляра скрипта. Эта строка не может содержать символ \ если только вы не размещаете объект в пространстве имён (см. раздел Примечания).
$iFlag [необязательный] Параметры поведения.
0 – Выход из скрипта с кодом выхода -1, если другой экземпляр уже существует.
1 – Возврат из функции без завершения скрипта.
2 – Разрешить доступ к объекту кому угодно в системе. Полезно при указании объекта "Global\" в многопользовательской среде.

Возвращаемое значение

Успех: Возвращает дескриптор объекта, используемого для синхронизации (мьютекс).
Ошибка: Возвращает 0.

Примечания

Вы можете разместить объект в пространстве имён, добавив префикс к имени объекта либо "Global\" либо "Local\". Объекты "Global\" в сочетании с флагом 2 полезны в многопользовательских средах.

Пример

#include <Misc.au3>
#include <MsgBoxConstants.au3>

If _Singleton("test", 1) = 0 Then
    MsgBox($MB_SYSTEMMODAL, "Warning", "An occurrence of test is already running")
    Exit
EndIf
MsgBox($MB_SYSTEMMODAL, "OK", "the first occurrence of test is running")
