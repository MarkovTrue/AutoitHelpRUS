Справочник ключевых слов

$tagOPENFILENAME

Содержит сведения, используемые функциями GetOpenFileName и GetSaveFileName для инициализации диалоговых окон "Открыть" и "Сохранить как".

Global Const $tagOPENFILENAME = "dword StructSize;hwnd hwndOwner;hwnd hInstance;ptr lpstrFilter;ptr lpstrCustomFilter;" & _
"dword nMaxCustFilter;dword nFilterIndex;ptr lpstrFile;dword nMaxFile;ptr lpstrFileTitle;int nMaxFileTitle;" & _
"ptr lpstrInitialDir;ptr lpstrTitle;dword Flags;short nFileOffset;short nFileExtension;ptr lpstrDefExt;ptr lCustData;" & _
"ptr lpfnHook;ptr lpTemplateName;ptr pvReserved;dword dwReserved;dword FlagsEx"

Параметры

StructSize Размер структуры в байтах.
hwndOwner Дескриптор окна-владельца диалогового окна. Может быть любым корректным дескриптором окна или NULL, если окна нет.
hInstance Если установлен флаг $OFN_ENABLETEMPLATEHANDLE в поле Flags, это дескриптор объекта памяти, содержащего шаблон диалогового окна.
Если установлен флаг $OFN_ENABLETEMPLATE, это дескриптор модуля, содержащего шаблон диалогового окна с именем, указанным в поле lpTemplateName.
Если ни один флаг не установлен, это поле игнорируется.
Если установлен флаг $OFN_EXPLORER, система использует указанный шаблон для создания диалогового окна-дочернего окна диалога Explorer-стиля по умолчанию.
Если флаг $OFN_EXPLORER не установлен, система использует шаблон для создания диалогового окна старого стиля, заменяющего диалоговое окно по умолчанию.
lpstrFilter Указатель на буфер, содержащий пары строк фильтра с завершением нулевым символом. Последняя строка в буфере должна быть завершена двумя нулевыми символами.
lpstrCustomFilter Указатель на статический буфер, содержащий пару строк фильтра с завершением нулевым символом для сохранения шаблона фильтра, выбранного пользователем.
Первая строка – это строка отображения, описывающая пользовательский фильтр, вторая строка – это шаблон фильтра, выбранный пользователем.
В первый раз, когда приложение создаёт диалоговое окно, вы указываете первую строку, которая может быть любой непустой строкой.
Когда пользователь выбирает файл, диалоговое окно копирует текущий шаблон фильтра во вторую строку.
Сохранённый шаблон фильтра может быть одним из шаблонов, указанных в буфере lpstrFilter, или это может быть шаблон фильтра, введённый пользователем.
Система использует строки для инициализации пользовательского фильтра файлов при следующем создании диалогового окна.
Если член nFilterIndex равен нулю, диалоговое окно использует пользовательский фильтр.
Если это поле NULL, диалоговое окно не сохраняет пользовательские шаблоны фильтров.
Если это поле не NULL, значение члена nMaxCustFilter должно указывать размер буфера lpstrCustomFilter в символах (TCHAR).
Для версии ANSI это количество байтов, для версии Unicode это количество символов.
nMaxCustFilter Размер буфера, определённого lpstrCustomFilter, в символах (TCHAR).
Для версии ANSI это количество байтов, для версии Unicode это количество символов.
Буфер должен быть не менее 40 символов. Это поле игнорируется, если lpstrCustomFilter равна NULL или указывает на пустую строку.
nFilterIndex Индекс текущего выбранного фильтра в элементе управления «Типы файлов».
Буфер, на который указывает lpstrFilter, содержит пары строк, определяющих фильтры.
Первая пара строк имеет значение индекса 1, вторая пара 2 и т.д.
Индекс нуль указывает на пользовательский фильтр, определённый lpstrCustomFilter.
Вы можете указать индекс при входе для указания начального описания фильтра и шаблона фильтра для диалогового окна.
Когда пользователь выбирает файл, nFilterIndex возвращает индекс текущего отображаемого фильтра.
Если nFilterIndex равен нулю и lpstrCustomFilter равна NULL, система использует первый фильтр в буфере lpstrFilter.
Если все три члена равны нулю или NULL, система не использует никакие фильтры и не показывает файлы в элементе управления списком файлов диалогового окна.
lpstrFile Указатель на буфер, содержащий имя файла, используемое для инициализации элемента управления редактирования имени файла.
Первый символ этого буфера должен быть NULL, если инициализация не требуется.
Когда функция _WinAPI_GetOpenFileName() или _WinAPI_GetSaveFileName() успешно возвращается, этот буфер содержит букву диска, путь, имя файла и расширение выбранного файла.
Если установлен флаг $OFN_ALLOWMULTISELECT и пользователь выбирает несколько файлов, буфер содержит текущий каталог, за которым следуют имена выбранных файлов.
Для диалоговых окон в стиле Explorer строки каталога и имени файла разделены NULL с дополнительным символом NULL после последнего имени файла.
Для диалоговых окон старого стиля строки разделены пробелом, и функция использует короткие имена файлов для имён файлов с пробелами.
Вы можете использовать функцию FindFirstFile для преобразования между длинными и короткими именами файлов.
Если пользователь выбирает только один файл, строка lpstrFile не имеет разделителя между путём и именем файла.
Если буфер слишком мал, функция возвращает False, и функция _WinAPI_CommDlgExtendedError() возвращает $FNERR_BUFFERTOOSMALL.
В этом случае первые два байта буфера lpstrFile содержат требуемый размер в байтах или символах.
nMaxFile Размер буфера, на который указывает lpstrFile, в символах (TCHAR).
Для версии ANSI это количество байтов, для версии Unicode это количество символов.
Буфер должен быть достаточно большим для хранения строк пути и имени файла (или строк), включая завершающий символ NULL.
Функции _WinAPI_GetOpenFileName() и _WinAPI_GetSaveFileName() возвращают False, если буфер слишком мал для размещения информации о файле.
Буфер должен быть не менее 256 символов.
lpstrFileTitle Указатель на буфер, получающий имя файла и расширение (без информации о пути) выбранного файла. Это поле может быть NULL.
nMaxFileTitle Размер буфера, на который указывает lpstrFileTitle, в символах (TCHAR).
Для версии ANSI это количество байтов, для версии Unicode это количество символов. Это поле игнорируется, если lpstrFileTitle равна NULL.
lpstrInitialDir Указатель на строку, завершённую нулевым символом, которая может указывать начальный каталог.
lpstrTitle Указатель на строку, размещаемую в строке заголовка диалогового окна. Если это поле NULL, система использует заголовок по умолчанию (т.е. «Сохранить как» или «Открыть»).
Flags Набор битовых флагов для инициализации диалогового окна. Когда диалоговое окно возвращается, оно устанавливает эти флаги для обозначения ввода пользователя. Это поле может быть комбинацией следующих флагов.
$OFN_ALLOWMULTISELECT – указывает, что списокимён файлов позволяет множественный выбор.
Если вы также установите флаг $OFN_EXPLORER, диалоговое окно использует пользовательский интерфейс стиля Explorer; в противном случае используется пользовательский интерфейс старого стиля.
$OFN_CREATEPROMPT – если пользователь указывает файл, которого не существует, этот флаг заставляет диалоговое окно запросить у пользователя разрешение на создание файла.
Если пользователь выбирает создание файла, диалоговое окно закрывается и функция возвращает указанное имя; в противном случае диалоговое окно остаётся открытым.
Если вы используете этот флаг с флагом $OFN_ALLOWMULTISELECT, диалоговое окно позволяет пользователю указать только один несуществующий файл.
$OFN_DONTADDTORECENT – предотвращает добавление системой ссылки на выбранный файл в каталог файловой системы, содержащий недавно использованные пользователем документы.
$OFN_ENABLEHOOK – включает функцию перехвата, указанную в члене lpfnHook.
$OFN_ENABLEINCLUDENOTIFY – заставляет диалоговое окно отправлять сообщения уведомления CDN_INCLUDEITEM процедуре перехвата OFNHookProc, когда пользователь открывает папку.
Диалоговое окно отправляет уведомление для каждого элемента в открытой папке.
Эти сообщения позволяют вам контролировать, какие элементы отображает диалоговое окно в списке элементов папки.
$OFN_ENABLESIZING – позволяет диалоговому окну стиля Explorer изменяться в размере с помощью мыши или клавиатуры.
По умолчанию диалоговые окна Open и Save As стиля Explorer позволяют изменяться в размере независимо от того, установлен ли этот флаг.
Этот флаг необходим только, если вы предоставляете процедуру перехвата или пользовательский шаблон. Диалоговое окно старого стиля не позволяет изменяться в размере.
$OFN_ENABLETEMPLATE – указывает, что член lpTemplateName указывает на имя ресурса шаблона диалога в модуле, определённом членом hInstance.
Если установлен флаг $OFN_EXPLORER, система использует указанный шаблон для создания диалогового окна, являющегося дочерним окном диалога стиля Explorer по умолчанию.
Если флаг $OFN_EXPLORER не установлен, система использует шаблон для создания диалогового окна старого стиля, заменяющего диалоговое окно по умолчанию.
$OFN_ENABLETEMPLATEHANDLE – указывает, что член hInstance определяет блок данных, содержащий предварительно загруженный шаблон диалогового окна.
Система игнорирует lpTemplateName, если этот флаг указан.
Если установлен флаг $OFN_EXPLORER, система использует указанный шаблон для создания диалогового окна, являющегося дочерним окном диалога стиля Explorer по умолчанию.
Если флаг $OFN_EXPLORER не установлен, система использует шаблон для создания диалогового окна старого стиля, заменяющего диалоговое окно по умолчанию.
$OFN_EXPLORER – указывает, что любые настройки, внесённые в диалоговое окно Open или Save As, используют новые методы настройки стиля Explorer.
По умолчанию диалоговые окна Open и Save As используют пользовательский интерфейс стиля Explorer независимо от того, установлен ли этот флаг.
Этот флаг необходим только, если вы предоставляете процедуру перехвата или пользовательский шаблон, либо устанавливаете флаг $OFN_ALLOWMULTISELECT.
Если вы хотите пользовательский интерфейс старого стиля, опустите флаг $OFN_EXPLORER и предоставьте замещающий шаблон старого стиля или процедуру перехвата.
Если вы хотите старый стиль, но не нуждаетесь в пользовательском шаблоне или процедуре перехвата, просто предоставьте процедуру перехвата, которая всегда возвращает False.
$OFN_EXTENSIONDIFFERENT – указывает, что пользователь ввёл расширение имени файла, отличающееся от расширения, указанного в lpstrDefExt.
Функция не использует этот флаг, если lpstrDefExt равна NULL.
$OFN_FILEMUSTEXIST – указывает, что пользователь может вводить только имена существующих файлов в поле ввода имени файла.
Если этот флаг указан и пользователь вводит неверное имя, процедура диалогового окна отображает предупреждение в окне сообщения.
Если этот флаг указан, также используется флаг $OFN_PATHMUSTEXIST. Этот флаг можно использовать в диалоговом окне Open. Его нельзя использовать с диалоговым окном Save As.
$OFN_FORCESHOWHIDDEN – заставляет отображать системные и скрытые файлы, переопределяя параметр пользователя на показывание или скрывание скрытых файлов.
Однако файл, отмеченный как системный и скрытый, не отображается.
$OFN_HIDEREADONLY – скрывает флажок только для чтения.
$OFN_LONGNAMES – для диалоговых окон старого стиля этот флаг заставляет диалоговое окно использовать длинные имена файлов.
Если этот флаг не указан или если также установлен флаг $OFN_ALLOWMULTISELECT, диалоговые окна старого стиля используют короткие имена файлов (формат 8.3) для имён файлов с пробелами.
Диалоговые окна стиля Explorer игнорируют этот флаг и всегда отображают длинные имена файлов.
$OFN_NOCHANGEDIR – восстанавливает текущий каталог в его исходное значение, если пользователь изменил каталог при поиске файлов.
Этот флаг неэффективен для GetOpenFileName.
$OFN_NODEREFERENCELINKS – указывает диалоговому окну вернуть путь и имя файла выбранного файла ярлыка (.LNK).
Если это значение не указано, диалоговое окно возвращает путь и имя файла, на который ссылается ярлык.
$OFN_NOLONGNAMES – для диалоговых окон старого стиля этот флаг заставляет диалоговое окно использовать короткие имена файлов (формат 8.3).
Диалоговые окна стиля Explorer игнорируют этот флаг и всегда отображают длинные имена файлов.
$OFN_NONETWORKBUTTON – скрывает и отключает кнопку сети.
$OFN_NOREADONLYRETURN – указывает, что возвращаемый файл не имеет выбранного флажка только для чтения и не находится в защищённом от записи каталоге.
$OFN_NOTESTFILECREATE – указывает, что файл не создаётся до закрытия диалогового окна.
Этот флаг следует указать, если приложение сохраняет файл на сетевом ресурсе с созданием без изменения.
Когда приложение указывает этот флаг, библиотека не проверяет защиту от записи, полный диск, открытую дверцу привода или защиту сети.
Приложения, использующие этот флаг, должны аккуратно выполнять операции с файлами, потому что файл не может быть открыт повторно после закрытия.
$OFN_NOVALIDATE – указывает, что диалоговые окна позволяют недействительные символы в возвращаемом имени файла.
Обычно вызывающее приложение использует процедуру перехвата, которая проверяет имя файла с помощью сообщения FILEOKSTRING.
Если текстовое поле в элементе управления редактированием пусто или содержит только пробелы, списки файлов и каталогов обновляются.
Если текстовое поле в элементе управления редактированием содержит что-либо другое, nFileOffset и nFileExtension устанавливаются на значения, созданные путём анализа текста.
Никакое расширение по умолчанию не добавляется к тексту, и текст не копируется в буфер, указанный lpstrFileTitle.
Если значение, указанное nFileOffset, меньше нуля, имя файла недействительно.
В противном случае имя файла действительно, и nFileExtension и nFileOffset можно использовать, как если бы флаг $OFN_NOVALIDATE не был указан.
$OFN_OVERWRITEPROMPT – заставляет диалоговое окно Save As создать окно сообщения, если выбранный файл уже существует.
Пользователь должен подтвердить, следует ли перезаписать файл.
$OFN_PATHMUSTEXIST – указывает, что пользователь может вводить только корректные пути и имена файлов.
Если этот флаг используется и пользователь вводит неверный путь и имя файла в поле ввода имени файла, функция диалогового окна отображает предупреждение в окне сообщения.
$OFN_READONLY – заставляет флажок только для чтения быть выбранным первоначально при создании диалогового окна.
Этот флаг указывает состояние флажка только для чтения при закрытии диалогового окна.
$OFN_SHAREAWARE – указывает, что если вызов функции OpenFile не удаётся из-за нарушения общего доступа сети, ошибка игнорируется и диалоговое окно возвращает выбранное имя файла.
Если этот флаг не установлен, диалоговое окно уведомляет процедуру перехвата при возникновении нарушения общего доступа сети для имени файла, указанного пользователем.
Если вы установите флаг $OFN_EXPLORER, диалоговое окно отправляет сообщение CDN_SHAREVIOLATION процедуре перехвата.
Если вы не установите $OFN_EXPLORER, диалоговое окно отправляет зарегистрированное сообщение SHAREVISTRING процедуре перехвата.
$OFN_SHOWHELP – заставляет диалоговое окно отображать кнопку справки.
Член hwndOwner должен указывать окно для получения зарегистрированных сообщений HELPMSGSTRING, которые диалоговое окно отправляет при нажатии пользователем кнопки справки.
Диалоговое окно стиля Explorer отправляет сообщение уведомления CDN_HELP процедуре перехвата при нажатии пользователем кнопки справки.
$OFN_USESHELLITEM – не использовать.
nFileOffset Смещение с нулевой индексацией, в символах (TCHAR), от начала пути к имени файла в строке, на которую указывает lpstrFile.
Для версии ANSI это количество байтов, для версии Unicode это количество символов.
nFileExtension Смещение с нулевой индексацией, в символах (TCHAR), от начала пути к расширению имени файла в строке, на которую указывает lpstrFile.
Для версии ANSI это количество байтов, для версии Unicode это количество символов.
lpstrDefExt Указатель на буфер, содержащий расширение по умолчанию.
lCustData Данные, определённые приложением, которые система передаёт процедуре перехвата, определённой членом lpfnHook.
lpfnHook Указатель на процедуру перехвата. Это поле игнорируется, если член Flags не включает флаг $OFN_ENABLEHOOK.
lpTemplateName Указатель на строку, завершённую нулевым символом, которая называет ресурс шаблона диалога в модуле, определённом членом hInstance.
pvReserved Зарезервировано. Должно быть установлено в NULL.
dwReserved Зарезервировано. Должно быть установлено в 0.
FlagsEx Набор битовых флагов для инициализации диалогового окна. В настоящее время это поле может быть нулевым или следующим флагом.
$OFN_EX_NOPLACESBAR – если этот флаг установлен, строка мест не отображается.
Если этот флаг не установлен, диалоговые окна стиля Explorer включают строку мест, содержащую значки часто используемых папок, таких как избранное и рабочий стол.
