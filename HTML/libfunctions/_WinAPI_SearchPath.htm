Справочник функций

_WinAPI_SearchPath

Поиск указанного файла в указанном пути.

#include <WinAPIFiles.au3>
_WinAPI_SearchPath ( $sFilePath [, $sSearchPath = ''] )

Параметры

$sFilePath Имя файла для поиска.
$sSearchPath [необязательный] Путь, в котором осуществляется поиск файла. Если этот параметр опущен или содержит пустую строку (по умолчанию), функция
ищет соответствующий файл, используя зависимый от реестра системный путь поиска (см. ниже).

Возвращаемое значение

Успех: Возвращает путь найденного файла.
Ошибка: Устанавливает @error ≠ 0; вызовите _WinAPI_GetLastError() для получения подробных сведений об ошибке.

Примечания

Если путь – пустая строка, _WinAPI_SearchPath() ищет соответствующий файл на основе текущего значения
следующего значения реестра:

    HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Session Manager\SafeProcessSearchMode

Когда это значение реестра установлено в 1, _WinAPI_SearchPath() сначала ищет в папках, указанных
в системном пути, а затем в текущей рабочей папке. Когда значение этого параметра реестра
установлено в 0, компьютер сначала ищет в текущей рабочей папке, а затем – в папках, указанных
в системном пути. Значение реестра по умолчанию для этого ключа – 0.

Не рекомендуется использовать эту функцию для поиска файла .dll, если предполагается использовать результат
в вызове функции _WinAPI_LoadLibrary(). Это может привести к поиску неправильного файла .dll, так как
порядок поиска функции _WinAPI_SearchPath() отличается от порядка поиска функции _WinAPI_LoadLibrary().
Если требуется найти и загрузить файл .dll, используйте функцию _WinAPI_LoadLibrary().

Режим поиска, используемый функцией _WinAPI_SearchPath(), также можно установить для отдельного процесса,
вызвав функцию _WinAPI_SetSearchPathMode().

См. также

_WinAPI_LoadLibrary, _WinAPI_SetSearchPathMode

Пример

#include <WinAPIFiles.au3>

Local $sFile = 'Notepad.exe'

ConsoleWrite(_WinAPI_SearchPath($sFile) & @CRLF)
ConsoleWrite(_WinAPI_SearchPath($sFile, @SystemDir) & @CRLF)
