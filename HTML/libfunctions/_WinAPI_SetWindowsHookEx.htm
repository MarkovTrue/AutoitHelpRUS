Справочник функций

_WinAPI_SetWindowsHookEx

Устанавливает определённую приложением процедуру перехватчика в цепь перехватчиков.

#include <WinAPISys.au3>
_WinAPI_SetWindowsHookEx ( $iHook, $pProc, $hDll [, $iThreadId = 0] )

Параметры

$iHook Задаёт тип устанавливаемой процедуры-хука. Параметр может принимать одно из следующих значений:
$WH_CALLWNDPROC - Устанавливает процедуру-хук, отслеживающую сообщения до их отправки системой целевой оконной процедуре
$WH_CALLWNDPROCRET - Устанавливает процедуру-хук, отслеживающую сообщения после их обработки целевой оконной процедурой
$WH_CBT - Устанавливает процедуру-хук, получающую уведомления, полезные приложениям типа CBT (computer-based training)
$WH_DEBUG - Устанавливает процедуру-хук, полезную для отладки других хуков
$WH_FOREGROUNDIDLE - Устанавливает процедуру-хук, вызываемую, когда основной поток приложения собирается перейти в состояние ожидания
$WH_GETMESSAGE - Устанавливает процедуру-хук, отслеживающую сообщения, помещаемые в очередь сообщений
$WH_JOURNALPLAYBACK - Устанавливает процедуру-хук, отправляющую сообщения, ранее записанные хуком $WH_JOURNALRECORD
$WH_JOURNALRECORD - Устанавливает процедуру-хук, записывающую входящие сообщения системной очереди сообщений
$WH_KEYBOARD - Устанавливает процедуру-хук, отслеживающую сообщения нажатий клавиш
$WH_KEYBOARD_LL - Устанавливает процедуру-хук, отслеживающую низкоуровневые события ввода с клавиатуры
$WH_MOUSE - Устанавливает процедуру-хук, отслеживающую сообщения мыши
$WH_MOUSE_LL - Устанавливает процедуру-хук, отслеживающую низкоуровневые события ввода мыши
$WH_MSGFILTER - Устанавливает процедуру-хук, отслеживающую сообщения, возникающие в результате ввода в диалоговом окне, окне сообщения, меню или полосе прокрутки
$WH_SHELL - Устанавливает процедуру-хук, получающую уведомления, полезные оболочечным приложениям
$WH_SYSMSGFILTER - Устанавливает процедуру-хук, отслеживающую сообщения, возникающие в результате ввода в диалоговом окне, окне сообщения, меню или полосе прокрутки.
$pProc Указатель на процедуру-хук. Если параметр $iThreadId равен нулю или задаёт идентификатор потока, созданного другим процессом,
параметр $pProc должен указывать на процедуру-хук, находящуюся в DLL.
Иначе $pProc может указывать на процедуру-хук в коде, связанном с текущим процессом.
$hDll Дескриптор DLL, содержащей процедуру-хук, на которую указывает параметр $pProc.
Параметр $hMod должен быть равен NULL, если параметр $iThreadId задаёт поток, созданный текущим процессом, и если процедура-хук находится в коде, связанном с текущим процессом.
$iThreadId [необязательный] Идентификатор потока, с которым должна быть связана процедура перехватчика. Если этот параметр равен нулю, процедура перехватчика связывается со всеми существующими потоками, работающими на том же рабочем столе, что и вызывающий поток.

Возвращаемое значение

Успех: Возвращает дескриптор процедуры-хука.
Ошибка: Возвращает 0; вызовите _WinAPI_GetLastError() для получения подробных сведений об ошибке.

См. также

_WinAPI_CallNextHookEx, _WinAPI_UnhookWindowsHookEx

Дополнительно

Искать SetWindowsHookEx в библиотеке MSDN

Пример

#include <MsgBoxConstants.au3>
#include <StructureConstants.au3>
#include <WinAPIConstants.au3>
#include <WinAPISys.au3>
#include <WindowsNotifsConstants.au3>

Global $g_hHook, $g_hStub_KeyProc, $g_sBuffer = ""

Example()

Func Example()
    OnAutoItExitRegister("Cleanup")

    Local $hMod

    $g_hStub_KeyProc = DllCallbackRegister("_KeyProc", "long", "int;wparam;lparam")
    $hMod = _WinAPI_GetModuleHandle(0)
    $g_hHook = _WinAPI_SetWindowsHookEx($WH_KEYBOARD_LL, DllCallbackGetPtr($g_hStub_KeyProc), $hMod)

    MsgBox($MB_SYSTEMMODAL, "", "Click OK, then in notepad type..." & _
            @CRLF & @CRLF & "Jon" & @CRLF & "AutoIt" & @CRLF & @CRLF & "Press Esc to exit script")

    Run("notepad.exe")
    WinWait("[CLASS:Notepad]")
    WinActivate("[CLASS:Notepad]")

    While 1
        Sleep(10)
    WEnd
EndFunc   ;==>Example

Func EvaluateKey($iKeycode)
    If (($iKeycode > 64) And ($iKeycode < 91)) _ ; a – z
            Or (($iKeycode > 96) And ($iKeycode < 123)) _ ; A – Z
            Or (($iKeycode > 47) And ($iKeycode < 58)) Then ; 0 - 9
        $g_sBuffer &= Chr($iKeycode)
        Switch $g_sBuffer
            Case "Jon"
                ToolTip("What can you say?")
            Case "AutoIt"
                ToolTip("AutoIt Rocks")
        EndSwitch
    ElseIf ($iKeycode > 159) And ($iKeycode < 164) Then
        Return
    ElseIf ($iKeycode = 27) Then ; клавиша Esc
        Exit
    Else
        $g_sBuffer = ""
    EndIf
EndFunc   ;==>EvaluateKey

; ===========================================================
; функция обратного вызова
; ===========================================================
Func _KeyProc($nCode, $wParam, $lParam)
    Local $tKEYHOOKS
    $tKEYHOOKS = DllStructCreate($tagKBDLLHOOKSTRUCT, $lParam)
    If $nCode < 0 Then
        Return _WinAPI_CallNextHookEx($g_hHook, $nCode, $wParam, $lParam)
    EndIf
    If $wParam = $WM_KEYDOWN Then
        EvaluateKey(DllStructGetData($tKEYHOOKS, "vkCode"))
    Else
        Local $iFlags = DllStructGetData($tKEYHOOKS, "flags")
        Switch $iFlags
            Case $LLKHF_ALTDOWN
                ConsoleWrite("$LLKHF_ALTDOWN" & @CRLF)
            Case $LLKHF_EXTENDED
                ConsoleWrite("$LLKHF_EXTENDED" & @CRLF)
            Case $LLKHF_INJECTED
                ConsoleWrite("$LLKHF_INJECTED" & @CRLF)
            Case $LLKHF_UP
                ConsoleWrite("$LLKHF_UP: scanCode - " & DllStructGetData($tKEYHOOKS, "scanCode") & @TAB & "vkCode - " & DllStructGetData($tKEYHOOKS, "vkCode") & @CRLF)
        EndSwitch
    EndIf
    Return _WinAPI_CallNextHookEx($g_hHook, $nCode, $wParam, $lParam)
EndFunc   ;==>_KeyProc

Func Cleanup()
    _WinAPI_UnhookWindowsHookEx($g_hHook)
    DllCallbackFree($g_hStub_KeyProc)
EndFunc   ;==>Cleanup
