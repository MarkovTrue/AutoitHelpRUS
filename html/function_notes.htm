Примечания к функциям

Функции в AutoIt относятся к объектам первого класса. Помимо прочего, это значит, что функцию можно присвоить переменной, передать как аргумент или вернуть из другой функции.
За исключением отдельных правил объявления, связанных с областью видимости (имена встроенных функций зарезервированы, а имена UDF можно переопределить только локально), имена функций не имеют особого статуса в языке.

#include <MsgBoxConstants.au3>

Local $vFunc = MsgBox

$vFunc($MB_OK, "Hi there!", "Lovely day today.")

 

Call(_Test)

Func _Test()
    ConsoleWrite("! Testing" & @CRLF)
EndFunc

 

Необязательные параметры

Многие функции содержат необязательные параметры, которые можно опустить. Но чтобы указать необязательный параметр, нужно задать все параметры перед ним!
Например, возьмём Run ( "filename", ["workingdir" [, flag]] ). Чтобы указать flag, вы обязаны указать workingdir.

Когда необязательный параметр нужно задать, а перед ним стоят другие необязательные параметры, для них нужно передать значение по умолчанию. Как правило, функции принимают ключевое слово Default, когда нужно использовать значение по умолчанию. Подробнее смотрите в описании соответствующего необязательного параметра.


Многие функции Win___ содержат необязательный параметр "text". Он помогает различать окна с одинаковыми заголовками.

Индикация успеха/ошибки во встроенных функциях AutoIt

Одни функции используют возвращаемое значение для индикации успеха/ошибки, другие устанавливают флаг @error. Некоторые делают и то, и другое....

Если используется метод возвращаемого значения, справка укажет ожидаемое значение для успеха/ошибки – но значение успеха обычно ненулевое, чтобы код легче читался...

If SomeUserFunc() Then ;...функция отработала.
If Not SomeUserFunc() Then ;...функция завершилась с ошибкой.



Когда в документации сказано, что возвращаемое значение = none, AutoIt всё равно возвращает значение, чтобы избежать ошибок. Обычно это 1, но полагаться на это значение не стоит.

Если используется метод флага @error, значение @error = 0 всегда означает успех. Другие значения @error указывают на проблему и описаны в справке по конкретной функции.

Local $sFileRead = FileReadLine("C:\someFile.txt")
If @error = -1 Then ; Достигнут конец файла.



Если функция использует метод флага @error, флаг нужно проверять сразу после возврата из функции, так как @error сбрасывается в 0 при входе в следующую функцию. Если флаг @error установлен, не стоит использовать или читать возвращаемое значение функции – в этом случае оно, как правило, не определено...

Индикация успеха/ошибки в пользовательских функциях

Возвращаемое значение, макросы @error и @extended обычно устанавливает автор с помощью функций Return/SetError/SetExtended в коде. Конкретные используемые значения смотрите в документации к соответствующей библиотеке.
