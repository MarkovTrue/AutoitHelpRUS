Справочник функций

_WinAPI_BroadcastSystemMessage

Отправляет сообщение указанным получателям.

#include <WinAPISysWin.au3>
_WinAPI_BroadcastSystemMessage ( $iMsg [, $wParam = 0 [, $lParam = 0 [, $iFlags = 0 [, $iRecipients = $BSM_ALLCOMPONENTS]]]] )

Параметры

$iMsg Отправляемое сообщение.
$wParam [необязательный] Дополнительные данные, специфичные для сообщения.
$lParam [необязательный] Дополнительные данные, специфичные для сообщения.
$iFlags [необязательный] Опция широковещания. Этот параметр может быть одним или несколькими из следующих значений:
$BSF_ALLOWSFW
$BSF_FLUSHDISK
$BSF_FORCEIFHUNG
$BSF_IGNORECURRENTTASK
$BSF_NOHANG
$BSF_NOTIMEOUTIFNOTHUNG
$BSF_POSTMESSAGE
$BSF_QUERY
$BSF_SENDNOTIFYMESSAGE
$iRecipients [необязательный] Получатели сообщения. Этот параметр может быть одним или несколькими из следующих значений:
$BSM_ALLCOMPONENTS (по умолчанию)
$BSM_ALLDESKTOPS
$BSM_APPLICATIONS
$BSM_INSTALLABLEDRIVERS
$BSM_NETDRIVER
$BSM_VXDS

Возвращаемое значение

Успех: Возвращает >0 или 0 (см. ниже), флаг @extended получает комбинацию значений $BSM_*, определяющих, какие получатели фактически получили сообщение.
Ошибка: (-1), вызовите _WinAPI_GetLastError() для получения расширенной информации об ошибке.

Примечания

Если параметр $iFlags равен $BSF_QUERY и по крайней мере один получатель вернул BROADCAST_QUERY_DENY для соответствующего сообщения, возвращаемое значение равно 0.

Система выполняет маршалирование только для системных сообщений (тех, которые находятся в диапазоне от 0 до (WM_USER-1)).
Для отправки других сообщений (тех >= WM_USER) в другой процесс необходимо выполнить пользовательское маршалирование.
