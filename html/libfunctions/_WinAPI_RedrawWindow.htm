Справочник функций

_WinAPI_RedrawWindow

Обновляет указанный прямоугольник или область в клиентской области окна.

#include <WinAPIGdi.au3>
_WinAPI_RedrawWindow ( $hWnd [, $tRECT = 0 [, $hRegion = 0 [, $iFlags = 5]]] )

Параметры

$hWnd Дескриптор окна.
$tRECT [необязательный] Структура $tagRECT, содержащая координаты прямоугольника обновления. Этот параметр
игнорируется, если параметр $hRegion определяет область.
$hRegion [необязательный] Определяет область обновления. Если параметры $hRegion и $tRECT равны 0, вся клиентская
область добавляется в область обновления.
$iFlags [необязательный] Задаёт флаги перерисовки. Этот параметр может быть комбинацией флагов, которые инвалидируют или проверяют окно, управляют перерисовкой и управляют затронутыми окнами:
$RDW_ERASE – заставляет окно получать сообщение WM_ERASEBKGND при перерисовке окна
$RDW_FRAME – заставляет любую часть неклиентской области окна, пересекающейся с областью обновления, получить сообщение WM_NCPAINT.
$RDW_INTERNALPAINT – заставляет сообщение WM_PAINT отправляться в окно, независимо от того, недействительна ли какая-либо часть окна.
$RDW_INVALIDATE – инвалидирует $tRECT или $hRegion. Если оба равны 0, инвалидируется всё окно.
$RDW_NOERASE – подавляет ожидающие сообщения $WM_ERASEBKGND
$RDW_NOFRAME – подавляет ожидающие сообщения $WM_NCPAINT
$RDW_NOINTERNALPAINT – подавляет ожидающие внутренние сообщения $WM_PAINT
$RDW_VALIDATE – проверяет $tRECT или $hRegion
$RDW_ERASENOW – заставляет затронутые окна получать сообщения $WM_NCPAINT и $WM_ERASEBKGND, если необходимо, перед возвратом функции
$RDW_UPDATENOW – заставляет затронутые окна получать сообщения $WM_NCPAINT, $WM_ERASEBKGND и $WM_PAINT, если необходимо, перед возвратом функции.
$RDW_ALLCHILDREN – включает дочерние окна в операцию перерисовки
$RDW_NOCHILDREN – исключает дочерние окна из операции перерисовки.

Возвращаемое значение

Успех: Возвращает True.
Ошибка: Возвращает False.

Примечания

Когда функция RedrawWindow используется для инвалидации части окна рабочего стола, окно рабочего стола не получает сообщение $WM_PAINT.
Для перерисовки рабочего стола приложение использует флаг $RDW_ERASE для создания сообщения $WM_ERASEBKGND.

Вышеуказанные константы требуют #include <APIGdiConstants.au3>

Пример

#include <MsgBoxConstants.au3>
#include <WinAPIGdi.au3>
#include <WinAPIGdiDC.au3>
#include <WinAPIHObj.au3>
#include <WinAPISysWin.au3>

If Not @Compiled Then Exit MsgBox($MB_ICONWARNING, "_WinAPI_CreatePen Example Script", _
        "When run from SciTE the script won't work properly because SciTE refreshes the screen. Hence the cross does not get fully drawn and disappears in a split second!" & _
        @CRLF & @CRLF & "Please compile the script and run the Exe.")

ShowCross(@DesktopWidth / 2, @DesktopHeight / 2, 20, 2, 0xFF, 3000)

Func ShowCross($iStart_x, $iStart_y, $iLength, $iWidth, $iColor, $iTime)
    Local $hDC, $hPen, $o_Orig

    $hDC = _WinAPI_GetWindowDC(0) ; DC всего экрана (рабочий стол)
    $hPen = _WinAPI_CreatePen($PS_SOLID, $iWidth, $iColor)
    $o_Orig = _WinAPI_SelectObject($hDC, $hPen)

    _WinAPI_DrawLine($hDC, $iStart_x - $iLength, $iStart_y, $iStart_x - 5, $iStart_y) ; по горизонтали влево
    _WinAPI_DrawLine($hDC, $iStart_x + $iLength, $iStart_y, $iStart_x + 5, $iStart_y) ; по горизонтали вправо
    _WinAPI_DrawLine($hDC, $iStart_x, $iStart_y - $iLength, $iStart_x, $iStart_y - 5) ; по вертикали вверх
    ; _WinAPI_DrawLine($hDC, $iStart_x, $iStart_y + $iLength, $iStart_x, $iStart_y + 5) ; vertical down
    _WinAPI_MoveTo($hDC, $iStart_x, $iStart_y + $iLength)
    _WinAPI_LineTo($hDC, $iStart_x, $iStart_y + 5)

    Sleep($iTime) ; показать перекрестие поверх экрана заданное число секунд

    ; обновить рабочий стол (убрать перекрестие)
    _WinAPI_RedrawWindow(_WinAPI_GetDesktopWindow(), 0, 0, $RDW_INVALIDATE + $RDW_ALLCHILDREN)

    ; освободить ресурсы
    _WinAPI_SelectObject($hDC, $o_Orig)
    _WinAPI_DeleteObject($hPen)
    _WinAPI_ReleaseDC(0, $hDC)
EndFunc   ;==>ShowCross
