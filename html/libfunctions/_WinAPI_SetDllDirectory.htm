Справочник функций

_WinAPI_SetDllDirectory

Добавляет каталог в путь поиска, используемый для поиска DLL для приложения.

#include <WinAPISys.au3>
_WinAPI_SetDllDirectory ( [$sDirPath = Default] )

Параметры

$sDirPath [необязательный] Каталог для добавления в путь поиска. Если этот параметр – пустая строка (""), вызов
удаляет текущий каталог из порядка поиска DLL по умолчанию. Если этот параметр не указан,
функция восстанавливает порядок поиска по умолчанию (по умолчанию).

Возвращаемое значение

Успех: Возвращает True.
Ошибка: Возвращает False; вызовите _WinAPI_GetLastError() для получения подробных сведений об ошибке.

Примечания

Эта функция влияет на все последующие вызовы функций _WinAPI_LoadLibrary() и _WinAPI_LoadLibraryEx().
Она также эффективно отключает безопасный режим поиска DLL, пока указанный каталог находится в пути поиска.

После вызова _WinAPI_SetDllDirectory() путь поиска DLL следующий:

1. Каталог, из которого было загружено приложение.
2. Каталог, указанный параметром $sFilePath.
3. Системный каталог.
4. 16-битный системный каталог.
5. Каталог Windows.
6. Каталоги, указанные в переменной окружения PATH.

См. также

_WinAPI_LoadLibrary, _WinAPI_LoadLibraryEx

Дополнительно

Искать SetDllDirectory в библиотеке MSDN

Пример

#include <WinAPIRes.au3>
#include <WinAPISys.au3>

If Not _WinAPI_SetDllDirectory(@ScriptDir & '\Extras') Then
    Exit
EndIf

Local $hModule = _WinAPI_LoadLibraryEx('Resources.dll', $LOAD_LIBRARY_AS_DATAFILE)

ConsoleWrite('Path: ' & _WinAPI_GetDllDirectory() & @CRLF)
ConsoleWrite('Module: ' & $hModule & @CRLF)

_WinAPI_FreeLibrary($hModule)
