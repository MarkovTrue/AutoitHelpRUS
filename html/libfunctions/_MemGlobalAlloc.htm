Справочник функций

_MemGlobalAlloc

Выделяет указанное количество байт из кучи.

#include <Memory.au3>
_MemGlobalAlloc ( $iBytes [, $iFlags = 0] )

Параметры

$iBytes Количество байт для выделения. Если этот параметр равен нулю и параметр $iFlags задаёт
$GMEM_MOVEABLE, функция возвращает дескриптор объекта памяти, помеченного как отброшенный.
$iFlags [необязательный] Атрибуты выделения памяти:
$GMEM_FIXED – Выделяет неподвижную память. Возвращаемое значение – указатель.
$GMEM_MOVEABLE – Выделяет перемещаемую память. Блоки памяти никогда не перемещаются в физической памяти, но могут перемещаться в пределах кучи по умолчанию.
Возвращаемое значение – дескриптор объекта памяти.
Для преобразования дескриптора в указатель используйте функцию _MemGlobalLock().
Это значение нельзя комбинировать с $GMEM_FIXED.
$GMEM_ZEROINIT – Инициализирует содержимое памяти нулями
$GHND – Объединяет $GMEM_MOVEABLE и $GMEM_ZEROINIT
$GPTR – Объединяет $GMEM_FIXED и $GMEM_ZEROINIT.

Возвращаемое значение

Успех: Возвращает дескриптор вновь выделенного объекта памяти.
Ошибка: Возвращает 0.

Примечания

Управление памятью Windows не предоставляет отдельную локальную кучу и глобальную кучу.
Если функция выполнится успешно, она выделит по крайней мере запрошенный объём памяти.
Если фактически выделено больше памяти, чем запрошено, процесс может использовать весь объём.
Для определения фактического количества выделённых байт используйте функцию _MemGlobalSize().
Если в куче нет достаточного количества свободного пространства для удовлетворения запроса, функция возвращает NULL.
Память, выделенная этой функцией, гарантированно выравнена на границе 8 байт.
Для выполнения динамически сгенерированного кода используйте функцию _MemVirtualAlloc() для выделения памяти и функцию _Mem_VirtualProtect() для предоставления доступа $PAGE_EXECUTE.
Для освобождения памяти используйте функцию _MemGlobalFree().
