Справочник функций

_WinNet_UseConnection

Подключает локальное устройство к сетевому ресурсу.

#include <WinNet.au3>
_WinNet_UseConnection ( $hWnd, $sLocalName, $sRemoteName [, $sUserName = 0 [, $sPassword = 0 [, $iType = 1 [, $iOptions = 1]]]] )

Параметры

$hWnd Дескриптор окна.
$sLocalName Имя локального устройства для переадресации, например "F:" или "LPT1". Строка обрабатывается независимо от регистра. Если 0, подключение к сетевому ресурсу происходит без переадресации локального устройства.
$sRemoteName Имя сетевого ресурса для подключения.
$sUsername [необязательный] Имя пользователя для подключения. Если 0, функция использует имя пользователя по умолчанию.
$sPassword [необязательный] Пароль для подключения. Если 0, используется пароль по умолчанию. Если строка пуста, пароль не используется.
$iType [необязательный] Указывает тип сетевого ресурса для подключения:     0 – Любой (только если $sLocalName пуст)     1 – Диск     2 – Принтер
$iOptions [необязательный] Параметры подключения. Может быть одним или несколькими из следующих:     1 – Подключение к сетевому ресурсу следует запомнить     2 – Операционная система может взаимодействовать с пользователем в целях аутентификации     4 – Система не использует параметры по умолчанию для имён пользователей или паролей без предоставления пользователю         возможности указать альтернативу.         Этот флаг игнорируется, если также не установлен бит 2 (интерактивный).     8 – Принудительная переадресация локального устройства при подключении     16 – Операционная система приглашает пользователя на аутентификацию через командную строку вместо графического интерфейса.         Этот флаг игнорируется, если также не установлен бит 2 (интерактивный).     32 – Если этот бит установлен и операционная система запрашивает учётные данные, они сохраняются диспетчером учётных данных.         Если диспетчер учётных данных отключен для сеанса входа вызывающей стороны, или если сетевой поставщик не поддерживает сохранение учётных данных, этот флаг игнорируется.         Этот флаг также игнорируется, если не установлен бит 5 (командная строка вместо графического интерфейса).

Возвращаемое значение

Успех: Возвращает массив со следующим форматом:     $aInfo[0] – Если True, подключение было выполнено с использованием переадресации локального устройства     $aInfo[1] – Если $sLocalName указывает локальное устройство, это имя локального устройства.         Если $sLocalName не указывает устройство и сеть требует переадресации локального устройства,         или если установлен бит 4 (принудительная переадресация), буфер получает имя переадресованного локального устройства.         В противном случае имя, скопированное в буфер, является имя удалённого ресурса.
Ошибка: Устанавливает @error ≠ 0.
