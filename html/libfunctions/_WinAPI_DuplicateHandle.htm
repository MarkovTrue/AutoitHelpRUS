_WinAPI_DuplicateHandle

Дублирует дескриптор объекта.

#include <WinAPIHObj.au3>
_WinAPI_DuplicateHandle ( $hSourceProcessHandle, $hSourceHandle, $hTargetProcessHandle, $iDesiredAccess, $iInheritHandle, $iOptions )

Параметры

$hSourceProcessHandle Дескриптор процесса с дубликатом дескриптора.
$hSourceHandle Задаёт дескриптор для дублирования.
$hTargetProcessHandle Дескриптор процесса, который должен получить дублированный дескриптор.
$iDesiredAccess Запрашиваемые права доступа для нового дескриптора.
$iInheritHandle Переменная, указывающая, является ли дескриптор наследуемым.
$iOptions Дополнительные действия.

Возвращаемое значение

Успех: Возвращает новый дескриптор.
Ошибка: Возвращает 0; вызовите _WinAPI_GetLastError() для получения подробных сведений об ошибке.

См. также

_WinAPI_CloseHandle, _WinAPI_OpenProcess

Дополнительно

Искать DuplicateHandle в библиотеке MSDN

Пример

#include <WinAPIHObj.au3>

Local $hCurProcessPseudo = _WinAPI_GetCurrentProcess()
ConsoleWrite("Pseudo handle for current process = " & $hCurProcessPseudo & @CRLF)

Local $hCurProcess = _WinAPI_DuplicateHandle($hCurProcessPseudo, $hCurProcessPseudo, $hCurProcessPseudo, Default, True, $DUPLICATE_SAME_ACCESS)
ConsoleWrite("Real handle for current process = " & $hCurProcess & @CRLF)

; ...

; Закрытие дескриптора, когда он больше не нужен
_WinAPI_CloseHandle($hCurProcess)
