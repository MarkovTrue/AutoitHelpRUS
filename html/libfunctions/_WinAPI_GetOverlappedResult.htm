_WinAPI_GetOverlappedResult

Получает результаты перекрытой операции.

#include <WinAPIFiles.au3>
_WinAPI_GetOverlappedResult ( $hFile, $tOverlapped, ByRef $iBytes [, $bWait = False] )

Параметры

$hFile Дескриптор файла, именованного канала или устройства связи. Это тот же дескриптор, который был
указан при запуске перекрытой операции вызовом ReadFile, WriteFile, ConnectNamedPipe,
TransactNamedPipe, DeviceIoControl или WaitCommEvent.
$tOverlapped Структура $tagOVERLAPPED или указатель на неё, который был указан при запуске перекрытой операции.
$iBytes Число байтов, фактически переданных при операции чтения или записи.
Для операции TransactNamedPipe это число байтов, прочитанных из канала.
Для операции DeviceIoControl это число байтов данных, возвращённых драйвером устройства.
Для операции ConnectNamedPipe или WaitCommEvent это значение не определено.
$bWait [необязательный] Если True, функция не возвращает значение до завершения операции.
Если False и операция всё ещё ожидается, функция возвращает False и функция GetLastError вернёт ERROR_IO_INCOMPLETE.

Возвращаемое значение

Успех: Возвращает True.
Ошибка: Возвращает False; вызовите _WinAPI_GetLastError() для получения подробных сведений об ошибке.
