Имя и путь файла

Если параметр функции назван "filename", подразумевается имя файла с полным путём. Хотя функции AutoIt часто принимают и относительный путь, корректная работа не гарантирована, поскольку некоторые файловые функции Windows API требуют полностью определённого пути. Если путь не указан и используется только простое имя файла, AutoIt создаст или будет искать его в текущей рабочей директории – её путь содержится в макросе @WorkingDir.

Рабочая директория при старте скрипта устанавливается в @ScriptDir и может быть изменена в ходе выполнения вызовом FileChangeDir. Также рабочая директория может изменяться автоматически некоторыми функциями – например, FileOpen/SaveDialog при успешном возврате или Run при указании параметра "workingdir".

На странице MSDN Naming Files, Paths, and Namespaces описаны подробности правильного написания имён файлов и путей.
