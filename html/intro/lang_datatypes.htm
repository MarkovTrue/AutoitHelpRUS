Типы данных

AutoIt использует реализацию Variant для представления переменных и констант. Переменные и константы могут содержать числовые или строковые данные; AutoIt сам решает, как интерпретировать тип данных в зависимости от ситуации, хотя при необходимости конкретный тип можно задать принудительно.

Это значит, что при использовании арифметического оператора над двумя переменными они трактуются как числа; при конкатенации (соединении) – как строки. Тип данных результата операции может не совпадать с типом исходных переменных:

Например, использование * для умножения двух значений возвращает число:

10 * 20 : результат – число 200
10 * "20" : результат – число 200
"10" * "20" : результат – число 200

А использование & для конкатенации двух значений возвращает строку:

10 & 20 : результат – строка "1020"

В каждом случае неявно вызывается функция Number() или String() для преобразования значения к нужному типу. Это может приводить к неожиданным результатам. Например, если строковое значение не содержит корректного числа, оно считается равным 0:

10 * "fgh" : результат – число 0.

Будьте внимательны с этим автоматическим преобразованием – оно может вызывать проблемы, если явно не задавать точный тип содержимого переменной. Например, значения, прочитанные из .ini-файла, ВСЕГДА строки, поэтому простое сравнение между ними может давать странные результаты – строки сравниваются лексикографически:

"11" считается меньше "2"

а не численно

11 считается больше 2

Рекомендуется использовать функции Number() и String(), чтобы привести содержимое переменных и констант к одному формату перед сравнением.

Помните, что для определения фактического типа переменных, литералов и констант можно использовать VarGetType().

Числа

Поддерживаются знаковые целые и числа с плавающей точкой: Int32, Int64 и Double. Битность целого зависит от его величины и подбирается автоматически.

Числа могут быть обычными десятичными: 2, 4.566, -7.

Также поддерживается научная нотация: например, 1.5e3 вместо 1500.

Целые числа можно представлять в шестнадцатеричной нотации, поставив перед ними 0x: например, 0x409 или 0x4fff.

Enum – ключевое слово языка, но не отдельный тип данных, поскольку перечисляемые значения напрямую отображаются на целые числа.

Строки

Нативная строка AutoIt использует набор символов UCS2 (первые 64K кодовых точек Unicode, также называемые BMP). Строки заключаются в двойные кавычки, например "this".

$s = "abc"
$s = "déçu"
$s = "" ; пустая строка

Если в строке нужна сама двойная кавычка, удвойте её:

"here is a ""double-quote"" - ok?"

Также можно использовать одинарные кавычки, например 'this' и 'here is a ' 'single-quote' ' – ok?'

Можно смешивать типы кавычек, чтобы упростить запись и не удваивать кавычки. Например, если в строках много двойных кавычек, объявляйте их в одинарных:

'This "sentence" contains "lots" of "double-quotes" does it not?'

гораздо проще, чем

"This ""sentence"" contains ""lots"" of ""double-quotes"" does it not?"

При вычислении в строках может выполняться подстановка переменных Env или Var в зависимости от настроек функции Opt().

Для кодировки исходных файлов рекомендуется использовать UTF8. Подробности преобразования строк в/из UCS2, UTF8 и т. д., а также работы с кодовыми страницами см. здесь.

Chr(0) внутри строк обрабатывается как любой другой символ. Только функции StringIs...() прекращают проверку при встрече Chr(0). Внутренне для ускорения используется функция C. StringIsSpace() не останавливается – для согласованности с StringStripWS().

Логические значения (Booleans)

Логические значения, и существуют только два: True и False. Их можно присваивать переменным и константам и использовать с операторами And, Or и Not.

Примеры:

$bBoolean1 = True
$bBoolean2 = False
$bBoolean3 = $bBoolean1 And $bBoolean2 : результат $bBoolean3 = False

$bBoolean1 = False
$bBoolean2 = Not $bBoolean1 : результат $bBoolean2 = True

Строки и числа можно использовать как логические значения. Пустая строка "" равна логическому False, как и число 0. Любое другое числовое значение равно логическому True.

Пример:

$iNumber1 = 0
$bBoolean1 = True
$bBoolean2 = $iNumber1 And $bBoolean1 : результат $bBoolean2 = False

Настоятельно не рекомендуется использовать Booleans в арифметических выражениях, но в этом случае действуют следующие правила:

Логическое True преобразуется в численное значение 1.
Логическое False преобразуется в численное значение 0.

Пример:

$bBoolean1 = True
$iNumber1 = 100
$iNumber2 = $bBoolean1 + $iNumber1 : результат $iNumber2 = 101

Конкатенация Boolean со строкой преобразует логическое значение в соответствующую строку "True" или "False":

Пример:

$bBoolean1 = True
$sString1 = "Test is: "
$sString2 = $sString1 & $bBoolean1 : результат $sString2 = "Test is: True"

Однако при сравнении строк с логическими значениями действуют такие правила:

Только пустая строка ("") считается логическим False.
Любое другое строковое значение (включая строку "0") считается логическим True.

Двоичные данные (Binary)

Тип Binary может хранить любое значение байта. При сохранении в строковую переменную преобразуется в шестнадцатеричное представление. Пример:

$dBin = Binary("abc")
$sString = String($dBin) : результат $sString = "0x616263"

Указатель (Pointer)

Типы Pointer хранят адрес памяти размером 32 или 64 бита в зависимости от версии AutoIt. При сохранении в строковую переменную преобразуются в шестнадцатеричное представление. Дескрипторы окон (HWnd), возвращаемые WinGetHandle(), относятся к типу указателя.

Переменные, созданные через DllStructCreate(), можно использовать как указатель, если в DllCall() используется тип "struct*". Указатель, полученный из DllStructGetPtr(), может быть передан в такой параметр с типом "struct*".

DllStruct

Этот тип данных используется для сопоставления данных AutoIt со структурами C/C++ при вызове функций в DLL.

Ключевые слова

AutoIt использует 4 ключевых слова как типы данных:

True        логическое True – например, 7 = 7.
False       логическое False – например, 5 >= 14.
Default   используется только для присвоения значения по умолчанию необязательным аргументам функции.
Null        неизвестное значение. (Обратите внимание: это ключевое слово НЕ является частью трёхзначной логики, как в других языках, например SQL.)

Объект (Object)

Object относится к "Component Object Model" (Microsoft). Создав объект, можно использовать AutoIt-код для взаимодействия с ним через различные свойства и/или методы объекта. Подробности использования COM в AutoIt см. здесь.

Function и User Function

В AutoIt есть как нативные функции, так и пользовательские; первые – часть ядра AutoIt, вторые – AutoIt-скрипты в include-библиотеках или собственно в вашем скрипте. Оба типа являются объектами первого класса – их можно присваивать переменным, использовать как параметр и/или возвращаемое значение другой функции. Подробнее см. здесь.

Массивы и Maps

В AutoIt есть 2 типа коллекций: массивы и Maps.

Массивы намного быстрее для произвольного доступа и могут иметь несколько измерений – размер измерения фиксируется при объявлении (хотя его можно изменить через ReDim()). Массивы индексируются целыми числами, отражающими порядок элементов, начиная с element[0]; элементы извлекаются по целочисленному индексу или перебираются простым циклом For...Next.

Maps лучше подходят для записей/словарей и имеют единственное измерение. Они индексируются целыми или строковыми ключами (целые не отражают порядок элементов) и динамически меняют размер при добавлении/удалении значений. К значению можно обращаться только по исходному ключу – ключи можно перебирать функцией MapKeys().

Подробнее см. здесь.

Типы данных и диапазоны

В таблице ниже приведены внутренние подтипы variant и их диапазоны.

Подтип данных Диапазон и примечания
Int32 32-битное знаковое целое число.
Int64 64-битное знаковое целое число.
Double Число с плавающей точкой двойной точности.
String Может содержать строки длиной до 2 147 483 647 символов.
Binary Двоичные данные, до 2 147 483 647 байтов.
Pointer Указатель на адрес памяти. 32 или 64 бита в зависимости от используемой версии AutoIt.

Некоторые функции AutoIt работают только с 32-битными числами (например, BitAND()) и автоматически преобразуют значения – такие функции отмечены в документации.
