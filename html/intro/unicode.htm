Поддержка Unicode

Поддерживаемые форматы текстовых файлов и скриптов и их обозначения в популярных редакторах:

Обозначение в AutoIt Notepad Notepad++ SciTE (стандартный редактор AutoIt)
UTF8 без BOM ANSI или UTF-8 в зависимости от содержимого (при сохранении BOM добавится принудительно) UTF-8 без BOM UTF-8
UTF8 с BOM UTF-8 UTF-8 UTF-8 с BOM
ANSI ANSI ANSI 8 bit / Code Page Property
UTF16 Little Endian Unicode UCS-2 Little Endian UCS-2 Little Endian
UTF16 Little Endian без BOM Unicode (при сохранении BOM добавится принудительно) UCS-2 Little Endian без BOM (нельзя выбрать) Не поддерживается
UTF16 Big Endian Unicode big endian UCS-2 Big Endian UCS-2 Big Endian
UTF16 Big Endian без BOM Не поддерживается UCS-2 Big Endian без BOM (нельзя выбрать) Не поддерживается

 

Рекомендуемый формат скрипта – UTF-8 с BOM: он лучше всего работает с notepad.exe и редактором SciTE для AutoIt и защищает от случайного сохранения скриптов в неподходящей кодовой странице.

Форматы ANSI не рекомендуются – они могут вызывать проблемы при запуске на машинах с другой локалью.

UTF16 BE или LE без BOM не рекомендуются. Они слабо поддерживаются. Даже с BOM они не очень распространены и занимают намного больше места на диске, чем UTF8.

  • Файловые операции с текстовыми файлами, которые открыты не через FileOpen() с явными флагами Unicode, автоматически определяют кодировку, как и большинство современных редакторов. Это касается всех файловых функций, принимающих имя файла, например FileRead("filename.txt"). Подробнее:
    • Файлы с BOM открываются в соответствующем BOM режиме. Проверяются BOM для UTF-8 и UTF-16.
    • Файлы UTF-8 и UTF-16 без BOM автоматически определяются и открываются в соответствующем режиме.
    • Файлы с нулевыми байтами по умолчанию открываются в режиме Binary ($FO_BINARY) (если только не определены как корректный UTF-16). Раньше они открывались в ANSI. Чтобы переопределить, используйте флаг $FO_ANSI.
    • Файлы, содержащие только символы 1–127, по умолчанию открываются в UTF-8 без BOM ($FO_UTF8_NOBOM). Раньше они открывались в ANSI. Чтобы переопределить, используйте флаг $FO_ANSI.
    • Файлы, содержащие только символы 1–255, по умолчанию открываются в режиме ANSI ($FO_ANSI).
    • Из-за вышесказанного FileGetEncoding() теперь возвращает 512 ($FO_ANSI) или 256 ($FO_UTF8_NOBOM) вместо 0, которое не было задокументировано, но обозначало ANSI.

    Текущие ограничения

    Есть несколько частей AutoIt, которые ещё не имеют полной поддержки Unicode:

    • Send и ControlSend – вместо них используйте ControlSetText или функции буфера обмена.
    • Операции с консолью преобразуются в ANSI.

    Эти ограничения будут устранены в будущих версиях, если получится.
