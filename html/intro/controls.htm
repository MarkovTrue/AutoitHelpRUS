Элементы управления

Одна из лучших возможностей AutoIt v3 – способность работать напрямую с определёнными типами контролов окна. Почти всё, что вы видите в окне, является каким-либо контролом. Кнопки, списки, поля ввода и текстовые метки – примеры контролов. Фактически Notepad – это один большой контрол "Edit"! Поскольку AutoIt работает напрямую с контролом, такой способ автоматизации надёжнее, чем простая отправка нажатий клавиш.


Примечание: AutoIt работает только со стандартными контролами Microsoft. Некоторые приложения создают собственные контролы, которые могут выглядеть как стандартные MS-контролы, но плохо поддаются автоматизации. Экспериментируйте!

 

С помощью AutoIt Window Info Tool можно перемещать мышь по интересующему окну и получать информацию о контроле, который сейчас находится под курсором.

В качестве параметра controlID, используемого в большинстве функций Control...(), можно применять специальное описание. Это описание позволяет идентифицировать контрол по следующим свойствам:

  • ID - внутренний идентификатор контрола. Control ID – это внутренний числовой идентификатор, который Windows присваивает каждому контролу. Обычно это лучший способ идентификации. Получить Control ID можно через AutoIt Window Info Tool, программы для слабовидящих, инструменты и API Microsoft.
  • TEXT - текст на контроле, например "&Next" на кнопке.
  • CLASS - внутреннее имя класса контрола, например "Edit" или "Button".
  • CLASSNN - значение ClassnameNN, как в предыдущих версиях AutoIt, например "Edit1".
  • NAME – внутреннее имя WinForms .NET Framework (если доступно).
  • REGEXPCLASS - имя класса контрола, заданное регулярным выражением.
  • X \ Y \ W \ H – позиция и размер контрола.
  • INSTANCE - номер экземпляра (с 1) при совпадении всех заданных свойств.

Одно или несколько свойств указываются в параметре controlID команды контрола в формате:

[PROPERTY1 : Value1; PROPERTY2:Value2]

Примечание: если этот специальный формат не используется, параметр интерпретируется как control ID (если число) или как ClassnameNN/текст контрола (если строка). Хотя специальный формат более многословен, он гораздо менее неоднозначен.

Если значение должно содержать ";", его нужно удвоить.

 

Например, отправить текст в первый контрол Edit окна Notepad:

ControlSend("Untitled - Notepad", "", "[CLASS:Edit; INSTANCE:1]", "This is some text")

или

ControlSend("Untitled - Notepad", "", "[CLASSNN:Edit1]", "This is some text")

или

ControlSend("Untitled - Notepad", "", "Edit1", "This is some text")

 

Например, кликнуть по контролу с ID 254 в окне "My Window":

ControlClick("My Window", "", "[ID:254]")

или

ControlClick("My Window", "", 254)

 

Например, установить текст контрола .NET WinForms "textBoxFolder" в значение "C:\Some\Folder":

ControlSetText("My Window", "", "[NAME:textBoxFolder]", "C:\Some\Folder")

 

Например, кликнуть по второму экземпляру контрола "Button" с текстом "Finish":

ControlClick("My Window", "", "[CLASS:Button; TEXT:Finish; INSTANCE:2]")

 

 

Дескриптор контрола (HWND)

С помощью функции ControlGetHandle можно определить дескриптор или HWND контрола. Дескриптор – это уникальный идентификатор, который Windows присваивает контролу. Дескриптор меняется каждый раз при создании контрола. Этот способ обращения к контролам обычно предназначен для пользователей, знакомых с работой с дескрипторами.
