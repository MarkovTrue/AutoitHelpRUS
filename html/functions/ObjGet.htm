Справочник функций

ObjGet

Возвращает ссылку на COM-объект из существующего процесса или файла.

ObjGet ( "filename" [, "classname" [, instance]] )

Параметры

filename Полный путь к файлу, содержащему объект (см. раздел Примечания).
classname [необязательный] Идентификатор класса. Может быть задан как ProgID или строковое представление CLSID.
instance [необязательный] Экземпляр объекта для ROT-объектов одного и того же (со)класса.

Возвращаемое значение

Успех: Возвращает объект.
Ошибка: Устанавливает @error ≠ 0.

Примечания

Если используется только идентификатор класса, параметр filename можно оставить пустым, но не опустить.
Для доступа к объектам из таблицы выполняемых объектов (ROT) передайте пустую строку; в этом случае третий параметр задаёт нужный экземпляр.
Во всех остальных случаях третий параметр игнорируется.

Если задано имя файла, classname необязателен – он нужен только для загрузки объекта конкретного класса.
Параметр classname не имеет значения по умолчанию: если указан, он обрабатывается.

Подробнее об объектах – в справочнике Obj/COM.

См. также

GUICtrlCreateObj, IsObj, ObjCreate, ObjEvent, ObjName

Пример

Пример 1

#include <MsgBoxConstants.au3>

; Пример получения объекта по имени класса.
; ; Для успешного выполнения примера Excel должен быть запущен.

Example()

Func Example()
    Local $oExcel = ObjGet("", "Excel.Application") ; Получить существующий объект Excel.

    If @error Then
        MsgBox($MB_SYSTEMMODAL, "", "Excel File Test" & @CRLF & "Error Getting an active Excel Object. Error code: " & Hex(@error, 8))
        Return False
    EndIf

    $oExcel.Visible = 1 ; Сделать Excel видимым.
    $oExcel.workbooks.add ; Добавить новую книгу.
EndFunc   ;==>Example

Пример 2

; Пример получения объекта по имени файла.
; ; В каталоге скрипта должен существовать файл Worksheet.xls.
; Это необходимо для работы примера.

#include <MsgBoxConstants.au3>

Example()

Func Example()
    Local $sFileName = @ScriptDir & "\Worksheet.xls"

    If Not FileExists($sFileName) Then
        MsgBox($MB_SYSTEMMODAL, "", "Excel File Test" & @CRLF & "Can't run this test, because you didn't create the Excel file " & $sFileName)
        Return False
    EndIf

    Local $oExcelDoc = ObjGet($sFileName) ; Получить объект Excel из существующего файла.

    If IsObj($oExcelDoc) Then

        ; Совет: раскомментируйте эти строки, чтобы сделать Excel видимым (авт.: DaleHohm).
        ; $oExcelDoc.Windows(1).Visible = 1; Отобразить первый лист книги.
        ; $oExcelDoc.Application.Visible = 1; Отобразить приложение (без этого Excel завершится).

        Local $sString = "" ; Строка для вывода.

        For $oProperty In $oExcelDoc.BuiltinDocumentProperties
            $sString &= $oProperty.Name & ":" & $oProperty.Value & @CRLF
        Next

        MsgBox($MB_SYSTEMMODAL, "", "Excel File Test:" & @CRLF & "The document properties of " & $sFileName & " are:" & @CRLF & @CRLF & $sString)

        $oExcelDoc.Close ; Закрыть документ Excel.

    Else
        MsgBox($MB_SYSTEMMODAL, "", "Excel File Test" & @CRLF & "Error: Could not open " & $sFileName & " as an Excel Object.")
    EndIf
EndFunc   ;==>Example
