Справочник функций

ProcessWaitClose

Приостанавливает выполнение скрипта до завершения указанного процесса.

ProcessWaitClose ( "process" [, timeout = 0] )

Параметры

process Имя или PID проверяемого процесса.
timeout [необязательный] Время ожидания в секундах. По умолчанию ожидание неограниченно.

Возвращаемое значение

Успех: Возвращает 1; @extended устанавливается в код завершения процесса (см. раздел Примечания).
Ошибка: Возвращает 0, если истёк таймаут ожидания.

Примечания

Имена процессов указываются без пути, например "notepad.exe" или "winword.exe".

PID – уникальный номер, идентифицирующий процесс. Его можно получить через ProcessExists() или Run().

Процесс опрашивается приблизительно каждые 250 миллисекунд.

Если процесс уже завершился, возвращается 1, а @extended устанавливается в 0xCCCCCCCC.

См. также

ProcessClose, ProcessExists, ProcessList, ProcessWait, RunAsWait, RunWait, WinGetProcess, WinWaitClose

Пример

; Ожидать завершения всех экземпляров Блокнота.
ProcessWaitClose("notepad.exe")

; Ожидать, пока пользователь закроет этот конкретный экземпляр Блокнота.
Local $iPID = Run("notepad.exe")
ProcessWaitClose($iPID)
