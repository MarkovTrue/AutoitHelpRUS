Справочник функций

DriveMapAdd

Подключает сетевой диск.

DriveMapAdd ( "device", "remote share" [, flags = 0 [, "user" [, "password"]]] )

Параметры

device Устройство для подключения, например "O:" или "LPT1:". Если передать пустую строку, подключение будет создано без назначения конкретной буквы диска. При указании "*" автоматически выбирается свободная буква диска.
remote share Удалённый сетевой ресурс для подключения в форме "\\server\share".
flags [необязательный] Комбинация следующих значений:
   $DMA_DEFAULT (0) = по умолчанию
   $DMA_PERSISTENT (1) = постоянное подключение
   $DMA_AUTHENTICATION (8) = показать диалог аутентификации при необходимости

Константы определены в "AutoItConstants.au3".
user [необязательный] Имя пользователя для подключения в формате "username" или "domain\username".
password [необязательный] Пароль для подключения.

Возвращаемое значение

Успех: Возвращает 1. (см. раздел Примечания).
Ошибка: Возвращает 0, если не удалось создать подключение, и устанавливает @error ≠ 0.
@error: 1 – неопределённая / иная ошибка. @extended содержит код возврата Windows API
2 – доступ к удалённому ресурсу запрещён
3 – устройство уже назначено
4 – неверное имя устройства
5 – неверный сетевой ресурс
6 – неверный пароль

Примечания

Примечание: при использовании "*" в параметре устройства вместо 1 или 0 будет возвращена выбранная буква диска, например "U:". В случае ошибки при использовании "*" возвращается пустая строка "".

Если указаны пользователь/пароль, они будут переданы удалённому компьютеру для проверки учётных данных.

См. также

DriveMapDel, DriveMapGet

Пример

#include <AutoItConstants.au3>

; Подключить диск X к \\myserver\stuff под текущим пользователем
DriveMapAdd("X:", "\\myserver\stuff")

; Подключить диск X к \\myserver2\stuff2, используя пользователя "jon" из домена "domainx" с паролем "tickle"
DriveMapAdd("X:", "\\myserver2\stuff2", $DMA_DEFAULT, "domainx\jon", "tickle")
